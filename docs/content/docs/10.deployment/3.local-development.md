## Quick Setup

You need two processes running in separate terminals:

::code-group
```bash [Terminal 1 — Convex]
npx convex dev
```
```bash [Terminal 2 — Nuxt]
nuxt dev --dotenv .env.local
```
::

## Environment File

Create a `.env.local` file in your project root:

```bash [.env.local]
CONVEX_URL=https://your-project.convex.cloud
```

::note
The `--dotenv .env.local` flag tells Nuxt to load this file. Convex Dev also reads `.env.local` automatically.
::

## How It Works

- **Convex Dev** watches your `convex/` directory and hot-pushes function changes to the Convex backend
- **Nuxt Dev** hot-reloads your frontend with HMR
- Changes to Convex functions are reflected in your app within seconds
- The `--dotenv .env.local` flag loads `CONVEX_URL` into Nuxt's runtime config

## Convex Dashboard Variables

For local development, set these in your Convex Dashboard:

| Variable | Value |
|----------|-------|
| `SITE_URL` | `http://localhost:3000` |
| `BETTER_AUTH_SECRET` | Any random string for development |

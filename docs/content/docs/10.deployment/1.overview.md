## Deployment Checklist

::steps{level="4"}

#### Deploy Convex Functions

Push your Convex functions to production:

```bash
npx convex deploy
```

#### Set Convex Environment Variables

In your [Convex Dashboard](https://dashboard.convex.dev) → Settings → Environment Variables:

- `SITE_URL` — your production app URL (e.g., `https://myapp.com`)
- `BETTER_AUTH_SECRET` — your auth signing secret
- OAuth credentials (if applicable)

#### Set Nuxt Environment Variables

In your hosting provider's dashboard:

- `NUXT_PUBLIC_CONVEX_URL` — your Convex deployment URL (e.g., `https://happy-animal-123.convex.cloud`)

#### Build and Deploy

```bash
nuxt build
```

Deploy the `.output` directory to your hosting provider.

::

## Key Points

- The Convex deployment URL (`CONVEX_URL`) is the same in development and production — it points to your Convex project
- `SITE_URL` in the Convex Dashboard must match your production domain for auth cookies to work
- The auth proxy works automatically — it runs as part of your Nuxt server
- All composables and components work the same in production as in development

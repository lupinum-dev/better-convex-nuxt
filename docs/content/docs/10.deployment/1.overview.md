## Core Production Shape

A deployed Trellis app usually has:

- one Nuxt deployment
- one Convex deployment
- Better Auth env where auth is enabled
- optional trusted-caller secret for server-to-server and MCP surfaces

## Minimum Production Concerns

- `CONVEX_URL` points the app at the correct Convex deployment
- `SITE_URL` matches the public app origin
- `BETTER_AUTH_SECRET` is set when auth is enabled
- `CONVEX_SITE_URL` is available when auth token exchange cannot be derived safely

## Advanced Production Concerns

Only some apps need:

- `CONVEX_TRUSTED_CALLER_KEY`
- MCP runtime setup
- SSR auth cache storage backed by something stronger than in-memory Nitro storage

## What Not To Forget

- browser auth and trusted-caller auth are different trust paths
- local-development shortcuts should not leak into production assumptions
- if observability matters, configure it at deploy time instead of treating it as a local-only feature

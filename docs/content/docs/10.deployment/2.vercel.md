## Setup

::steps{level="4"}

#### Add Environment Variables

In your Vercel project settings → Environment Variables:

| Variable | Value |
|----------|-------|
| `NUXT_PUBLIC_CONVEX_URL` | `https://your-project.convex.cloud` |

#### Update Convex Dashboard

Set `SITE_URL` to your Vercel production domain:

```
https://myapp.vercel.app
```

#### Deploy

Vercel automatically detects Nuxt and uses the correct build command (`nuxt build`).

::

## Preview Deployments

Vercel creates unique URLs for each preview deployment (e.g., `https://myapp-abc123.vercel.app`). Add a wildcard to trusted origins so auth works on previews:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    auth: {
      trustedOrigins: ['https://*.vercel.app']
    }
  }
})
```

::warning
For preview deployments, you also need to update `SITE_URL` in the Convex Dashboard to match the preview URL, or auth cookie validation may fail. Consider using a stable preview domain if possible.
::

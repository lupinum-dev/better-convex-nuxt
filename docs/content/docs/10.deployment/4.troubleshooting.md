## Common Issues

### "CONVEX_URL is not set"

**Cause:** The module can't find a Convex URL. Nuxt does not load `.env.local` by default -- it only reads `.env`.

**Fix:** Set `CONVEX_URL` in `.env.local` and run Nuxt with `--dotenv .env.local`:

```bash
nuxt dev --dotenv .env.local
```

### Auth Proxy Returns 404

**Cause:** The auth proxy route at `/api/auth/[...]` is not registered. This route is what forwards authentication requests from your app's domain to Convex (see [How It Works](/docs/guide/how-it-works#auth-flow-overview)).

**Fix:** Ensure `auth: true` (or auth config object) is set in your `nuxt.config.ts`:

```ts
convex: {
  auth: true
}
```

### CORS Errors

**Cause:** The auth proxy rejects requests from an untrusted origin. The proxy exists because session cookies must be set on your app's domain, not Convex's domain (`*.convex.site`) -- so all auth requests pass through it, and it validates the request origin.

**Fix:** Add your domain to `trustedOrigins`:

```ts
convex: {
  auth: {
    trustedOrigins: ['https://your-domain.com']
  }
}
```

### Token Exchange Fails

**Cause:** During SSR, the server plugin reads the session cookie and exchanges it for a Convex JWT via the auth proxy. This fails when `SITE_URL` in the Convex Dashboard doesn't match your app's origin, because Better Auth can't validate the request.

**Fix:** Set `SITE_URL` to your exact app origin:
- Development: `http://localhost:3000`
- Production: `https://myapp.com`

### Flash of Unauthenticated Content

**Cause:** Auth state hasn't settled before rendering. The module's [two-plugin architecture](/docs/guide/how-it-works#the-two-plugins) hydrates auth from server to client, but components must opt in to waiting for it.

**Fix:** Use auth-aware rendering:

```vue
<ConvexAuthenticated>
  <Dashboard />
</ConvexAuthenticated>
<ConvexUnauthenticated>
  <LoginPage />
</ConvexUnauthenticated>
```

Or use `await` with queries to block rendering:

```vue
<script setup lang="ts">
const { data } = await useConvexQuery(api.users.me, {})
</script>
```

### SSR Hydration Mismatch

**Cause:** Server and client render different content. The module uses [two plugins](/docs/guide/how-it-works#the-two-plugins) -- the server plugin fetches data via HTTP and stores it in `useState`, then the client plugin picks it up during hydration. A mismatch means the client rendered with different data than the server.

**Fix:** Ensure queries used in the template have `server: true` (the default) and use `await` to block rendering until data is ready.

### "Failed to fetch auth token"

**Cause:** The server can't reach the Convex auth endpoint. The auth flow is: session cookie → auth proxy → Convex HTTP action → JWT. Any break in this chain produces this error.

**Fix:**
1. Check that `CONVEX_URL` is correct
2. Verify the Convex site URL is accessible (auto-derived from `CONVEX_URL`)
3. Check that `BETTER_AUTH_SECRET` is set in both Nuxt and Convex Dashboard
4. Ensure `SITE_URL` in Convex Dashboard matches your app origin

### Permissions Query Fails

**Cause:** The permission context query doesn't exist or has errors.

**Fix:**
1. Ensure `permissions: true` is set in config
2. Run `npx convex dev` to sync your functions
3. Check that `api.auth.getPermissionContext` exists and returns `{ role, userId }`

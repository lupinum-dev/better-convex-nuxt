title: "Server Queries"
description: "Run Convex queries from Nuxt server routes and API endpoints."

## Overview

`serverConvexQuery` lets you run Convex queries from server routes, API endpoints, and server middleware. It handles authentication automatically using the request's session cookie.

## Basic Usage

```ts [server/api/tasks.get.ts]
import { api } from '~~/convex/_generated/api'

export default defineEventHandler(async (event) => {
  const tasks = await serverConvexQuery(api.tasks.list, { status: 'active' })
  return tasks
})
```

## Call Signatures

The function supports two signatures:

```ts
// Auto-resolves the H3 event (requires experimental.asyncContext in nuxt.config)
serverConvexQuery(query, args?, options?)

// Explicit event (always works)
serverConvexQuery(event, query, args?, options?)
```

::tip
If you use the auto-resolve form, enable `experimental: { asyncContext: true }` in your `nuxt.config.ts`.
::

## Options

::field-group
::field{name="auth" type="'auto' | 'required' | 'none'"}
Authentication mode. Default: `'auto'`.
- `'auto'` — Uses session cookie if available, runs without auth otherwise
- `'required'` — Throws if no auth token can be resolved
- `'none'` — Never attaches auth, runs as unauthenticated
::
::field{name="authToken" type="string"}
Explicit JWT token override. Bypasses the cookie-based auth flow.
::
::

## Auth Modes

```ts [server/api/public-data.get.ts]
// No auth needed
export default defineEventHandler(async () => {
  return await serverConvexQuery(api.posts.listPublic, {}, { auth: 'none' })
})
```

```ts [server/api/my-data.get.ts]
// Require auth — throws if user is not authenticated
export default defineEventHandler(async (event) => {
  return await serverConvexQuery(event, api.users.me, {}, { auth: 'required' })
})
```

## Example: Server Middleware

```ts [server/middleware/load-user.ts]
import { api } from '~~/convex/_generated/api'

export default defineEventHandler(async (event) => {
  try {
    const user = await serverConvexQuery(event, api.users.me, {}, { auth: 'auto' })
    event.context.user = user
  } catch {
    // No auth — continue without user
  }
})
```

title: "Private Bridge"
description: "Make privileged server-only calls to Convex bypassing user authentication."
---

## Overview

The private bridge allows your Nuxt server to make privileged Convex calls using a shared secret key, bypassing user authentication entirely. This is useful for admin operations, cron jobs, and webhook handlers that need full access to your Convex backend.

## Setup

::steps{level="4"}

#### Set the Bridge Key

Add a `PRIVATE_BRIDGE_KEY` environment variable to both your Nuxt app and Convex dashboard:

```bash [.env.local]
PRIVATE_BRIDGE_KEY=your-secret-key-here
```

Set the same value in your Convex dashboard environment variables.

#### Use in Server Routes

Use `serverConvexMutation` or `serverConvexAction` with the bridge key as the auth token:

```ts [server/api/admin/cleanup.post.ts]
import { api } from '~~/convex/_generated/api'

export default defineEventHandler(async (event) => {
  const bridgeKey = process.env.PRIVATE_BRIDGE_KEY
  if (!bridgeKey) throw createError({ statusCode: 500, message: 'Bridge key not configured' })

  await serverConvexMutation(
    event,
    api.admin.cleanupExpiredSessions,
    {},
    { auth: 'none', authToken: bridgeKey }
  )

  return { success: true }
})
```

::

## Use Cases

- **Admin operations**: bulk data updates, user management
- **Cron jobs**: scheduled cleanup, report generation
- **Webhooks**: processing events from external services (Stripe, GitHub)
- **Background tasks**: data migrations, batch processing

## Security

::warning
The private bridge key grants full access to your Convex backend. Never expose it to the client. Only use it in server routes and API endpoints.
::

- Store the key in environment variables, never in source code
- Use `auth: 'none'` to skip cookie-based auth when using the bridge key
- Validate incoming requests (e.g., webhook signatures) before executing privileged operations

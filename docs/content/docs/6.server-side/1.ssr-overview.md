## Overview

Better Convex Nuxt provides full SSR support. Queries run on the server during the initial render, and the client seamlessly picks up with real-time WebSocket subscriptions — no data refetch, no flash of loading state.

## The SSR Lifecycle

::steps{level="4"}

#### Request Arrives

A browser request hits the Nuxt server. The server plugin runs first.

#### Auth Token Resolution

The server reads the `better-auth.session_token` cookie from the request and exchanges it for a Convex JWT via the auth proxy. This JWT authenticates server-side queries.

#### Server-Side Query Execution

Queries with `server: true` (the default) execute via HTTP against the Convex backend. Results are stored in Nuxt's `useState` for hydration.

#### HTML Rendering

Nuxt renders the page with the fetched data already available. No loading spinners in the initial HTML.

#### Client Hydration

The client picks up the server-rendered data from `useState`. Auth state (token, user) is also hydrated — no flash of unauthenticated content.

#### WebSocket Subscription

The client plugin creates a `ConvexClient` and starts WebSocket subscriptions for all active queries. Data updates in real-time from this point forward.

::

## Server vs Client Queries

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

// Runs on server during SSR, then subscribes on client (default)
const { data: tasks } = await useConvexQuery(api.tasks.list, {})

// Skips SSR, only runs on client
const { data: stats } = await useConvexQuery(api.stats.live, {}, {
  server: false
})
</script>
```

## The `await` Pattern

Using `await` with `useConvexQuery` blocks rendering until data is ready:

```vue
<script setup lang="ts">
// Blocks SSR render until tasks are loaded
const { data: tasks } = await useConvexQuery(api.tasks.list, {})
// tasks.value is guaranteed to be available in the template
</script>
```

Without `await`, the component renders immediately with `data` as `null` and `status` as `'pending'`:

```vue
<script setup lang="ts">
// Non-blocking — renders immediately, data loads in background
const { data: tasks, status } = useConvexQuery(api.tasks.list, {})
</script>

<template>
  <div v-if="status === 'pending'">Loading...</div>
  <div v-else>{{ tasks }}</div>
</template>
```

## Global Defaults

Configure SSR behavior for all queries:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    query: {
      server: true,    // Run queries during SSR (default)
      subscribe: true  // Keep WebSocket subscriptions (default)
    }
  }
})
```

## Sessions

Enable sessions when MCP state should persist across calls:

```ts
export default defineNuxtConfig({
  modules: ['better-convex-nuxt', '@nuxtjs/mcp-toolkit'],
  mcp: {
    sessions: true,
  },
})
```

Then use `useMcpSession()` from `#convex/mcp` inside a generic MCP tool:

```ts
import { defineMcpTool, useMcpSession } from '#convex/mcp'
import { z } from 'zod'

interface SessionData {
  preferredSearch?: string
}

export default defineMcpTool({
  name: 'set-session-preference',
  inputSchema: {
    preferredSearch: z.string(),
  },
  handler: async ({ preferredSearch }) => {
    const session = useMcpSession<SessionData>()
    await session.set('preferredSearch', preferredSearch)
    return `Stored "${preferredSearch}" in this MCP session.`
  },
})
```

The playground app uses this pattern and verifies the `MCP-Session-Id` header in the MCP smoke suite.

## Dynamic Tool Registration

Use `useMcpServer()` when a session should gain or lose tools at runtime:

```ts
import { defineMcpTool, useMcpServer } from '#convex/mcp'
import { z } from 'zod'

export default defineMcpTool({
  name: 'register-session-shortcut',
  inputSchema: {
    name: z.string(),
    message: z.string(),
  },
  handler: async ({ name, message }) => {
    const mcp = useMcpServer()
    mcp.registerTool(name, { description: `Shortcut: ${message}` }, async () => message)
    return `Registered "${name}".`
  },
})
```

This is session-scoped when MCP sessions are enabled. The playground demo registers and removes tools in one live session to prove the behavior.

## Code Mode

Keep code mode opt-in. Use a dedicated handler when the tool catalog is large or workflows are routinely multi-step:

```ts
import { defineMcpHandler } from '#convex/mcp'
import createNote from './tools/create-note'
import deleteNote from './tools/delete-note'
import listNotes from './tools/list-notes'
import searchNotes from './tools/search-notes'
import updateNote from './tools/update-note'

export default defineMcpHandler({
  name: 'notes-agent',
  route: '/mcp/notes-agent',
  experimental_codeMode: true,
  tools: [listNotes, searchNotes, createNote, updateNote, deleteNote],
})
```

Recommendation:

- standard tool mode for small and medium catalogs
- evaluate code mode when the server exposes many tools or common tasks require several tool calls in sequence

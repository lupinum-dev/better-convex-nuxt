## Middleware

Middleware runs after auth/permissions/rate-limit checks but before the handler. Use it for audit logging, resource pre-fetching, or custom authorization:

```ts
export default defineConvexTool({
  schema,
  name: 'update-note',
  auth: 'required',

  middleware: async (args, ctx, next) => {
    // ctx.mcpAuth — resolved identity (or null)
    // ctx.event — H3 event
    // ctx.can(permission, resource?) — permission check

    console.log(`[audit] update-note: user=${ctx.mcpAuth?.userId}`)

    // Must call next() and return its result
    return next()
  },

  handler: async (args) => { ... },
})
```

### Blocking from Middleware

Return `wrapError()` instead of `next()` to block execution:

```ts
middleware: async (args, ctx, next) => {
  const note = await serverConvexQuery(api.notes.get, { id: args.id })
  if (!note) {
    return wrapError('not_found', `Note "${args.id}" not found.`)
  }

  if (!ctx.can('post.update', { ownerId: note.userId })) {
    return wrapError('auth', 'You do not have permission to update this note.')
  }

  return next()
}
```

### Middleware Contract

- Must return a `CallToolResult` (either from `next()` or from `wrapError`/`wrapSuccess`)
- If middleware returns `undefined` (forgetting `return next()`), the framework returns a `server` error
- Single function — compose multiple middleware internally if needed

## Enabled Guards

Conditionally show or hide a tool per-request:

```ts
export default defineConvexTool({
  schema,
  name: 'export-notes',
  auth: 'required',

  // Tool only appears in tools/list when authenticated
  enabled: (event) => {
    return !!event.context.mcpAuth
  },

  handler: async (args) => { ... },
})
```

Use cases:
- Hide admin tools from non-admin users
- Feature-flag new tools via environment variables
- Disable tools during maintenance

## Tags

Tag tools for filtering and categorization:

```ts
export default defineConvexTool({
  schema,
  name: 'bulk-delete-notes',
  tags: ['bulk', 'dangerous'],
  handler: async (args) => { ... },
})
```

Tags are metadata — the framework doesn't filter on them, but MCP clients and agent orchestrators can use them to select tool subsets.

## Operation Types & Annotations

The `operation` option controls auto-derived MCP annotations:

| Operation | `readOnlyHint` | `destructiveHint` | `idempotentHint` | `openWorldHint` |
|-----------|---------------|-------------------|------------------|----------------|
| `'query'` | `true` | `false` | `true` | `false` |
| `'mutation'` (default) | `false` | from `destructive` | `false` | `false` |
| `'action'` | `false` | from `destructive` | `false` | `true` |

- `query` — read-only, deterministic, idempotent. Safe to call repeatedly.
- `mutation` — writes to Convex. Deterministic (no external calls).
- `action` — may have external side effects (API calls, file writes). Non-deterministic.

### Overriding Annotations

```ts
export default defineConvexTool({
  schema,
  operation: 'query',
  annotations: {
    openWorldHint: true, // override: this query hits an external API
  },
  handler: async (args) => { ... },
})
```

Only specify overrides — the rest are derived from `operation`.

## Input Examples

Auto-generated from field `examples` in your schema metadata:

```ts
// Schema with examples
const meta = {
  description: 'Search notes',
  fields: {
    query: { description: 'Search text', examples: ['meeting', 'TODO'] },
  },
}
// → inputExamples: [{ query: 'meeting' }]
```

Override with explicit examples:

```ts
export default defineConvexTool({
  schema,
  inputExamples: [
    { query: 'meeting' },
    { query: 'TODO' },
    { query: 'project update' },
  ],
  handler: async (args) => { ... },
})
```

## Cache

Pass cache configuration through to `@nuxtjs/mcp-toolkit`:

```ts
export default defineConvexTool({
  schema,
  operation: 'query',
  cache: { ttl: 60 }, // cache for 60 seconds
  handler: async (args) => { ... },
})
```

Only appropriate for read-only tools with results that don't change frequently.

## Group

Tools are auto-grouped by directory. Override manually if needed:

```ts
export default defineConvexTool({
  schema,
  group: 'admin',
  handler: async (args) => { ... },
})
```

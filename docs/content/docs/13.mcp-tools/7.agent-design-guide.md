description: 'Write MCP tools that are easy for agents to choose and use correctly.'
---

## Name Tools Clearly

Good tool names are short verb-noun pairs that describe the action:

| Good | Bad | Why |
|------|-----|-----|
| `create-page` | `page-handler` | Describes the action, not the implementation |
| `list-published-pages` | `get-pages` | Specific about what is returned |
| `publish-page` | `update-page-status` | Matches how a user thinks about it |
| `delete-draft` | `remove` | Includes the resource type |

## Describe The Action, Not The Implementation

Tool descriptions help agents decide which tool to use. Write them from the user's perspective:

```ts
const createNote = defineArgs({
  description: 'Create a note and return its new ID',
  args: {
    title: v.string(),
  },
  meta: {
    title: {
      description: 'Short user-facing note title',
      examples: ['Sprint planning'],
    },
  },
})
```

Good descriptions: "Create a new draft page in the CMS", "Publish the selected draft page to the public site".

Bad descriptions: "Calls the publishPage mutation via the bridge", "Wrapper around internal.miniCmsBridge.publishPage".
```

## Prefer One Tool Per Action

Do not overload one tool with multiple modes. Give agents small, predictable operations:

- `create-note`
- `update-note`
- `delete-note`

## Return Structured Data

```ts
export default projectTool({
  call: internal.notes.createNote,
  schema: createNote,
  capability: 'createNote',
  respond: ({ result, args }) => ({
    data: { id: result, title: args.title },
    summary: `Created note "${args.title}"`,
  }),
  meta: {
    name: 'create-note',
  },
})
```

## Guard Destructive Tools

Use `destructive`, `preview`, `maxItems`, and `rateLimit` together when the operation can remove or rewrite data.

## Prefer Indexed Or Bounded Search Tools

For Convex-backed MCP tools:

- prefer `withIndex(...)` or search indexes for unbounded search/list operations
- avoid `.collect()` plus in-memory filtering for growing tables
- if the dataset is intentionally small in a demo, bound the read explicitly with `take(...)`

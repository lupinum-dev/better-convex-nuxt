description: "Write MCP tools that are easy for agents to choose and use correctly."
---

## Describe The Action, Not The Implementation

```ts
const createNote = defineArgs({
  description: 'Create a note and return its new ID',
  args: {
    title: v.string(),
  },
  meta: {
    title: {
      description: 'Short user-facing note title',
      examples: ['Sprint planning'],
    },
  },
})
```

## Prefer One Tool Per Action

Do not overload one tool with multiple modes. Give agents small, predictable operations:

- `create-note`
- `update-note`
- `delete-note`

## Return Structured Data

```ts
export default defineTool({
  schema: createNote,
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.notes.add, args)
    return ctx.ok({ id, title: args.title }, `Created note "${args.title}"`)
  },
})
```

## Guard Destructive Tools

Use `destructive`, `preview`, `maxItems`, and `rateLimit` together when the operation can remove or rewrite data.

## Prefer Indexed Or Bounded Search Tools

For Convex-backed MCP tools:

- prefer `withIndex(...)` or search indexes for unbounded search/list operations
- avoid `.collect()` plus in-memory filtering for growing tables
- if the dataset is intentionally small in a demo, bound the read explicitly with `take(...)`

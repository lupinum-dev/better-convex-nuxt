---
title: "Shared Schema"
description: "Define tool and handler args once, then reuse them across Convex and MCP."
---

## The Problem

Without a shared args definition, the same input shape gets repeated in Convex handlers, server validation, and MCP tools.

## `defineArgs`

```ts
// shared/schemas/note.ts
import { defineArgs } from 'better-convex-nuxt/schema'
import { v } from 'convex/values'

export const createNote = defineArgs({
  description: 'Create a note',
  args: {
    title: v.string(),
    content: v.string(),
  },
  meta: {
    title: {
      label: 'Title',
      description: 'Short note title',
      examples: ['Meeting notes'],
    },
    content: {
      label: 'Content',
      description: 'Main note body',
    },
  },
})
```

That one definition gives you:

- `createNote.validators` for Convex function args
- `createNote.meta` for tool descriptions or generated forms
- `createNote.zod` for a Zod view
- `createNote.parse()` for runtime validation

## Reuse In Convex

```ts
export const create = mutation({
  args: createNote.validators,
  handler: async (ctx, args) => {
    return await ctx.db.insert('notes', {
      ...args,
      createdAt: Date.now(),
    })
  },
})
```

## Reuse In MCP

```ts
import { defineTool } from '#convex/mcp'

export default defineTool({
  schema: createNote,
  auth: 'required',
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.notes.add, args)
    return ctx.ok({ id }, `Created note "${args.title}"`)
  },
})
```

## Why `shared/` Exists

`shared/` is not a Nuxt convention. It is a runtime-boundary folder.

Files there are imported by both:

- `convex/` files, bundled by Convex
- Nuxt `server/` files, bundled by Nitro

Keep them runtime-neutral so both build systems can load them safely.

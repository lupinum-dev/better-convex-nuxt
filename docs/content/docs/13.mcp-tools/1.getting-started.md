## Prerequisites

Your Nuxt project should already use `better-convex-nuxt`. Then add the MCP dependencies:

```bash
pnpm add @nuxtjs/mcp-toolkit convex-helpers zod
```

Enable MCP in `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  modules: ['better-convex-nuxt', '@nuxtjs/mcp-toolkit'],
})
```

## File Placement

MCP definitions live in the standard Nuxt MCP Toolkit directories. `#convex/mcp` re-exports both the Convex-aware `defineTool()` helper and the generic Nuxt MCP primitives.

```text
server/
  mcp/
    index.ts
    tools/
      list-notes.ts
      create-note.ts
    prompts/
      plan-note-workflow.ts
    resources/
      playground-mcp-guide.ts
```

The handler entrypoint is minimal:

```ts
import { defineMcpHandler } from '#convex/mcp'

export default defineMcpHandler({})
```

## First Tool

```ts
// server/mcp/tools/list-notes.ts
import { defineTool } from '#convex/mcp'
import { defineArgs } from 'better-convex-nuxt/args'
import { api } from '~~/convex/_generated/api'

const listNotes = defineArgs({
  description: 'List all notes',
  args: {},
})

export default defineTool({
  schema: listNotes,
  auth: 'optional',
  handler: async (_args, ctx) => {
    const notes = await ctx.query(api.notes.list, {})
    return ctx.ok({ notes }, `Found ${notes.length} notes`)
  },
})
```

The framework handles:

- input validation from your shared args definition
- JSON-schema generation for MCP clients
- structured success and error envelopes
- auth gating when `auth` is set
- trusted-caller injection automatically when `scoped: true`
- prompt/resource/handler exports from the same `#convex/mcp` entrypoint

## Mutation Tool

```ts
// server/mcp/tools/create-note.ts
import { defineTool } from '#convex/mcp'
import { defineArgs } from 'better-convex-nuxt/args'
import { api } from '~~/convex/_generated/api'
import { v } from 'convex/values'

const createNote = defineArgs({
  description: 'Create a note',
  args: {
    title: v.string(),
    content: v.string(),
  },
  meta: {
    title: { description: 'Short title', examples: ['Meeting notes'] },
    content: { description: 'Main body text' },
  },
})

export default defineTool({
  schema: createNote,
  auth: 'required',
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.notes.add, args)
    return ctx.ok({ id }, `Created note "${args.title}"`)
  },
})
```

## Prompt And Resource Example

```ts
// server/mcp/prompts/plan-note-workflow.ts
import { defineMcpPrompt } from '#convex/mcp'
import { z } from 'zod'

export default defineMcpPrompt({
  description: 'Plan an MCP note workflow',
  inputSchema: {
    goal: z.string().describe('User goal to accomplish'),
  },
  handler: async ({ goal }) =>
    `Use the note tools in this app to accomplish: ${goal}`,
})
```

```ts
// server/mcp/resources/playground-mcp-guide.ts
import { defineMcpResource } from '#convex/mcp'

export default defineMcpResource({
  uri: 'app://playground/mcp-guide',
  handler: async (uri: URL) => ({
    contents: [{
      uri: uri.toString(),
      mimeType: 'text/markdown',
      text: '# MCP Guide\n\nUse tools for actions, resources for context, prompts for reusable workflows.',
    }],
  }),
})
```

## Why `defineTool` Looks Different From Convex Functions

Nuxt can generate `#convex/mcp` because MCP tools run inside the Nuxt server runtime.

Convex function files run in Convex's runtime, so they import raw `query()` / `mutation()` and your own `convex/auth/*` helpers directly.

## Prerequisites

Your Nuxt project should already have `better-convex-nuxt` configured. Then add the MCP dependencies:

```bash
pnpm add @nuxtjs/mcp-toolkit convex-helpers zod
```

Enable MCP in `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  modules: ['better-convex-nuxt', '@nuxtjs/mcp-toolkit'],
})
```

## File Placement

MCP tools live in `server/mcp/tools/`. Each file exports a default tool definition and is auto-discovered by `@nuxtjs/mcp-toolkit`.

```
server/
  mcp/
    index.ts          # MCP handler (required)
    tools/
      list-notes.ts   # → tool "list-notes"
      create-note.ts  # → tool "create-note"
```

The handler file is minimal:

```ts
// server/mcp/index.ts
export default defineMcpHandler({})
```

## Your First Tool — Read

A query tool that lists all notes:

```ts
// server/mcp/tools/list-notes.ts
import { defineConvexTool } from 'better-convex-nuxt/mcp'
import { defineConvexSchema } from 'better-convex-nuxt/schema'
import { serverConvexQuery } from 'better-convex-nuxt/server'
import { api } from '~~/convex/_generated/api'

const schema = defineConvexSchema({}, { description: 'List all notes' })

export default defineConvexTool({
  schema,
  operation: 'query',
  handler: async () => {
    return await serverConvexQuery(api.notes.list, {})
  },
})
```

That's it. The framework handles:

- **Input validation** — Convex validators are converted to Zod/JSON Schema automatically
- **Result envelope** — your return value is wrapped in `{ ok: true, data: ... }`
- **Error categorization** — exceptions become `{ ok: false, error: { category, message, retryable } }`
- **MCP annotations** — `operation: 'query'` auto-derives `readOnlyHint: true`, `destructiveHint: false`

## Your First Tool — Write

A mutation tool that creates a note with input validation:

```ts
// server/mcp/tools/create-note.ts
import { defineConvexTool, withSummary } from 'better-convex-nuxt/mcp'
import { defineConvexSchema } from 'better-convex-nuxt/schema'
import { serverConvexMutation } from 'better-convex-nuxt/server'
import { api } from '~~/convex/_generated/api'

const schema = defineConvexSchema(
  {
    title: v.string(),
    content: v.string(),
  },
  {
    description: 'Create a new note',
    fields: {
      title: { description: 'The note title', examples: ['Meeting Notes'] },
      content: { description: 'The note body text' },
    },
  },
)

export default defineConvexTool({
  schema,
  name: 'create-note',
  handler: async (args) => {
    const noteId = await serverConvexMutation(api.notes.add, args)
    return withSummary({ id: noteId, title: args.title }, `Created note "${args.title}"`)
  },
})
```

Key differences from the read tool:

- **Field metadata** — `description` and `examples` help AI agents understand what to pass
- **`withSummary`** — gives agents a human-readable response alongside the structured data
- **Default operation** — `'mutation'` is the default, so no need to specify it

## What the Framework Does

Every `defineConvexTool` call produces a fully-featured MCP tool with:

| Feature | Automatic | Manual |
|---------|-----------|--------|
| Input schema (JSON Schema) | From Convex validators | — |
| Field descriptions | From schema metadata | — |
| Result envelope (`ok`/`error`) | Always | — |
| MCP annotations | From `operation` | Override via `annotations` |
| Error categorization | From exception type | `wrapError(category, msg)` |
| Input examples | From field `examples` | Override via `inputExamples` |
| Auth, permissions, safety | — | Opt-in via options |

## Testing with curl

Start your dev server and send JSON-RPC requests to `/mcp`:

```bash
# Initialize session
curl -s http://localhost:3000/mcp \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"test","version":"1.0.0"}}}' | jq .

# List available tools
curl -s http://localhost:3000/mcp \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -H "Mcp-Session-Id: YOUR_SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' | jq .

# Call a tool
curl -s http://localhost:3000/mcp \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -H "Mcp-Session-Id: YOUR_SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"list-notes","arguments":{}}}' | jq .
```

Replace `YOUR_SESSION_ID` with the session ID from the `initialize` response.

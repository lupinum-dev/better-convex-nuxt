description: 'Protect destructive MCP tools with confirmation, previews, and request caps.'

## Destructive Flow

Set `destructive: true` to require a second confirmed call before the handler runs.

```ts
export default defineTool({
  schema: deleteNote,
  name: 'delete-note',
  destructive: true,
  handler: async (args, ctx) => {
    await ctx.mutation(api.notes.remove, { id: args.id })
    return ctx.ok({ deleted: true, id: args.id })
  },
})
```

## Preview

Add `preview` to show the agent what will happen before confirmation:

```ts
export default defineTool({
  schema: deleteNote,
  name: 'delete-note',
  destructive: true,
  preview: async (args, ctx) => {
    const note = await ctx.query(api.notes.get, { id: args.id })
    if (!note) return ctx.blocked('Note not found')

    return ctx.preview(`Will permanently delete "${note.title}"`, { affects: { notes: 1 } })
  },
  handler: async (args, ctx) => {
    await ctx.mutation(api.notes.remove, { id: args.id })
    return ctx.ok({ deleted: true, id: args.id })
  },
})
```

## Bulk Caps

Use `maxItems` when an array argument should stay bounded:

```ts
export default defineTool({
  schema: bulkDeleteNotes,
  destructive: true,
  maxItems: { field: 'ids', limit: 10 },
  handler: async (args, ctx) => {
    for (const id of args.ids) {
      await ctx.mutation(api.notes.remove, { id })
    }
    return ctx.ok({ deleted: args.ids.length })
  },
})
```

## Rate Limiting

```ts
export default defineTool({
  schema: bulkDeleteNotes,
  rateLimit: { max: 5, window: '1m' },
  handler: async (_args, ctx) => {
    return ctx.ok({ ok: true })
  },
})
```

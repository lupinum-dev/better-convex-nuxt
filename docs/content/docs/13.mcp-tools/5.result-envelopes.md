---
title: 'Result Envelopes'
description: 'Structured MCP responses that are easy for agents to consume.'
---

Every `defineTool()` result is wrapped in a consistent envelope.

## Success

```json
{
  "ok": true,
  "data": { "id": "abc123", "title": "Meeting Notes" }
}
```

## Error

```json
{
  "ok": false,
  "error": {
    "category": "not_found",
    "message": "Note \"abc123\" not found.",
    "retryable": false
  }
}
```

## Summary Helpers

Use `ctx.ok(...)` for the common case:

```ts
handler: async (args, ctx) => {
  const id = await ctx.mutation(api.notes.add, args)
  return ctx.ok({ id }, `Created note "${args.title}"`)
}
```

Use `withSummary(...)` when you want to return structured data directly:

```ts
return withSummary({ results, total: results.length }, `Found ${results.length} notes`)
```

## Custom Errors

Use `wrapError(...)` in middleware or edge cases:

```ts
middleware: async (args, ctx, next) => {
  const note = await ctx.query(api.notes.get, { id: args.id })
  if (!note) {
    return wrapError('not_found', `Note "${args.id}" not found.`)
  }
  return next()
}
```

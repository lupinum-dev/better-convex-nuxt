## Auth Modes

Every tool can be:

- `auth: 'required'`
- `auth: 'optional'`
- omitted for public access

```ts
export default defineTool({
  schema,
  auth: 'required',
  handler: async (args, ctx) => {
    return ctx.ok({ actor: ctx.actor, args })
  },
})
```

## `McpAuthIdentity`

```ts
interface McpAuthIdentity {
  readonly role: string
  readonly userId: string
  readonly tenantId?: string
}
```

The default resolver reads `event.context.mcpAuth`.

## Tool-Level Actor Checks

Use `check` for actor-only checks that should happen before the handler runs:

```ts
export default defineTool({
  schema,
  auth: 'required',
  check: (actor) => ['owner', 'admin', 'member'].includes(actor.role),
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.todos.create, args)
    return ctx.ok({ id })
  },
})
```

`check` now affects both execution and default visibility. If an actor cannot run the tool, `tools/list` should not advertise it either.

## Resource-Aware Checks

When the decision depends on a specific resource, load it inside middleware or the handler and use your own app checks:

```ts
export default defineTool({
  schema,
  auth: 'required',
  handler: async (args, ctx) => {
    const todo = await ctx.query(api.todos.get, { id: args.id })
    if (!todo) return ctx.error('not_found', 'Todo not found')
    if (!(ctx.actor?.role === 'admin' || todo.ownerId === ctx.actor?.userId)) {
      return ctx.error('forbidden', 'You cannot update this todo')
    }

    await ctx.mutation(api.todos.setCompleted, args)
    return ctx.ok({ updated: true })
  },
})
```

Keep resource-aware visibility explicit. The framework will not infer discovery rules from database lookups. If a record-driven condition must hide a tool, add an explicit `enabled` guard or leave the tool visible and enforce in middleware/handler.

## Scoped MCP Calls

Set `scoped: true` when the target Convex functions use `appQuery` / `appMutation` from `convex/functions.ts`. The tool layer injects the hidden trusted-caller payload automatically, and the Convex handler still keeps auth explicit with `const actor = await ctx.actor()`.

## Auth Modes

Every tool can be:

- `auth: 'required'`
- `auth: 'optional'`
- omitted for public access

```ts
export default defineTool({
  schema,
  auth: 'required',
  handler: async (args, ctx) => {
    return ctx.ok({ actor: ctx.actor, args })
  },
})
```

## `McpAuthIdentity`

```ts
interface McpAuthIdentity {
  readonly role: string
  readonly userId: string
  readonly tenantId?: string
}
```

The default resolver reads `event.context.mcpAuth`.

## Custom Auth Resolver

```ts
export const { defineTool } = createConvexTools({
  checkPermission,
  resolveAuth: async (event) => {
    const session = await getSession(event)
    if (!session) return null
    return {
      role: session.role,
      userId: session.userId,
      tenantId: session.tenantId,
    }
  },
})
```

## Declarative Permissions

Use `require` for permission checks that should happen before the handler runs:

```ts
export default defineTool({
  schema,
  auth: 'required',
  require: 'todo.create',
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.todos.create, args)
    return ctx.ok({ id })
  },
})
```

When the tool factory is created from your app permission config, `require` is type-checked.

## Ownership-Aware Checks

Use `ctx.can(...)` when the decision depends on a specific resource:

```ts
export default defineTool({
  schema,
  auth: 'required',
  handler: async (args, ctx) => {
    const todo = await ctx.query(api.todos.get, { id: args.id })
    if (!todo) return ctx.error('not_found', 'Todo not found')
    if (!ctx.can('todo.update', todo)) {
      return ctx.error('forbidden', 'You cannot update this todo')
    }

    await ctx.mutation(api.todos.setCompleted, args)
    return ctx.ok({ updated: true })
  },
})
```

## Scoped MCP Calls

Set `scoped: true` when the target Convex functions expect tenant-scoped builders. The tool layer will inject service auth automatically so tool authors do not manage `_serviceKey` or `_serviceActor` manually.

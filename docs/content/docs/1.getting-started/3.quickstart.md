The goal is narrow on purpose: sign in, load your own todo list, create a todo that only your account can see, and stop there. Do not add tenant isolation or MCP yet.

## What You Will Build

By the end of this page, the app will have:

- Better Auth routes served through Convex
- one app-owned actor resolver
- one protected query that lists the signed-in user's todos
- one protected mutation that creates a todo for that user
- one Nuxt page that signs in and renders the personal list

This tutorial follows the same shape as [`examples/02-auth-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/02-auth-todo).

## Before You Start

You should already have the installation baseline from [Installation](/docs/getting-started/installation).

For this auth-enabled path, keep these values configured:

- `CONVEX_URL`
- `SITE_URL`
- `BETTER_AUTH_SECRET`

If the baseline is not stable yet, stop and run:

```bash
npx trellis doctor
```

## Step 1: Enable The Module Auth Path

Start with the auth-ready module config:

```ts
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

This keeps Trellis on the signed-in path from the start.

## Step 2: Register Better Auth In Convex

Create the auth bridge in `convex/auth.ts`:

```ts
import { defineAuth } from '@lupinum/trellis/auth'

import { components, internal } from './_generated/api'
import { mutation } from './_generated/server'
import authConfig from './auth.config'

export const { authComponent, createAuth, createUserIfNeeded } = defineAuth(
  { components, internal, mutation, authConfig },
  {
    emailPassword: true,
  },
)
```

Then register the HTTP routes in `convex/http.ts`:

```ts
import { httpRouter } from 'convex/server'

import { authComponent, createAuth } from './auth'

const http = httpRouter()

authComponent.registerRoutes(http, createAuth)

export default http
```

If you want the default file set instead of writing those by hand, `npx trellis init auth` scaffolds the same core auth files.

## Step 3: Resolve An App-Owned Actor

This first protected app is still personal, not tenant-scoped. The actor can stay minimal.

`convex/auth/actor.ts`:

```ts
import { getAuth } from '@lupinum/trellis/auth'
import type { GenericActionCtx, GenericMutationCtx, GenericQueryCtx } from 'convex/server'

import type { DataModel } from '../_generated/dataModel'

export type Actor = { kind: 'user'; userId: string } | null

type AuthTodoCtx =
  | GenericQueryCtx<DataModel>
  | GenericMutationCtx<DataModel>
  | GenericActionCtx<DataModel>

export async function getActor(ctx: AuthTodoCtx): Promise<Actor> {
  const auth = await getAuth(ctx)
  if (!auth) return null
  if (!('db' in ctx)) {
    throw new Error('AuthTodo actor resolution requires a query or mutation context.')
  }

  const user = await ctx.db
    .query('users')
    .withIndex('by_auth_id', (q: any) => q.eq('authId', auth.subject))
    .first()

  if (!user) return null

  return {
    kind: 'user',
    userId: user.authId,
  }
}
```

Then wire that actor into Trellis in `convex/functions.ts`:

```ts
import { defineTrellis } from '@lupinum/trellis/functions'

import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'
import { getActor } from './auth/actor'

export const { mutation, query, raw } = defineTrellis(
  { query: generatedQuery, mutation: generatedMutation },
  {
    actor: getActor,
  },
)
```

This is the first important boundary: Trellis owns the protected function pipeline, but your app still decides what a valid actor is.

## Step 4: Add One Guard, One Table, And Two Handlers

Define the smallest useful guard in `convex/auth/checks.ts`:

```ts
import { defineGuard } from '@lupinum/trellis/auth'

import type { Actor } from './actor'

export const isAuthenticated = defineGuard<Actor>('Authenticated', (actor) => actor !== null)
```

Create a small schema in `convex/schema.ts`:

```ts
import { defineSchema, defineTable } from 'convex/server'
import { v } from 'convex/values'

export default defineSchema({
  users: defineTable({
    authId: v.string(),
    email: v.optional(v.string()),
    displayName: v.optional(v.string()),
    createdAt: v.number(),
    updatedAt: v.number(),
  }).index('by_auth_id', ['authId']),

  todos: defineTable({
    ownerId: v.string(),
    title: v.string(),
    completed: v.boolean(),
    createdAt: v.number(),
  }).index('by_owner', ['ownerId']),
})
```

Then create the protected query and mutation in `convex/todos.ts`:

```ts
import { createTodo } from '../shared/schemas/todo'
import { isAuthenticated } from './auth/checks'
import { mutation, query } from './functions'

export const list = query({
  args: {},
  guard: isAuthenticated,
  handler: async (ctx) => {
    const actor = await ctx.actor()

    return await ctx.db
      .query('todos')
      .withIndex('by_owner', (q) => q.eq('ownerId', actor.userId))
      .order('desc')
      .collect()
  },
})

export const create = mutation({
  args: createTodo.args,
  guard: isAuthenticated,
  handler: async (ctx, args) => {
    const actor = await ctx.actor()

    return await ctx.db.insert('todos', {
      ownerId: actor.userId,
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})
```

That is enough for the first protected app. One guard protects both handlers, and ownership is explicit in the stored row.

## Step 5: Build The Nuxt Page

Use `useConvexAuth()` for session state, `useConvexAuthActions()` for sign-in and sign-up transitions, `useConvexQuery()` for the protected list, and `useConvexMutation()` for creation.

This is the smallest shape that matches the example flow:

```vue
<script setup lang="ts">
import * as z from 'zod'

import { api } from '#trellis/api'

const { client, isAuthenticated, isPending, user, signOut } = useConvexAuth()
const authAction = useConvexAuthActions()

const title = ref('')
const createTodo = useConvexMutation(api.todos.create)

const actorReady = computed(() => isAuthenticated.value)
const todoArgs = computed(() => (isAuthenticated.value && actorReady.value ? {} : undefined))

const { data: todos, pending: todosPending } = await useConvexQuery(api.todos.list, todoArgs)

const signInSchema = z.object({
  email: z.string().email(),
  password: z.string().min(1),
})

async function handleSignIn(payload: { email: string; password: string }) {
  if (!client) throw new Error('Auth client unavailable.')
  await authAction.execute(() => client.signIn.email(payload), { redirectTo: '/' })
}

async function handleCreateTodo() {
  await createTodo({ title: title.value })
  title.value = ''
}
</script>
```

The full page in the example also includes sign-up, sign-out, loading states, and delete and toggle actions. Keep your first pass smaller if the app does not need those immediately.

## What Success Looks Like

You are done with this tutorial when:

1. you can create an account
2. sign in without page-specific auth glue code
3. see an empty personal list
4. create a todo
5. sign in as a different account and confirm the first account's todo does not appear

That last check matters. The whole point of this tutorial is that protection is backend-owned, not just hidden in the page.

## Common First Failures

- Auth is enabled, but `SITE_URL` or `BETTER_AUTH_SECRET` is missing.
- Better Auth routes were defined in `convex/auth.ts`, but not registered in `convex/http.ts`.
- The actor resolver returns `null` because the `users` row was never created or cannot be found by `authId`.
- The page subscribes to the protected query before auth is ready.
- The todo handler writes rows, but does not store `ownerId`, so the list query cannot scope correctly.

Build the smallest Trellis app that feels real: load data on the page, add a record, and watch the list update without a manual refetch.

## What You Will Build

By the end of this page, the app will have:

- one shared args definition
- one public query that lists todos
- one public mutation that creates a todo
- one Nuxt page that renders the list and adds new items
- one visible live update when the mutation settles

This tutorial follows the same shape as [`examples/01-public-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/01-public-todo).

## Before You Start

You should already have the installation baseline from [Installation](/docs/getting-started/installation).

For this smallest path, the only required runtime value is:

- `CONVEX_URL`

If the baseline is not healthy yet, stop and run:

```bash
npx trellis doctor
```

## Step 1: Reuse One Shared Args Definition

Create `shared/schemas/todo.ts`:

```ts
import { defineArgs } from '@lupinum/trellis/args'
import { v } from 'convex/values'

export const createTodo = defineArgs({
  description: 'Create a public todo item',
  args: {
    title: v.string(),
  },
})
```

This keeps the input contract runtime-neutral so both Convex handlers and other Trellis-aware surfaces can reuse it safely later.

## Step 2: Add The Smallest Trellis Runtime And Handlers

Create `convex/todos.ts`:

```ts
import { open, requireRecord } from '@lupinum/trellis/auth'
import { defineTrellis } from '@lupinum/trellis/functions'
import { v } from 'convex/values'

import { createTodo } from '../shared/schemas/todo'
import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'

const { mutation, query } = defineTrellis({
  query: generatedQuery,
  mutation: generatedMutation,
})

export const list = query({
  args: {},
  guard: open,
  handler: async (ctx) => {
    return await ctx.db.query('todos').order('desc').collect()
  },
})

export const create = mutation({
  args: createTodo.args,
  guard: open,
  handler: async (ctx, args) => {
    return await ctx.db.insert('todos', {
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})

export const toggle = mutation({
  args: { id: v.id('todos') },
  guard: open,
  load: async (ctx, args) => {
    const todo = await ctx.db.get(args.id)
    requireRecord(todo, 'Todo')
    return { todo }
  },
  handler: async (ctx, args, { todo }) => {
    await ctx.db.patch(args.id, {
      completed: !todo.completed,
    })
  },
})
```

This is the whole point of the first live query path: Trellis should still feel small when the app is small.

## Step 3: Render The Query In The Page

Create `pages/index.vue`:

```vue
<script setup lang="ts">
import { computed, ref } from 'vue'

import { api } from '#trellis/api'

const { data: todos, pending, error } = await useConvexQuery(api.todos.list, {})
const createTodo = useConvexMutation(api.todos.create)

const title = ref('')
const todoItems = computed(() => todos.value ?? [])

async function handleCreate() {
  await createTodo({ title: title.value })
  title.value = ''
}
</script>

<template>
  <form @submit.prevent="handleCreate">
    <input v-model="title" placeholder="Write something small and concrete" required />
    <button type="submit">Add</button>
  </form>

  <p v-if="error">{{ error.message }}</p>
  <p v-else-if="pending">Loading...</p>
  <ul v-else>
    <li v-for="todo in todoItems" :key="todo._id">
      {{ todo.title }}
    </li>
  </ul>
</template>
```

The important thing is what you do **not** have to write:

- no manual fetch endpoint
- no explicit refetch after creation
- no transport-specific glue between Nuxt and Convex

## Step 4: Verify The Live Update

Run the app and add a todo. The success condition is visible:

- the page renders the list
- submitting the form creates a todo
- the list updates without a manual refresh

That is the first real Trellis win.

## Common mistakes

- Adding auth or actor resolution before the public baseline is working
- Treating `useConvexQuery()` like a one-shot fetch and manually refetching after writes
- Hiding the shared args shape inside one handler file instead of keeping it reusable

Install Trellis into a Nuxt app, choose the right baseline, and verify the wiring before you write app code.

This page covers package install, module registration, environment setup, and the first verification pass. It does not cover app-owned auth code, permission context, or MCP tools.

## Prerequisites

Before you install Trellis, you need:

- Nuxt 4
- Node 18 or newer
- a Convex project or local Convex dev environment

If you want a known-good baseline instead of a blank app, start from one of the runnable examples in [`examples/`](https://github.com/lupinum-dev/trellis/tree/main/examples).

## Install The Package

Install Trellis and Convex in your Nuxt app:

```bash
pnpm add convex @lupinum/trellis
```

Then register the module in `nuxt.config.ts`.

## Choose The Correct Baseline

Trellis auth defaults to enabled. That is the right default for most real apps, but it is the wrong default for a public-only baseline.

### Public-Only Baseline

Use this when you want to start with queries and mutations only.

```ts
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: false,
  },
})
```

### Auth-Ready Baseline

Use this when the app needs sign-in flows from the start.

```ts
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

## Environment Variables

### Always Needed

Trellis needs a Convex deployment URL through `convex.url` or one of these environment variables:

- `CONVEX_URL`
- `NUXT_PUBLIC_CONVEX_URL`

For most apps, `trellis.url: process.env.CONVEX_URL` is the clearest baseline.

### Needed When Auth Is Enabled

If auth is enabled, keep these values configured:

- `SITE_URL` for Better Auth callbacks and trusted-origin checks
- `BETTER_AUTH_SECRET` in both the local environment and the Convex Better Auth deployment

### Sometimes Needed When Auth Is Enabled

Trellis can derive the Convex HTTP Actions host from `CONVEX_URL` for standard `.convex.cloud` and local `127.0.0.1:3210` setups.

Set one of these explicitly only when that derivation is not reliable for your setup:

- `CONVEX_SITE_URL`
- `NUXT_PUBLIC_CONVEX_SITE_URL`

That usually matters when you use a custom HTTP Actions domain or when token exchange cannot be derived cleanly from the deployment URL.

## Verify The Setup

Run the built-in doctor command from the app root:

```bash
npx trellis doctor
```

Use the result mechanically:

- If the app is public-only, make sure `auth: false` is set so auth warnings are not expected.
- If the app uses auth, make sure `SITE_URL`, `BETTER_AUTH_SECRET`, and Better Auth route registration are all in place.
- If the doctor warns about missing Convex URL configuration, fix that before writing app code.

## First Visible Success

After installation, the first useful proof is a page that renders one query:

```vue
<script setup lang="ts">
import { api } from '#trellis/api'

const { data: tasks, status } = await useConvexQuery(api.tasks.list, {
  status: 'active',
})
</script>

<template>
  <div v-if="status === 'pending'">Loading...</div>
  <ul v-else-if="status === 'success'">
    <li v-for="task in tasks" :key="task._id">
      {{ task.text }}
    </li>
  </ul>
</template>
```

If you want a working reference instead of building from scratch:

- public-only: [`01-public-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/01-public-todo)
- auth-ready: [`02-auth-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/02-auth-todo)
- workspace baseline: [`03-team-workspace`](https://github.com/lupinum-dev/trellis/tree/main/examples/03-team-workspace)

## Common Early Mistakes

- Leaving auth enabled while trying to build a public-only baseline.
- Setting `CONVEX_URL` manually inside an example that already injects local Convex URLs for you.
- Treating `trellis doctor` as optional after an auth or env failure.
- Starting with the workspace example when the app only needs a public or personal baseline.

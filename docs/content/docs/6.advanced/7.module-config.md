All options are set under the `convex` key in `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    // Options go here
  },
})
```

## Options Reference

### Core

| Option    | Type     | Default      | Description                                                                                                                                                            |
| --------- | -------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `url`     | `string` | `undefined`  | Convex deployment URL (e.g., `https://your-app.convex.cloud`). Usually set via `CONVEX_URL` env var.                                                                   |
| `siteUrl` | `string` | Auto-derived | Convex HTTP Actions URL (e.g., `https://your-app.convex.site`). Auto-derived from `url` by replacing `.convex.cloud` with `.convex.site`. Override for custom domains. |

::callout{icon="i-lucide-info"}
**`CONVEX_URL` / `CONVEX_SITE_URL` are read at build time.** The module resolves them once during `nuxt build`/`nuxt prepare` and bakes the value into `runtimeConfig.public.convex`. To override the URL at deploy time (without rebuilding), set `NUXT_PUBLIC_CONVEX_URL` (and `NUXT_PUBLIC_CONVEX_SITE_URL`) - Nuxt's native runtime-config env override applies these on both server and client. A `CONVEX_URL` that is only present at deploy time (not at build time) will not be picked up and produces a working SSR / broken client mismatch.
::

### Authentication

| Option           | Type                    | Default             | Description                                                                                                                                       |
| ---------------- | ----------------------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth`           | `ConvexAuthConfigInput` | `{ enabled: true }` | Authentication config. Set `auth.enabled = false` to disable auth features. See [Authentication](/docs/auth-security/authentication).             |
| `authRoute`      | `string`                | `'/api/auth'`       | Route path for the auth proxy.                                                                                                                    |
| `trustedOrigins` | `string[]`              | `[]`                | Additional trusted origins for CORS on the auth proxy. Supports wildcards (e.g., `'https://preview-*.vercel.app'`).                               |
| `skipAuthRoutes` | `string[]`              | `[]`                | Routes that skip auth checks entirely. Supports globs (`/docs/**`). See [Skipping Auth](/docs/auth-security/authentication#skipping-auth-checks). |

::callout{icon="i-lucide-triangle-alert"}
**`trustedOrigins` (and `skipAuthRoutes`) concatenate, they don't override, if set in two places.** The module merges `convex.trustedOrigins` into `runtimeConfig.public.convex.trustedOrigins` with [`defu`](https://github.com/unjs/defu), which concatenates array values from both sources rather than letting one replace the other. If you set `trustedOrigins: ['https://a.example']` under `convex: {}` in `nuxt.config.ts` **and** also set `runtimeConfig.public.convex.trustedOrigins: ['https://b.example']` (e.g. via a separate config layer or an env-driven override), the resolved list is `['https://b.example', 'https://a.example']` — both, not whichever you meant to be authoritative. Set `trustedOrigins` in exactly one place to avoid surprises; there is currently no way to have a later source fully replace an earlier array here.
::

### Auth Proxy Limits

| Option      | Type                | Default | Description                                                        |
| ----------- | ------------------- | ------- | ------------------------------------------------------------------ |
| `authProxy` | `AuthProxyDefaults` | `{}`    | Request/response body-size limits for the Better Auth proxy route. |

```ts [nuxt.config.ts]
convex: {
  authProxy: {
    maxRequestBodyBytes: 1_048_576,  // 1 MiB
    maxResponseBodyBytes: 1_048_576, // 1 MiB
  },
}
```

### Query Defaults

| Option     | Type            | Default | Description                                                                                           |
| ---------- | --------------- | ------- | ----------------------------------------------------------------------------------------------------- |
| `defaults` | `QueryDefaults` | `{}`    | Default options for `useConvexQuery` and `useConvexPaginatedQuery`. Per-query options override these. |

```ts [nuxt.config.ts]
convex: {
  defaults: {
    server: true,    // Run queries on server during SSR (default)
    subscribe: true, // Enable real-time subscriptions (default)
    auth: 'auto',    // Attach auth token when available (default)
  },
}
```

### Upload

| Option   | Type             | Default | Description                             |
| -------- | ---------------- | ------- | --------------------------------------- |
| `upload` | `UploadDefaults` | `{}`    | Default options for upload composables. |

```ts [nuxt.config.ts]
convex: {
  upload: {
    maxConcurrent: 3,  // Max concurrent uploads for useConvexUploadQueue
  },
}
```

### Caching

| Option      | Type               | Default                       | Description                                                                                   |
| ----------- | ------------------ | ----------------------------- | --------------------------------------------------------------------------------------------- |
| `authCache` | `AuthCacheOptions` | `{ enabled: false, ttl: 60 }` | SSR auth token caching. See [Performance](/docs/advanced/performance#ssr-auth-token-caching). |

```ts [nuxt.config.ts]
convex: {
  authCache: {
    enabled: true,
    ttl: 60, // 60 seconds (max)
  },
}
```

::callout{icon="i-lucide-info"}
Signing out or revoking sessions through the built-in auth proxy clears the cached JWT for that session after the upstream revocation succeeds. A token request already in flight when revocation lands can still re-cache a JWT for up to `ttl` seconds (clamped ≤60s). A session revoked through another channel (e.g. an admin action or a direct Better Auth API call, bypassing the proxy) can also yield a cached token until `ttl` expires - keep `ttl` low if that window matters for your threat model.
::

### Permissions

| Option        | Type      | Default | Description                                                                                                      |
| ------------- | --------- | ------- | ---------------------------------------------------------------------------------------------------------------- |
| `permissions` | `boolean` | `false` | Enable permission composables (`createPermissions` factory). See [Permissions](/docs/auth-security/permissions). |

### Logging & Debug

| Option    | Type                         | Default | Description                                              |
| --------- | ---------------------------- | ------- | -------------------------------------------------------- |
| `logging` | `false \| 'info' \| 'debug'` | `false` | Module log level. See [Logging](/docs/advanced/logging). |
| `debug`   | `ConvexDebugOptions`         | `{}`    | Enable high-verbosity trace logs for specific channels.  |

```ts [nuxt.config.ts]
convex: {
  logging: 'info',
  debug: {
    authFlow: false,        // Detailed auth flow logs (both client and server)
    clientAuthFlow: false,  // Client-only auth flow trace
    serverAuthFlow: false,  // Server-only auth flow trace
  },
}
```

## Full Example

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    url: process.env.CONVEX_URL,
    // siteUrl: 'https://custom-domain.example.com', // Only if using custom domain

    auth: { enabled: true },
    skipAuthRoutes: ['/', '/pricing', '/docs/**'],

    defaults: {
      server: true,
      subscribe: true,
    },

    upload: {
      maxConcurrent: 3,
    },

    authCache: {
      enabled: true,
      ttl: 60,
    },

    permissions: true,

    logging: process.env.NODE_ENV === 'development' ? 'debug' : false,
  },
})
```

All options are set under the `convex` key in `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    // Options go here
  },
})
```

## Options Reference

### Core

| Option    | Type     | Default      | Description                                                                                                                                                            |
| --------- | -------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `url`     | `string` | `undefined`  | Convex deployment URL (e.g., `https://your-app.convex.cloud`). Usually set via `CONVEX_URL` env var.                                                                   |
| `siteUrl` | `string` | Auto-derived | Convex HTTP Actions URL (e.g., `https://your-app.convex.site`). Auto-derived from `url` by replacing `.convex.cloud` with `.convex.site`. Override for custom domains. |

::callout{icon="i-lucide-info"}
**`CONVEX_URL` / `CONVEX_SITE_URL` are read at build time.** The module resolves them once during `nuxt build`/`nuxt prepare` and bakes the value into `runtimeConfig.public.convex`. To override the URL at deploy time (without rebuilding), set `NUXT_PUBLIC_CONVEX_URL` (and `NUXT_PUBLIC_CONVEX_SITE_URL`) - Nuxt's native runtime-config env override applies these on both server and client. A `CONVEX_URL` that is only present at deploy time (not at build time) will not override runtime config; use the `NUXT_PUBLIC_*` variables for deploy-time overrides.
::

### Authentication

| Option | Type                         | Default       | Description                                                                                                                        |
| ------ | ---------------------------- | ------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `auth` | `false \| ConvexAuthOptions` | installs auth | Omitted or an object installs authentication with defaults. Set `auth: false` for a Convex-only build with no Better Auth runtime. |

Every auth-only build option lives inside `auth`; there is no top-level `authRoute`, `trustedOrigins`, route-skip list, `authCache`, or `authProxy` anymore.

```ts [nuxt.config.ts]
convex: {
  auth: {
    route: '/api/auth',             // Auth proxy route path (default)
    trustedOrigins: [],             // Additional trusted origins for CORS
    cache: { ttl: 60 },             // Omit/false to disable SSR auth-token caching
    proxy: {
      maxRequestBodyBytes: 1_048_576,
      maxResponseBodyBytes: 1_048_576,
    },
    routeProtection: {
      redirectTo: '/auth/signin',
      preserveReturnTo: true,
    },
  },
}
```

::callout{icon="i-lucide-triangle-alert"}
**`trustedOrigins` concatenates, it doesn't override, if set in two places.** The module merges `convex.auth.trustedOrigins` into runtime config with [`defu`](https://github.com/unjs/defu), which concatenates array values from both sources rather than letting one replace the other. Set `trustedOrigins` in exactly one place to avoid surprises.
::

For a Convex-only build with no Better Auth runtime at all:

```ts [nuxt.config.ts]
convex: {
  auth: false,
}
```

See [Authentication](/docs/auth-security/authentication).

### Query Defaults

| Option     | Type            | Default | Description                                                                                           |
| ---------- | --------------- | ------- | ----------------------------------------------------------------------------------------------------- |
| `defaults` | `QueryDefaults` | `{}`    | Default options for `useConvexQuery` and `useConvexPaginatedQuery`. Per-query options override these. |

```ts [nuxt.config.ts]
convex: {
  defaults: {
    server: true,    // Run queries on server during SSR (default)
    subscribe: true, // Enable real-time subscriptions (default)
    waitTimeoutMs: 10_000, // First subscribe result timeout; 0 waits indefinitely
  },
}
```

Per-query auth mode (`required` | `optional` | `none`) is set per call via the `auth` option — it is `'optional'` unless overridden, and is not configurable at the module level.

`waitTimeoutMs` applies only to awaited subscribe-mode queries while waiting for
the first WebSocket result. Set it to `0` to wait indefinitely. Negative,
non-numeric, or non-finite values fall back to `10_000`.

### Upload

| Option   | Type             | Default | Description                             |
| -------- | ---------------- | ------- | --------------------------------------- |
| `upload` | `UploadDefaults` | `{}`    | Default options for upload composables. |

```ts [nuxt.config.ts]
convex: {
  upload: {
    maxConcurrent: 3,  // Max concurrent uploads for useConvexUploadQueue
  },
}
```

### Caching

SSR auth-token caching lives under `auth.cache` (see [Authentication](#authentication) above), not as a top-level option.

```ts [nuxt.config.ts]
convex: {
  auth: {
    cache: {
      ttl: 60, // 60 seconds (max)
    },
  },
}
```

Omitting `cache` (or setting `cache: false`) disables the SSR auth-token cache. An object value enables it with a clamped `ttl`.

::callout{icon="i-lucide-info"}
Signing out or revoking sessions through the built-in auth proxy clears the cached JWT for that session after the upstream revocation succeeds. A token request already in flight when revocation lands can still re-cache a JWT for up to `ttl` seconds (clamped ≤60s). A session revoked through another channel (e.g. an admin action or a direct Better Auth API call, bypassing the proxy) can also yield a cached token until `ttl` expires - keep `ttl` low if that window matters for your threat model.
::

### Logging & Debug

| Option       | Type                         | Default | Description                                                          |
| ------------ | ---------------------------- | ------- | -------------------------------------------------------------------- |
| `logging`    | `false \| 'info' \| 'debug'` | `false` | Module log level. See [Logging](/docs/advanced/logging).             |
| `auth.debug` | `ConvexDebugOptions`         | `{}`    | Enable high-verbosity auth trace logs (requires `logging: 'debug'`). |

```ts [nuxt.config.ts]
convex: {
  logging: 'info',
  auth: {
    debug: {
      authFlow: false,        // Detailed auth flow logs (both client and server)
      clientAuthFlow: false,  // Client-only auth flow trace
      serverAuthFlow: false,  // Server-only auth flow trace
    },
  },
}
```

## Full Example

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    url: process.env.CONVEX_URL,
    // siteUrl: 'https://custom-domain.example.com', // Only if using custom domain

    auth: {
      cache: { ttl: 60 },
      routeProtection: {
        redirectTo: '/auth/signin',
        preserveReturnTo: true,
      },
    },

    defaults: {
      server: true,
      subscribe: true,
    },

    upload: {
      maxConcurrent: 3,
    },

    logging: process.env.NODE_ENV === 'development' ? 'debug' : false,
  },
})
```

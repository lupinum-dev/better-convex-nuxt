Use `useConvexStorageUrl()` when you already have a Convex `storageId` and need a browser-usable URL.

The composable is:

```ts
const fileUrl = useConvexStorageUrl(api.files.getUrl, storageId)
```

## Why This Exists

`useConvexStorageUrl(...)` wraps `useConvexQuery(...)` with one useful default:

- if `storageId` is `null` or `undefined`, the query is skipped automatically

That means you do not need a separate guard around the query just to avoid an empty request.

## Reactive Input

The `storageId` can be:

- a plain value
- a `ref`
- a `computed`

The returned URL is also reactive, so it updates when the underlying `storageId` changes.

## Common Pairing With Uploads

The normal pattern is:

```ts
const { upload, data: storageId } = useConvexUpload(api.files.generateUploadUrl)
const fileUrl = useConvexStorageUrl(api.files.getUrl, storageId)
```

That keeps upload and display on one reactive path.

## Common mistakes

- Querying for a URL manually when `useConvexStorageUrl(...)` already handles skip behavior
- Forgetting that the returned value can be `null` while loading, skipped, or missing

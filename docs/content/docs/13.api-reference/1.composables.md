This page covers the main runtime composables you are most likely to look up while building a Nuxt app with Trellis.

For task flow, use the guides. For the full generated inventory, use [API Surface](/docs/api-reference/api-surface).

## `useConvexQuery`

```ts
useConvexQuery(query, args?, options?)
```

Default query composable for Nuxt apps.

Use it when the component should:

- fetch during SSR by default
- hydrate into the client
- continue as a live Convex subscription by default

Important options:

- `server?: boolean`
  Controls whether the first result should participate in SSR. Defaults to `true`, falling back to module `query.server`.
- `subscribe?: boolean`
  Controls whether the query upgrades into a live subscription after the first result. Defaults to `true`, falling back to module `query.subscribe`.
- `default?: () => RawT | undefined`
  Fallback value used before the first real result arrives.
- `transform?: (input) => output`
  Maps the raw query result into UI-facing data.
- `keepPreviousData?: boolean`
  Keeps the last settled result visible during arg transitions.

Return shape:

- `data`
- `error`
- `refresh()`
- `clear()`
- `pending`
- `status`
- `isStale`

Status notes:

- nullish args skip execution and produce `status === 'skipped'`
- `keepPreviousData` keeps the previous result visible while `pending` stays `true`
- `subscribe: false` changes the call into a one-shot HTTP-backed read

Failure behavior:

- execution failures surface through `error`
- query state stays bound to the current args, not whichever older request settles last

Example:

```ts
const { data, status, pending, isStale } = await useConvexQuery(api.tasks.list, {
  status: 'active',
}, {
  keepPreviousData: true,
})
```

## `useConvexPaginatedQuery`

```ts
useConvexPaginatedQuery(query, args?, options?)
```

Paginated variant for cursor-based list queries.

Important options:

- `initialNumItems: number`
  Initial page size. Defaults to `10` when omitted.
- `server?: boolean`
- `subscribe?: boolean`
- `default?: () => Item[]`
- `transform?: (results) => transformedResults`
- `keepPreviousData?: boolean`

Return shape:

- `results`
- `status`
- `isLoading`
- `isStale`
- `isExhausted`
- `hasNextPage`
- `loadMore(numItems)`
- `error`
- `refresh()`
- `reset()`

Constraints:

- use it only with Trellis-supported paginated query refs
- pass nullish args to skip the whole pagination surface cleanly

Example:

```ts
const { results, hasNextPage, loadMore, status } = await useConvexPaginatedQuery(
  api.messages.list,
  { channelId },
  { initialNumItems: 20 },
)
```

## `useCachedQuery`

```ts
useCachedQuery(query, args, {
  from: { query, args, find },
  ...queryOptions
})
```

Seeds a detail query from an already-loaded parent query.

Important options:

- `from.query`
  Already-loaded source query.
- `from.args`
  Args used for that source query cache key.
- `from.find(items)`
  Extracts the detail value from the cached source result.

Return shape:

- the full `useConvexQuery(...)` surface
- `isFromCache`

Use it when list-to-detail navigation should render immediately instead of flashing a pending state.

## `useConvexMutation`

```ts
useConvexMutation(mutation, options?)
```

Default client-side write path for normal state changes.

Important options:

- `optimisticUpdate?: (ctx, args) => void`
- `onSuccess?: (result, args) => void`
- `onError?: (error, args) => void`
- `validate?: ValidateOption`

Return shape:

- callable mutation function
- `data`
- `status`
- `pending`
- `error`
- `reset()`

Constraints and behavior:

- create the composable synchronously; await the returned function call
- latest request wins for the tracked state
- development warns on silent failures if `.error` is never observed

Example:

```ts
const createTodo = useConvexMutation(api.todos.create, {
  onSuccess: () => toast.add({ title: 'Created', color: 'success' }),
})

await createTodo({ title: 'Ship docs' })
```

## `useConvexAction`

```ts
useConvexAction(action, options?)
```

Tracked client-side wrapper for Convex actions.

Its option and return surface matches `useConvexMutation(...)` closely:

- `validate`
- `onSuccess`
- `onError`
- `data`
- `status`
- `pending`
- `error`
- `reset()`

Use it when the backend work needs action semantics, such as third-party calls or longer-running side effects.

## `useConvexAuth`

```ts
useConvexAuth()
```

Primary auth-state composable. Available only when `trellis.auth` is enabled.

Return shape:

- `user`
- `isAuthenticated`
- `isPending`
- `isAnonymous`
- `isSessionExpired`
- `client`
- `refreshAuth()`
- `authError`
- `signOut()`

Important semantics:

- auth state is pre-populated during SSR when session cookies are available
- `refreshAuth()` re-synchronizes Convex auth state; it does not guarantee a logged-in session exists
- `client` is the Better Auth client surface for direct sign-in and provider actions

## `useConvexUpload`

```ts
useConvexUpload(generateUploadUrlMutation, options?)
```

High-level upload composable with queue state, progress, cancellation, and retry-safe bookkeeping.

Important options:

- `maxConcurrent?: number`
- `continueOnError?: boolean`
- `allowedTypes?: string[]`
- `maxSizeBytes?: number`
- `onSuccess?: (storageId, file) => void`
- `onError?: (error, file) => void`
- `onProgress?: (info, file) => void`
- `onQueueIdle?: () => void`

Return shape:

- callable upload function plus `upload(...)`
- `data`
- `status`
- `pending`
- `progress`
- `error`
- `items`
- `cancelItem(id)`
- `cancelAll()`
- `clearFinished()`
- `reset()`

Constraints:

- the mutation must return upload URLs through the Trellis upload flow
- validation options such as type and file-size checks happen client-side before upload dispatch

## `useConvexStorageUrl`

```ts
useConvexStorageUrl(getUrlQuery, storageId)
```

Small wrapper over `useConvexQuery(...)` for storage IDs.

Behavior:

- returns a `ComputedRef<string | null>`
- skips automatically when `storageId` is nullish
- refreshes reactively when the storage ID changes

Use it when you already have a `storageId` and need a browser-usable URL.

## Lower-Traffic Runtime Helpers

These remain intentionally lighter here and are fully discoverable through [API Surface](/docs/api-reference/api-surface):

- `useConvexConnectionState`
- `useConvexAuthActions`
- `useConvexSignIn`
- `useConvexSignUp`
- `useConvexPasswordReset`
- `usePermissions`
- `useAuthGuard`
- optimistic helper utilities such as `prependTo`, `appendTo`, `removeFrom`, and `updateIn`

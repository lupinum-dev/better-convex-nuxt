This page covers the Trellis helpers you use from Nitro routes, middleware, and other Nuxt server files.

## `serverConvexQuery`

```ts
serverConvexQuery(event, fn, args?, options?)
```

Runs a Convex query through the Nuxt server runtime.

## `serverConvexMutation`

```ts
serverConvexMutation(event, fn, args?, options?)
```

Runs a Convex mutation through the Nuxt server runtime.

## `serverConvexAction`

```ts
serverConvexAction(event, fn, args?, options?)
```

Runs a Convex action through the Nuxt server runtime.

## `ServerConvexOptions`

Important options:

- `auth?: 'auto' | 'required' | 'none' | 'trusted'`
- `authToken?: string`
- `actor?: { userId: string }`
- `trustedCallerKey?: string`

Auth mode behavior:

- `auto`
  Use the session cookie when available. This is the default.
- `required`
  Fail if auth cannot be resolved.
- `none`
  Never attach auth.
- `trusted`
  Use the trusted-caller path for server-to-server scoped calls.

Failure behavior:

- missing Convex URL configuration throws a `ConvexCallError`
- auth resolution failures are surfaced as server helper errors with context
- request failures preserve Convex error codes, status, category, and issues when available

## `createServerConvexCaller(...)`

```ts
createServerConvexCaller(event, options?)
```

Creates a request-scoped caller object with:

- `query(fn, args?)`
- `mutation(fn, args?)`
- `action(fn, args?)`

Use it when one Nitro request needs several Convex calls and you do not want to thread `event` through each call manually.

Important constraint:

- it uses `auth: 'none'` intentionally, so protected root refs should receive an explicit forwarded principal when business authorization matters

Example:

```ts
const convex = createServerConvexCaller(event, { principal })
const runbook = await convex.query(internal.runbooks.getForAutomation, { id })
```

## What This Surface Is Not

These helpers are not a second authorization model.

They change how a Nuxt server request reaches Convex. They do not replace:

- Trellis guards
- actor resolution
- tenant boundaries
- backend authorization

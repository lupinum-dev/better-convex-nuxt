Reference for the backend-facing Trellis surfaces used in Convex function files.

## `defineTrellis(...)`

Primary entrypoint for building protected query, mutation, and action builders on top of Convex.

```ts
defineTrellis(builders, options?)
```

### Options

| Option | Purpose |
| --- | --- |
| `principal` | How transport identity is resolved |
| `actor` | How a resolved principal maps to the app actor |
| `tenantIsolation` | Tables that stay tenant-scoped by default |
| `services` | Explicit high-trust service lanes |
| `observability` | Semantic event emission config |
| `destructiveSafety` | Required before registering destructive operations |
| `rls` | Row-level security rules |
| `triggers` | Convex triggers to install on the Trellis runtime path |
| `onSuccess` | Post-success side-work hooks for queries/mutations/actions |

### Returns

| Field | Purpose |
| --- | --- |
| `query` | Protected query builder |
| `mutation` | Protected mutation builder |
| `action` | Protected action builder (when action builders provided) |
| `raw.query` / `raw.mutation` / `raw.action` | Escape-hatch builders |

### DB surface

| Surface | Use |
| --- | --- |
| `ctx.db` | Default safe path |
| `ctx.db.crossTenant` | Explicit tenant-boundary bypass — keeps most runtime structure |
| `ctx.db.raw` | Full escape hatch — emits observability on use |

See [Cross-tenant and raw access](/docs/permissions/cross-tenant-and-raw-access).

## `defineOperation(...)`

Reusable business-action primitive for flows that project cleanly across UI, server, and MCP.

```ts
defineOperation({
  id,
  name,
  kind,
  args,
  guard,
  load,
  authorize,
  preview,
  handler,
})
```

### When to use

- Shared `guard`, `load`, and `authorize` across preview + execute.
- One action needs stable identity across transports.
- Destructive action that needs Trellis-enforced preview/confirm.

### Destructive requirements

::warning
All four must be present for `kind: 'destructive'`:
- `kind` set to `'destructive'`.
- `id` on the operation.
- `preview(...)` function.
- `destructiveSafety` configured in `defineTrellis(...)` before registration.
::

### Projection helpers

| Helper | Purpose |
| --- | --- |
| `mutation(operation)` | Register the execute path |
| `previewOf(operation)` | Produce the preview query projection |

## Handler context accessors

Protected handler context adds these Trellis surfaces:

| Accessor | Returns |
| --- | --- |
| `ctx.principal()` | Resolved transport identity |
| `ctx.actor()` | App-owned actor derived from the principal |
| `ctx.observe(...)` | Emits a semantic observability event |
| `ctx.db` | Default database surface (tenant-scoped) |
| `ctx.db.crossTenant` | Explicit cross-tenant surface |
| `ctx.db.raw` | Escape-hatch surface |

::note
Principal and actor are async runtime facts, not compile-time assumptions. `await ctx.principal()` every time you need it inside a handler.
::

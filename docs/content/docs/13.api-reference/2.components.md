Trellis registers these auth boundary components globally only when `trellis.auth` is enabled.

They are page-shell helpers. They do not replace route protection or backend authorization.

## `<ConvexAuthenticated>`

Renders its default slot only when:

- auth has resolved
- the user is authenticated

It does not render during the pending auth state.

Usage:

```vue
<ConvexAuthenticated>
  <Dashboard />
</ConvexAuthenticated>
```

## `<ConvexUnauthenticated>`

Renders its default slot only when:

- auth has resolved
- the user is unauthenticated

It stays silent during the pending auth state, which prevents login UI from flashing before auth bootstrap finishes.

## `<ConvexAuthLoading>`

Renders its default slot only while auth state is still pending.

Use it for loading shells or skeletons around auth-gated page boundaries.

## `<ConvexAuthError>`

Renders only when auth has failed or ended in an error state.

The default slot receives these props:

- `retry()`
  Re-runs auth refresh without a full page reload.
- `retrying`
- `error`
- `structuredError`
- `isRecoverable`

Important semantics:

- it detects explicit auth failures and token-decode-style failures
- it is intended for recoverable auth-state UI, not for business authorization errors

Usage:

```vue
<ConvexAuthError v-slot="{ retry, retrying, structuredError }">
  <p>{{ structuredError?.message }}</p>
  <button :disabled="retrying" @click="retry">Try again</button>
</ConvexAuthError>
```

## When To Use These Components

Use them when you need:

- page-level auth shells
- clear authenticated vs unauthenticated rendering branches
- a pending auth loading state without writing boilerplate

Do not use them as a substitute for:

- `definePageMeta({ convexAuth: true })`
- backend guards
- resource-level authorization

---
title: 'Workspace To Workspace MCP'
description: 'Hard-cut a tenant-scoped app over to explicit trusted-caller auth for MCP and service flows.'
---

## Goal

Keep tenant scoping, but make the actor resolution path explicit for MCP and service-auth callers.

## Replace The Starter Files

Overwrite the workspace starter core with the MCP-aware starter:

```bash
npx better-convex-nuxt init permissions --model workspace-mcp --force
```

This is a direct replacement of the core auth files, not an added compatibility layer.

## What Changes

- `convex/auth/actor.ts`
  Changes from `getActor(ctx)` to `getActor(ctx, args?)`.
- `convex/auth/actor.ts`
  Adds `getServiceCaller(args)` handling and a service actor branch.
- `convex/auth/actor.ts`
  Keeps browser auth explicit through the same file via `resolveActor(ctx, auth)`.

The rest of the starter stays mostly the same: tenant scoping, `_can`, and the Better Auth bridge do not need a second abstraction layer.

## Handler Changes

Update protected Convex handlers to pass the raw args into actor resolution:

```ts
const actor = await getActor(ctx, args)
```

Update MCP or service-exposed handlers to widen their validators:

```ts
args: withServiceAuth(createRunbook.args)
```

This keeps the shared schema business-only while making the hidden trusted-caller transport explicit in Convex.

## What Does Not Change

- shared schemas stay in `shared/` and stay runtime-neutral
- `defineTool()` still owns tool-local auth, safety, and scope behavior
- tenant ownership still uses `ensureTenant()` and `loadResource()`

Use [`examples/11-mcp-reference`](../../../../examples/11-mcp-reference/README.md) as the reference for the full trusted-caller path.

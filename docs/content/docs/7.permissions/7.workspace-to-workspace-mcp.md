---
title: 'Workspace To Workspace MCP'
description: 'Hard-cut a tenant-scoped app over to explicit trusted-caller auth for MCP and server flows.'
---

## Goal

Keep the canonical workspace-scoped model, but make the actor resolution path explicit for MCP and
trusted callers.

## Replace The Starter Files

Overwrite the workspace starter core with the MCP-aware starter:

```bash
npx trellis init permissions --model workspace-mcp --force
```

This is a direct replacement of the core auth files, not an added compatibility layer.

## What Changes

- `convex/auth/actor.ts`
  Usually stays on `defineActor.fromAuth()` unless your actor needs extra fields.
- `convex/functions.ts`
  Keeps trusted-caller transport inside `createApp(...)` and exposes `app` plus explicit `raw`.
- protected handlers
  Switch from raw `query` / `mutation` to `app.query(...)` / `app.mutation(...)`.
  Keep auth structural with `guard`, optional `load`, and optional `authorize`.

The rest of the starter stays mostly the same: tenant scoping, `_can`, and the Better Auth bridge do not need a second abstraction layer.

The canonical single-workspace contract does not change:

- `workspaceId` stays the tenant foreign key
- role and tenant still resolve from the current `users` row
- `_can`, `defineCapabilities(...)`, and `loadTenantResource()` stay the frontend and backend boundary pattern

## Handler Changes

Add a project-local builder file once:

```ts
// convex/functions.ts
import { createApp } from '@lupinum/trellis/functions'

import { mutation, query } from './_generated/server'
import { getActor } from './auth/actor'

export const { app, raw } = createApp(query, mutation, {
  actor: getActor,
})

export { query, mutation }
```

Then use those builders in protected handlers:

```ts
import { app } from './functions'

export const create = app.mutation({
  guard: canCreateRunbook,
  args: createRunbook.args,
  handler: async (ctx, args) => {
    const actor = await ctx.actor()
  },
})
```

This keeps the shared schema business-only while moving hidden trusted-caller transport into one setup file instead of every handler. Reach for `raw.query(...)` or `raw.mutation(...)` only when you intentionally need the escape hatch.

## Notes

- `ctx.actor()` is memoized only within a single handler invocation. A second call in the same handler reuses the resolved actor, but a new query or mutation starts fresh.
- `tenantIsolation.tables` stays explicit in v1. List only tables that actually store the tenant field. Security policy should not depend on naming conventions.
- Future schema-based inference is possible, but it is intentionally out of scope for v1.
- For the full override matrix on `actor`, trusted callers, RLS, triggers, and `onSuccess`, see [`/docs/permissions/overrides-and-advanced-patterns`](/docs/permissions/overrides-and-advanced-patterns).

## What Does Not Change

- shared schemas stay in `shared/` and stay runtime-neutral
- `defineTool()` still owns tool-local auth, safety, and scope behavior
- tenant ownership still uses `ensureTenant()` and `loadTenantResource()`

Use [`examples/07-mcp-reference`](../../../../examples/07-mcp-reference/README.md) as the reference for the full trusted-caller path.

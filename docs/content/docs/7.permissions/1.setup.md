## Overview

::tip
Use `createPermissions` when you need **reactive, client-side permission checks** that update in real-time -- for example, conditionally showing edit/delete buttons based on role and ownership. If you only check permissions on the backend inside your Convex functions, you don't need this.
::

Better Convex Nuxt provides a `createPermissions` factory that generates reactive permission composables from your backend permission logic.

## Setup

::steps{level="4"}

#### Enable Permissions

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    permissions: true
  }
})
```

#### Create a Permission Context Query

Create a Convex query that returns the user's role and ID:

```ts [convex/auth.ts]
import { query } from './_generated/server'

export const getPermissionContext = query({
  handler: async (ctx) => {
    const identity = await ctx.auth.getUserIdentity()
    if (!identity) return null

    const user = await ctx.db
      .query('users')
      .withIndex('by_token', (q) => q.eq('tokenIdentifier', identity.tokenIdentifier))
      .unique()

    if (!user) return null
    return { role: user.role, userId: user._id, orgId: user.orgId }
  },
})
```

#### Define Permission Logic

Create a file that defines your permission checking function:

```ts [convex/permissions.config.ts]
export type Permission =
  | 'post.create'
  | 'post.read'
  | 'post.update'
  | 'post.delete'
  | 'post.publish'
  | 'org.settings'

export type Resource = {
  ownerId?: string
  [key: string]: unknown
}

export function checkPermission(
  ctx: { role: string; userId: string } | null,
  permission: Permission,
  resource?: Resource,
): boolean {
  if (!ctx) return false

  // Admins can do everything
  if (ctx.role === 'admin') return true

  switch (permission) {
    case 'post.read':
      return true
    case 'post.create':
      return ctx.role === 'editor' || ctx.role === 'admin'
    case 'post.update':
    case 'post.delete':
      return resource?.ownerId === ctx.userId
    case 'post.publish':
      return ctx.role === 'editor'
    case 'org.settings':
      return ctx.role === 'admin'
    default:
      return false
  }
}
```

#### Create the Composable

```ts [composables/usePermissions.ts]
import { createPermissions } from '#imports'
import { api } from '~~/convex/_generated/api'
import { checkPermission } from '~~/convex/permissions.config'

export const { usePermissions, usePermissionGuard } = createPermissions({
  query: api.auth.getPermissionContext,
  checkPermission,
})
```

::

## How It Works

The `createPermissions` factory:

1. Creates a `usePermissions` composable that fetches the permission context via a Convex query
2. Creates a `usePermissionGuard` composable for page-level protection
3. Permission checks are **reactive** — they update when the user's role or context changes
4. The permission context query is a standard Convex query with real-time subscriptions

## Factory Options

::field-group
::field{name="query" type="FunctionReference<'query'>" required}
A Convex query that returns the user's permission context (`{ role, userId, orgId? }`). Returns `null` for unauthenticated users.
::
::field{name="checkPermission" type="CheckPermissionFn" required}
A function that checks if a user with the given context has a specific permission, optionally against a resource.
::

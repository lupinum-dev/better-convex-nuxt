## Overview

::tip
Use `createPermissions` when you need **reactive, client-side permission checks** that update in real time. If you only enforce permissions inside Convex handlers, you do not need this.
::

Better Convex Nuxt provides a `createPermissions` factory that generates reactive permission composables from your backend permission logic.

## Setup

::steps{level="4"}

#### Enable Permissions

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    permissions: {
      config: '~/convex/permissions.config'
    }
  }
})
```

#### Create a Permission Context Query

Create a Convex query that returns the user's permission context:

```ts [convex/auth.ts]
import { query } from './_generated/server'

export const getPermissionContext = query({
  handler: async (ctx) => {
    const identity = await ctx.auth.getUserIdentity()
    if (!identity) return null

    const user = await ctx.db
      .query('users')
      .withIndex('by_token', (q) => q.eq('tokenIdentifier', identity.tokenIdentifier))
      .unique()

    if (!user) return null
    return {
      role: user.role,
      userId: user._id,
      tenantId: user.organizationId,
    }
  },
})
```

#### Define Permission Logic

Create a permission config. `definePermissions()` generates the standard role / own / any checker for you:

```ts [convex/permissions.config.ts]
import { definePermissions } from 'better-convex-nuxt/convex'

export const permissionConfig = definePermissions({
  roles: ['owner', 'admin', 'member', 'viewer'] as const,
  rules: {
    global: {
      'org.settings': { roles: ['owner'] },
    },
    post: {
      create: { roles: ['owner', 'admin', 'member'] },
      update: { own: ['member'], any: ['owner', 'admin'] },
      delete: { own: ['member'], any: ['owner', 'admin'] },
    },
  },
})
```

#### Create the Composable

```ts [composables/usePermissions.ts]
import { createPermissions } from '#imports'
import { api } from '~~/convex/_generated/api'
import { permissionConfig } from '~~/convex/permissions.config'

export const { usePermissions, usePermissionGuard } = createPermissions({
  query: api.auth.getPermissionContext,
  checkPermission: permissionConfig.checkPermission,
})
```

That tiny `composables/usePermissions.ts` file is intentional. Nuxt auto-imports files in
`composables/`, so this app-local bridge is how your project gets a zero-import `usePermissions()`
without pretending the module can guess your permission-context query automatically.

::

## How It Works

The `createPermissions` factory:

1. Creates a `usePermissions` composable that fetches the permission context via a Convex query
2. Creates a `usePermissionGuard` composable for page-level protection
3. Permission checks are **reactive** — they update when the user's role or context changes
4. The permission context query is a standard Convex query with real-time subscriptions

## Factory Options

::field-group
::field{name="query" type="FunctionReference<'query'>" required}
A Convex query that returns the user's permission context (`{ role, userId, tenantId? }`). Returns `null` for unauthenticated users.
::
::field{name="checkPermission" type="CheckPermissionFn" required}
A function that checks if a user with the given context has a specific permission, optionally against a resource.
::
::

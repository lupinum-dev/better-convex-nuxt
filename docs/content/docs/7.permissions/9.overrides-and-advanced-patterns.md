## Overview

The canonical server surface is now:

- `createApp(query, mutation, options)`
- `app.query(...)` / `app.mutation(...)` for normal handlers
- `raw.query(...)` / `raw.mutation(...)` for intentional escape hatches

The rule is simple:

- use `app.*` by default
- keep policy in `convex/auth/*`
- drop to `raw.*` only when the structured surface is the wrong fit for that one handler

## What `createApp(...)` Owns

`createApp(...)` composes:

- hidden trusted-caller transport
- lazy, memoized `ctx.actor()`
- optional tenant isolation
- optional custom row-level security
- optional triggers
- optional `onSuccess` hooks

It returns:

- `app`
  The canonical structured handler builders. These require `guard`, support `load`, and support `authorize`.
- `raw`
  The explicit escape hatch. These preserve the actor-aware builder behavior without the structural phases.

## Default Pattern

```ts [convex/functions.ts]
import { createApp } from '@lupinum/trellis/functions'

import { mutation, query } from './_generated/server'
import { getActor } from './auth/actor'

export const { app, raw } = createApp(query, mutation, {
  actor: getActor,
  tenantIsolation: {
    tables: ['projects', 'tasks'],
  },
})

export { query, mutation }
```

Then author normal handlers through `app`:

```ts [convex/tasks.ts]
export const create = app.mutation({
  guard: canCreateTask,
  args: createTask.args,
  handler: async (ctx, args) => {
    const actor = await ctx.actor()
    return await ctx.db.insert('tasks', {
      title: args.title,
      ownerId: actor.userId,
      workspaceId: actor.tenantId,
      createdAt: Date.now(),
    })
  },
})
```

## When `raw.*` Is Correct

Use `raw.query(...)` or `raw.mutation(...)` only when the structured shape is not the right tool:

- upload URL generation
- transport probes and harness coverage
- very low-level adapters that should stay close to raw Convex
- deliberately public or passthrough handlers where `guard`, `load`, and `authorize` add no value

Example:

```ts [convex/files.ts]
export const generateUploadUrl = raw.mutation({
  args: {},
  handler: async (ctx) => {
    const actor = await ctx.actor()
    enforce(actor, 'Generate upload URL', isAuthenticated)
    return await ctx.storage.generateUploadUrl()
  },
})
```

That is acceptable because the raw path is explicit in the declaration.

## Override Matrix

| Concern                  | Default                                | Extend                          | Replace / Escape Hatch                      |
| ------------------------ | -------------------------------------- | ------------------------------- | ------------------------------------------- |
| Actor resolution         | `defineActor.fromAuth()`               | `.extend(...)` / `.filter(...)` | `createApp(..., { actor })`                 |
| Trusted caller transport | On by default                          | `trustedCaller: false`          | your own transport outside `createApp(...)` |
| Tenant isolation         | explicit `tenantIsolation.tables`      | custom `field`                  | `rls.rules` or a custom lower-level seam    |
| Handler structure        | `app.query(...)` / `app.mutation(...)` | `load` and `authorize`          | `raw.query(...)` / `raw.mutation(...)`      |
| Permission context       | `definePermissionContext(...)`         | `extend(...)`                   | bespoke `app.query(...)`                    |
| Resource capabilities    | `defineCapabilities(...).attach(...)`  | separate per-resource maps      | bespoke mapping inside one handler          |

## Trusted Caller Notes

If you enable trusted callers, keep that configuration inside `createApp(...)`. The point of the seam is that the shared schema stays business-only while the transport stays hidden in `convex/functions.ts`.

## Do Not Reintroduce The Old Split

The repo no longer teaches the legacy split between one low-level factory, one wrapper builder, and custom `appQuery` / `appMutation` names.

That transition state is gone from the canonical path. If you need something lower level, use `raw.*` explicitly.

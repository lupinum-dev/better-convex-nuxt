## Overview

`usePermissions()` provides reactive permission checks that update automatically when the user's role or context changes.

## Basic Usage

```vue
<script setup lang="ts">
import { usePermissions } from '~/composables/usePermissions'
import { api } from '~~/convex/_generated/api'

const { can, role, isAuthenticated, pending } = usePermissions()
const { data: post } = await useConvexQuery(api.posts.get, () => ({ id: route.params.id }))
</script>

<template>
  <div v-if="pending">Loading permissions...</div>
  <article v-else-if="post">
    <h1>{{ post.title }}</h1>
    <p>{{ post.content }}</p>

    <div class="actions">
      <button v-if="can('post.update', post).value">Edit</button>
      <button v-if="can('post.delete', post).value">Delete</button>
      <button v-if="can('post.publish').value">Publish</button>
    </div>

    <p>Your role: {{ role }}</p>
  </article>
</template>
```

## Return Value

::field-group
::field{name="can(permission, resource?)" type="(permission: string, resource?: Resource) => ComputedRef<boolean>"}
Returns a reactive `ComputedRef<boolean>` indicating whether the user has the given permission. Pass an optional `resource` for ownership checks.
::
::field{name="user" type="ComputedRef<PermissionContext | null>"}
The full permission context returned by your Convex query.
::
::field{name="role" type="ComputedRef<string | null>"}
The user's current role.
::
::field{name="orgId" type="ComputedRef<string | null>"}
The user's organization ID (if provided by your permission context query).
::
::field{name="isAuthenticated" type="ComputedRef<boolean>"}
Whether a valid permission context exists.
::
::field{name="pending" type="Ref<boolean>"}
Whether the permission context is still loading.
::
::

## Ownership Checks

Pass a resource object as the second argument to `can()` for ownership-based permissions:

```vue
<script setup lang="ts">
const { can } = usePermissions()

// Check if user can edit this specific post (ownership check)
const canEdit = can('post.update', { ownerId: post.value?.authorId })
</script>
```

Your `checkPermission` function receives the resource and can compare `ctx.userId` with `resource.ownerId`.

## Conditional Rendering

Since `can()` returns a `ComputedRef`, use `.value` in templates:

```vue
<template>
  <!-- Correct -->
  <button v-if="can('post.create').value">New Post</button>

  <!-- Also works — assign to a variable first -->
  <button v-if="canCreate.value">New Post</button>
</template>
```

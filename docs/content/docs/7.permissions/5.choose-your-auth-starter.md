## Starter Matrix

| App shape                                              | Command                                                   | What it scaffolds                                                                                    | Read next                                                                      |
| ------------------------------------------------------ | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Authenticated app without tenants                      | `npx better-convex-nuxt add auth --starter personal`      | Better Auth bridge, `getActor(ctx)`, personal ownership helpers, `convex/test.setup.ts`              | [`examples/02-auth-todo`](../../../../examples/02-auth-todo/README.md)         |
| Tenant-scoped app                                      | `npx better-convex-nuxt add auth --starter workspace`     | Better Auth bridge, tenant actor, `ensureTenant()`, `loadResource()`, `_can`, `convex/test.setup.ts` | [`examples/03-team-todo`](../../../../examples/03-team-todo/README.md)         |
| Tenant-scoped app with MCP or server-to-server callers | `npx better-convex-nuxt add auth --starter workspace-mcp` | Workspace starter plus explicit `getActor(ctx, args)` and trusted-caller support                     | [`examples/11-mcp-reference`](../../../../examples/11-mcp-reference/README.md) |

If you prefer prompts, run `npx better-convex-nuxt add auth` and pick the starter interactively.

## Additive Blocks

Start with one of the starters above, then add vertical helpers as needed:

- `npx better-convex-nuxt add auth:crm` for visibility and redaction helpers
- `npx better-convex-nuxt add auth:lms` for enrollment and prerequisite helpers
- `npx better-convex-nuxt add auth:ecommerce` for service auth and idempotency helpers
- `npx better-convex-nuxt add auth:freemium` for plan and usage-limit helpers
- `npx better-convex-nuxt add auth:collaboration` for share-token and page-access helpers
- `npx better-convex-nuxt add auth:agency` for cross-tenant membership helpers

The block gallery lives in [`/docs/permissions/saas-examples`](/docs/permissions/saas-examples).

## What The Starter Owns

Every starter copies plain app code into your repo. You keep and modify:

- `convex/auth.ts`
- `convex/auth.config.ts`
- `convex/http.ts`
- `convex/convex.config.ts`
- `convex/auth/*`
- `convex/test.setup.ts`
- `composables/usePermissions.ts`

Nothing here is a hidden runtime builder. The scaffold is only a starting point.

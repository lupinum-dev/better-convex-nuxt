## Starter Matrix

| App shape                                              | Command                                                                          | What it scaffolds                                                                                                 | Read next                                                                      |
| ------------------------------------------------------ | -------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Authenticated app without tenants                      | `npx better-convex-nuxt init auth` then `init permissions --model personal`      | Better Auth bridge, `getActor(ctx)`, personal ownership helpers, `convex/test.setup.ts`                           | [`examples/02-auth-todo`](../../../../examples/02-auth-todo/README.md)         |
| Tenant-scoped app                                      | `npx better-convex-nuxt init auth` then `init permissions --model workspace`     | Better Auth bridge, canonical workspace actor, `ensureTenant()`, `loadResource()`, `_can`, `convex/test.setup.ts` | [`examples/03-team-todo`](../../../../examples/03-team-todo/README.md)         |

## What The Starter Owns

Every starter copies plain app code into your repo. You keep and modify:

- `convex/auth.ts`
- `convex/auth.config.ts`
- `convex/http.ts`
- `convex/convex.config.ts`
- `convex/auth/*`
- `convex/test.setup.ts`
- `composables/usePermissions.ts`

Nothing here is a hidden runtime builder. The scaffold is only a starting point.

For the workspace boundary files themselves, use the examples as the canonical references:

- single-workspace template: [`examples/03-team-todo`](../../../../examples/03-team-todo/README.md)
- memberships template: [`examples/10-agency-portal`](../../../../examples/10-agency-portal/README.md)

The canonical starter contract is:

- `workspaceId` on scoped tables
- `by_workspace` indexes
- `users.authId`, `users.role`, and `users.workspaceId`
- `ownerId` storing the auth-subject string

If your app follows that shape, `createTestContext({ schema, modules })` works without extra config.
Reach for the advanced testing overrides only when your schema intentionally diverges.

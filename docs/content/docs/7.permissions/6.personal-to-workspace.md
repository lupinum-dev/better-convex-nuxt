---
title: 'Personal To Workspace'
description: 'Hard-cut a personal-auth app over to tenant-scoped auth.'
---

## Goal

Replace a plain authenticated app with a tenant-scoped one. This is a hard cutover, not a dual-path migration.

## Replace The Starter Files

Overwrite the personal starter core with the workspace starter:

```bash
npx trellis init permissions --model workspace --force
```

That replaces the generated auth setup and core auth files with the tenant-scoped versions.

## What Changes

- `convex/auth/actor.ts`
  Switch to the canonical workspace actor shape, usually via `defineActor.fromAuth()`.
- `convex/functions.ts`
  Switches to `const { app, raw } = createApp(...)`.
- `convex/auth/checks.ts`
  Switch from personal ownership checks to role-aware workspace checks.
- `composables/usePermissions.ts`
  Point the frontend at `api.workspaces.getPermissionContext`.

## Handler Changes

Update handlers from personal ownership to tenant scoping:

- Before: raw `query` / `mutation`
- After: `app.query(...)` / `app.mutation(...)` from `convex/functions.ts`

- Before: `const actor = await ctx.actor()`
- After: `const actor = await ctx.actor()`

The call site stays simple, but the returned actor now carries `role` and `tenantId`.

- Before: `loadOwnedResource(actor, doc, 'Todo')`
- After: direct `ensureTenant(actor, doc)` or `loadTenantResource(actor, doc, 'Todo')`

- Before: write rows keyed by `ownerId`
- After: keep `ownerId` and add `workspaceId`

## Data Model Follow-Up

The starter cutover only replaces code. You still need to align your tables:

- add a tenant table such as `workspaces`
- add `workspaceId` to tenant-owned records
- add a `role` field to `users`
- add the indexes used by the scaffolded actor and tenant queries

Use [`examples/03-team-workspace`](../../../../examples/03-team-workspace/README.md) as the smallest working reference.

---
title: "Permission Guards"
description: "Protect pages based on user permissions with automatic redirects."
---

## Overview

`usePermissionGuard` protects pages by redirecting users who lack the required permission.

## Basic Usage

```vue
<script setup lang="ts">
import { usePermissionGuard } from '~/composables/usePermissions'

usePermissionGuard({
  permission: 'org.settings',
  redirectTo: '/dashboard',
})
</script>

<template>
  <h1>Organization Settings</h1>
  <!-- Only renders if user has 'org.settings' permission -->
</template>
```

## Options

::field-group
::field{name="permission" type="string" required}
The permission required to access this page.
::
::field{name="redirectTo" type="RouteLocationRaw"}
Where to redirect if the user lacks the permission. Default: `'/'`.
::
::field{name="resource" type="Resource"}
Optional resource for ownership-based permission checks.
::
::field{name="loginPath" type="RouteLocationRaw"}
Where to redirect if the user is not authenticated at all. Default: `'/auth/signin'`.
::
::

## Behavior

1. **Waits for permissions to load** — no redirect while `pending` is `true`
2. **Redirects unauthenticated users** to `loginPath`
3. **Redirects unauthorized users** to `redirectTo`
4. **Anti-loop protection** — tracks pending redirects to prevent rapid redirect loops

## With Resource

```vue
<script setup lang="ts">
import { usePermissionGuard } from '~/composables/usePermissions'

const route = useRoute()
const { data: post } = await useConvexQuery(api.posts.get, () => ({
  id: route.params.id,
}))

usePermissionGuard({
  permission: 'post.update',
  resource: { ownerId: post.value?.authorId },
  redirectTo: `/posts/${route.params.id}`,
})
</script>
```

---
title: "Auth Guards"
description: "Protect pages with backend-computed capability keys."
---

## Overview

`useAuthGuard` redirects when the current user is unauthenticated or lacks a required capability key from the permission context query.

## Basic Usage

```vue
<script setup lang="ts">
useAuthGuard({
  can: 'workspace.members',
  redirectTo: '/',
})
</script>
```

## Options

::field-group
::field{name="can" type="string" required}
Capability key to read from the backend context query.
::
::field{name="redirectTo" type="RouteLocationRaw"}
Destination for authenticated users who lack the capability. Default: `'/'`.
::
::field{name="loginPath" type="RouteLocationRaw"}
Destination for unauthenticated users. Default: `'/auth/signin'`.
::
::

## Important Constraint

`useAuthGuard` is for global capability keys from your permission context query. Resource-specific authorization stays on the backend and should flow into page components via `_can` or other backend-owned response data.

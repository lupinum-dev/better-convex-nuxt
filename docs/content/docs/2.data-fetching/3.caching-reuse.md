## The Goal

Use already-loaded query data to make navigation feel instant, while still fetching fresh data in the background.

## Supported Pattern (No Extra Cache API)

This library does **not** expose `useConvexCached` / `useConvexData`, and does not expose a public cache-key
helper for reading another route's `useAsyncData` entry from the outside.

Instead, use:

- `useConvexQuery(...)` — each call manages its own SSR/subscription cache internally.
- `initialData` — seed a query with an already-known value (for example a field from a parent list item)
  so the UI renders instantly while the full record loads in the background.
- `defineSharedConvexQuery(...)` — share one live query state across multiple components/pages within
  the same app instance, instead of re-fetching from scratch. See [Shared Queries](/docs/data-fetching/queries#shared-queries).

## Example: List -> Detail with an Instant Placeholder

Pass along the fields you already have (for example from a list item the user clicked) as `initialData`,
so the detail page renders immediately while the full record streams in.

```vue [app/pages/posts/index.vue]
<script setup lang="ts">
import { api } from '#convex/api'

const { data: posts } = await useConvexQuery(api.posts.list, {})
</script>

<template>
  <div>
    <NuxtLink
      v-for="post in posts"
      :key="post._id"
      :to="{ path: `/posts/${post.slug}`, state: { preview: post } }"
    >
      {{ post.title }}
    </NuxtLink>
  </div>
</template>
```

```vue [app/pages/posts/[slug].vue]
<script setup lang="ts">
import { api } from '#convex/api'

const route = useRoute()
const slug = computed(() => route.params.slug as string)
const preview = route.state?.preview as { title: string; slug: string } | undefined

const { data: post, pending } = await useConvexQuery(
  api.posts.getBySlug,
  computed(() => ({ slug: slug.value })),
  { initialData: () => preview },
)
</script>

<template>
  <div v-if="pending && !post">Loading...</div>
  <article v-else-if="post">
    <h1>{{ post.title }}</h1>
    <div v-if="post.content">{{ post.content }}</div>
    <div v-else>Loading full content...</div>
  </article>
</template>
```

## Best Practices

- Use `useConvexQuery` on detail pages to avoid navigation blocking.
- Keep list queries lightweight (exclude heavy fields like full content).
- Handle the "placeholder present, full data still loading" state gracefully (`pending && !post`).
- Use `defineSharedConvexQuery` when multiple components need to share one live subscription instead of
  each mounting its own.

## Important Note

Caching and reuse are handled entirely by `useConvexQuery` (via `subscribe: false`) and
`defineSharedConvexQuery`. There is no separate caching composable or query-key helper to reach for —
see the [API Surface](/docs/advanced/api-surface) for the full supported list.

Use this page as a pre-launch checklist, not as a tutorial.

- [ ] `CONVEX_URL` points at the intended production deployment.
- [ ] `SITE_URL` matches the real public app origin.
- [ ] `BETTER_AUTH_SECRET` is set anywhere auth is enabled.
- [ ] `CONVEX_SITE_URL` is set when your auth token exchange path needs it.
- [ ] `CONVEX_TRUSTED_CALLER_KEY` is set anywhere trusted-caller or MCP traffic exists.
- [ ] `trellis.permissions.query` points at a real permission-context query when the UI depends on configured permissions.
- [ ] Route protection and permission guards are both exercised in staging.
- [ ] Destructive operations have preview and confirmation flows where Trellis requires them.
- [ ] Any `ctx.db.crossTenant` and `ctx.db.raw` seams are intentional and reviewed.
- [ ] If SSR auth cache is enabled, its storage choice matches the deployment topology.
- [ ] `pnpm check:docs:links` is green if docs are part of the release.
- [ ] The relevant test lanes have run for the changed surface.

## Suggested Gate

For this repo, the strongest default gate is still:

```bash
pnpm test:types && pnpm lint && pnpm test:contracts
```

Use broader gates when the touched surface reaches deeper:

```bash
pnpm test
pnpm run release:verify
```

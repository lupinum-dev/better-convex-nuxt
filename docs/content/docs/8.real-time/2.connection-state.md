## Overview

`useConvexConnectionState` provides reactive access to the WebSocket connection status, letting you build offline indicators, reconnecting banners, and pending mutation counters.

## Basic Usage

```vue
<script setup lang="ts">
const {
  isConnected,
  isReconnecting,
  pendingMutations,
  shouldShowOfflineUi
} = useConvexConnectionState()
</script>

<template>
  <div v-if="shouldShowOfflineUi" class="offline-banner">
    <span v-if="isReconnecting">Reconnecting...</span>
    <span v-else>You are offline</span>
    <span v-if="pendingMutations > 0">
      ({{ pendingMutations }} pending changes)
    </span>
  </div>
</template>
```

## Return Value

::field-group
::field{name="state" type="Readonly<Ref<ConnectionState>>"}
The full connection state object.
::
::field{name="isConnected" type="ComputedRef<boolean>"}
`true` when the WebSocket is active and connected.
::
::field{name="isReconnecting" type="ComputedRef<boolean>"}
`true` when the client was previously connected but has lost the connection and is attempting to reconnect.
::
::field{name="pendingMutations" type="ComputedRef<number>"}
Number of mutations waiting to be sent or confirmed.
::
::field{name="shouldShowOfflineUi" type="ComputedRef<boolean>"}
Delayed flag that suppresses offline UI during the initial hydration grace period (500ms). Use this instead of `!isConnected` to avoid a flash of offline UI during page load.
::
::

## Full ConnectionState

The `state` ref contains the complete connection information:

| Property | Type | Description |
|----------|------|-------------|
| `isWebSocketConnected` | `boolean` | WebSocket is active |
| `hasEverConnected` | `boolean` | Has connected at least once |
| `hasInflightRequests` | `boolean` | Requests in progress |
| `timeOfOldestInflightRequest` | `Date \| null` | Oldest pending request |
| `connectionCount` | `number` | Total connections made |
| `connectionRetries` | `number` | Retry attempts |
| `inflightMutations` | `number` | Pending mutations |
| `inflightActions` | `number` | Pending actions |

## Hydration Grace Period

`shouldShowOfflineUi` waits 500ms after hydration before showing offline UI. This prevents a brief flash of "offline" content that can occur during the WebSocket connection setup after SSR hydration.

```vue
<!-- Good: uses grace period -->
<div v-if="shouldShowOfflineUi">Offline</div>

<!-- Avoid: may flash briefly during hydration -->
<div v-if="!isConnected">Offline</div>
```

## Singleton Pattern

`useConvexConnectionState` uses a singleton pattern — calling it from multiple components shares the same underlying subscription to the Convex client's connection state.

## Runtime Hook

If you need side effects instead of template bindings, listen to `convex:connection:changed`.
It fires only when the derived phase changes, not on every raw counter update.

See [Runtime Hooks](/docs/api-reference/runtime-hooks) for the full hook table and payload reference.

```ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('convex:connection:changed', ({ state, previousState, connection }) => {
    console.log(`Convex connection ${previousState} -> ${state}`)

    if (state === 'reconnecting' && connection.inflightMutations > 0) {
      console.warn('Connection lost with pending mutations')
    }
  })
})
```

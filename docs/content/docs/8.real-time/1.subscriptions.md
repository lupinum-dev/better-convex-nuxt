## Overview

By default, every query in better-convex-nuxt maintains a live WebSocket subscription to the Convex backend. When any client changes the data, all subscribed clients receive the update instantly.

## How It Works

1. During **SSR**, queries run via HTTP against the Convex backend
2. On the **client**, the Convex client opens a WebSocket connection
3. Each `useConvexQuery` call creates a **subscription** for its query + args combination
4. When the backend data changes, the subscription pushes the update to the client
5. The `data` ref updates automatically — no polling, no manual refetch

## Disabling Subscriptions

For data that doesn't need real-time updates, disable subscriptions:

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

// One-shot query — fetches once, no live updates
const { data: config } = await useConvexQuery(api.app.getConfig, {}, {
  subscribe: false
})
</script>
```

## Global Default

Disable subscriptions globally and opt-in per query:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    query: {
      subscribe: false // All queries are one-shot by default
    }
  }
})
```

```vue
<script setup lang="ts">
// Override: enable subscription for this specific query
const { data: messages } = await useConvexQuery(api.chat.messages, {}, {
  subscribe: true
})
</script>
```

## Subscription Deduplication

When multiple components subscribe to the same query with the same args, they share a single WebSocket subscription. The subscription is released when the last component using it unmounts.

This happens automatically — no configuration needed.

## Subscription Lifecycle

- **Created** when a component using `useConvexQuery` mounts
- **Shared** when multiple components use the same query + args
- **Released** when the last consuming component unmounts (via `onScopeDispose`)

::tip
Use the `shared` option on `useConvexQuery` to explicitly share state across components that use the same query. See [Shared Queries](/docs/data-fetching/shared-queries).
::

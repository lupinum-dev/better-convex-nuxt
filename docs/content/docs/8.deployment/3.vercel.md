This is the recommended concrete deployment walkthrough for less senior users.

## What You Will Configure

You will set values in two places:

1. Vercel project environment variables
2. Convex Dashboard environment variables

If auth is enabled, both sides matter.

## 1. Prepare Convex

Make sure your Convex project already has:

- `convex/convex.config.ts` with the Better Auth component enabled
- `convex/http.ts` registering Better Auth routes
- `convex/auth.ts` reading `SITE_URL`

If `convex/http.ts` does not register Better Auth routes, the Nuxt auth proxy will have nothing to forward requests to.

## 2. Set Convex Dashboard Env Vars

In Convex Dashboard, set:

| Variable | Example |
| -------- | ------- |
| `SITE_URL` | `https://your-app.vercel.app` or your custom domain |
| `BETTER_AUTH_SECRET` | generated secret |
| `GITHUB_CLIENT_ID` | only if GitHub auth is enabled |
| `GITHUB_CLIENT_SECRET` | only if GitHub auth is enabled |

If you use static JWKS, also set:

| Variable | Example |
| -------- | ------- |
| `JWKS` | output from `npx convex run auth:getLatestJwks | npx convex env set JWKS` |

## 3. Set Vercel Env Vars

In Vercel, set:

| Variable | Example |
| -------- | ------- |
| `CONVEX_URL` | `https://your-app.convex.cloud` |
| `CONVEX_SITE_URL` | `https://your-app.convex.site` only if you need an explicit override |

Only add `NUXT_PUBLIC_CONVEX_URL` / `NUXT_PUBLIC_CONVEX_SITE_URL` if you intentionally rely on Nuxt public runtime env fallback behavior.

If you enable shared SSR auth cache with Redis, also set:

| Variable | Example |
| -------- | ------- |
| `REDIS_URL` | `redis://...` |

## 4. Configure `nuxt.config.ts`

Your Nuxt config should stay simple:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],
  convex: {
    url: process.env.CONVEX_URL,
    siteUrl: process.env.CONVEX_SITE_URL,
  },
})
```

`siteUrl` is safe to keep as an explicit env-backed override, but most apps can omit it when `CONVEX_URL` is a normal `*.convex.cloud` URL.

## 5. OAuth Callback Checklist

If you use GitHub auth:

- GitHub callback URL should be `https://your-app.vercel.app/api/auth/callback/github`
- if you use a custom domain, update the callback URL to that domain
- `SITE_URL` in Convex Dashboard must match the domain you expect users to sign in on

## 6. SSR Auth Proxy Expectations

With SSR enabled:

- the module registers `/api/auth` and `/api/auth/**` on your Nuxt app
- browser requests hit your Vercel deployment first
- Nuxt forwards those requests to Convex HTTP actions
- Better Auth cookies stay on your app domain

That means:

- `SITE_URL` should be your app domain
- `CONVEX_SITE_URL` should be your Convex HTTP actions domain

## Common Mistakes

- Setting `SITE_URL` to `https://your-app.convex.site` instead of your app domain
- Forgetting to register Better Auth routes in `convex/http.ts`
- Only setting `BETTER_AUTH_SECRET` locally and not in Convex Dashboard
- Adding `CONVEX_DEPLOYMENT` but forgetting `CONVEX_URL`
- Using an OAuth callback URL that points at Convex instead of your app

## After Deploying

Verify:

1. The app loads and can run queries.
2. `/api/auth/get-session` works from the deployed app.
3. OAuth redirects return to your app domain.
4. Protected SSR pages do not redirect-loop.

If something fails, go to [Deployment Troubleshooting](/docs/deployment/troubleshooting).

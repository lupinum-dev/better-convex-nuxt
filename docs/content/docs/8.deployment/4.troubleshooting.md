## `siteUrl` Missing

Symptoms:

- auth proxy errors mentioning missing `siteUrl`
- server plugin warnings about failing to resolve a usable site URL

Fix:

- set `CONVEX_URL` first
- if the module cannot derive the HTTP actions host, set `CONVEX_SITE_URL` explicitly
- remember that custom domains and non-standard hosts usually need an explicit `siteUrl`

## `SITE_URL` vs `CONVEX_SITE_URL` Mixed Up

Symptoms:

- OAuth redirects go to the wrong host
- callback URLs mismatch
- auth requests appear to target Convex directly when they should go through your app

Fix:

- `SITE_URL` should be your app origin, like `https://app.example.com`
- `CONVEX_SITE_URL` should be your Convex HTTP actions origin, like `https://your-app.convex.site`

## Better Auth Routes Not Registered

Symptoms:

- `/api/auth/*` returns upstream 404s
- local/dev logs mention missing Better Auth routes

Fix:

- confirm `convex/http.ts` calls `authComponent.registerRoutes(http, createAuth)`
- make sure those Convex changes are deployed

## OAuth Callback Mismatch

Symptoms:

- provider rejects the callback
- login succeeds at the provider but fails on return

Fix:

- provider callback must point at your app domain, not Convex
- example GitHub callback:
  `https://app.example.com/api/auth/callback/github`
- `SITE_URL` in Convex Dashboard should match the same app domain

## Redis Problems

Only relevant if you enabled shared SSR auth cache.

Symptoms:

- auth cache works locally but not across instances
- runtime errors when Nitro initializes storage

Fix:

- confirm `REDIS_URL` exists in the Nuxt host environment
- confirm `nitro.storage['cache:convex:auth']` is configured
- if you do not need shared cache, remove the Redis config and use the default in-memory cache

## JWKS Problems

Only relevant if you opted into static JWKS.

Symptoms:

- JWT verification failures after deployment
- auth starts failing after key rotation

Fix:

- regenerate `JWKS`
- set the new value in Convex Dashboard
- redeploy if your deployment flow snapshots env values

This page is the source of truth for environment variables used by this module and the official auth examples in this repo.

## The Short Version

- `CONVEX_URL` is the primary Nuxt module input.
- `CONVEX_SITE_URL` is optional unless auto-derivation is wrong or intentionally overridden.
- `NUXT_PUBLIC_CONVEX_URL` and `NUXT_PUBLIC_CONVEX_SITE_URL` are supported fallbacks, not the preferred first choice for most apps.
- `SITE_URL` belongs to Better Auth inside Convex. It is not the same thing as `convex.siteUrl`.
- `BETTER_AUTH_SECRET` belongs in Convex Dashboard when auth is enabled.

## Nuxt App Environment

These are read by the Nuxt module and app host runtime.

| Variable | Required? | Who reads it | Where to set it | Example | Notes |
| -------- | --------- | ------------ | --------------- | ------- | ----- |
| `CONVEX_URL` | Yes | `better-convex-nuxt` module/runtime | Nuxt host env or `nuxt.config.ts` process env | `https://your-app.convex.cloud` | Primary module input. Preferred default for most apps. |
| `CONVEX_SITE_URL` | Usually no | `better-convex-nuxt` module/runtime | Nuxt host env or `nuxt.config.ts` process env | `https://your-app.convex.site` | Optional when the module can derive it from `CONVEX_URL`. Required for custom/non-standard HTTP actions domains. |
| `NUXT_PUBLIC_CONVEX_URL` | No | `better-convex-nuxt` runtime fallback | Nuxt public env | `https://your-app.convex.cloud` | Supported fallback when you need the value exposed through Nuxt public runtime config. Do not duplicate it unless you have a platform reason. |
| `NUXT_PUBLIC_CONVEX_SITE_URL` | No | `better-convex-nuxt` runtime fallback | Nuxt public env | `https://your-app.convex.site` | Supported fallback for the site URL override. Usually unnecessary unless you rely on public runtime env injection. |
| `CONVEX_DEPLOYMENT` | No | Convex CLI, not this module | Local `.env.local` or CI only if Convex tooling needs it | `dev:calm-river-123` | Metadata written by the Convex CLI. The module does not use it. |

## Convex Dashboard Environment

These are read by your Convex backend code and Better Auth setup.

| Variable | Required? | Who reads it | Where to set it | Example | Notes |
| -------- | --------- | ------------ | --------------- | ------- | ----- |
| `SITE_URL` | Yes when auth is enabled | Better Auth inside `convex/auth.ts` | Convex Dashboard env vars | `https://app.example.com` | Your app origin. Used for redirects, callbacks, and Better Auth base URL in Convex. |
| `BETTER_AUTH_SECRET` | Yes when auth is enabled | Better Auth | Convex Dashboard env vars | `openssl rand -base64 32` output | Required for production auth. Do not keep this only in local Nuxt env. |

## Provider Credentials

These depend on the auth providers you actually enable.

| Variable | Required? | Who reads it | Where to set it | Example | Notes |
| -------- | --------- | ------------ | --------------- | ------- | ----- |
| `GITHUB_CLIENT_ID` | Only if GitHub auth is enabled | `convex/auth.ts` | Convex Dashboard env vars | `Iv1.1234567890abcdef` | Callback URL should point at your app origin plus `/api/auth/callback/github`. |
| `GITHUB_CLIENT_SECRET` | Only if GitHub auth is enabled | `convex/auth.ts` | Convex Dashboard env vars | `ghs_xxxxxxxxxxxx` | Keep server-side only. |

If you add Google, Discord, or another provider, treat them the same way: provider credentials live wherever your Convex Better Auth config reads them.

## Optional Performance / Shared Infra

Only set these when you opt into the related feature.

| Variable | Required? | Who reads it | Where to set it | Example | Notes |
| -------- | --------- | ------------ | --------------- | ------- | ----- |
| `REDIS_URL` | Only if you configure Nitro shared auth cache | Nuxt Nitro storage config | Nuxt host env | `redis://default:password@host:6379` | Used when `convex.authCache.enabled` is on and you want shared cache across instances. |
| `JWKS` | Only if you opt into static JWKS | Convex auth config / Better Auth plugin | Convex Dashboard env vars | JSON blob generated from `auth:getLatestJwks` | Optional optimization for high-traffic auth workloads. |

## Local Development Layout

For the official repo examples and most local apps:

- keep Nuxt-facing vars in `.env.local`
- run Nuxt with `--dotenv .env.local`
- set Convex auth vars in Convex Dashboard or via `npx convex env set ... --env-file .env.local` during local work
- if you opt into `USE_LOCAL_CONVEX=true`, keep hosted `CONVEX_URL` values in `.env.local` and let
  your Nuxt config replace them with fixed localhost URLs at runtime
- local mode still needs `SITE_URL`, `AUTH_BASE_URL`, `AUTH_TRUSTED_ORIGINS`, and
  `BETTER_AUTH_SECRET` on the backend side
- if you add an app-local private bridge, `CONVEX_PRIVATE_BRIDGE_KEY` is app-local, server-only,
  and should never live in `runtimeConfig.public`

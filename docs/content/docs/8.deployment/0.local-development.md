Use hosted Convex as the default mode and opt into local Convex explicitly.

## Goals

- Keep `.env.local` stable for hosted development.
- Switch local backend mode on with one env flag.
- Use fixed localhost URLs so `convex.url` and `convex.siteUrl` are predictable.
- Preserve the same Better Auth proxy flow in both modes.

## Recommended Setup

Install the optional local backend plugin:

```bash
pnpm add -D convex-vite-plugin
```

In `nuxt.config.ts`, derive Convex URLs from `USE_LOCAL_CONVEX` and start the local backend only in that mode.
`convex-vite-plugin` stays a dev-only tool for local DX. It is not part of the published
`better-convex-nuxt` runtime.

```ts [nuxt.config.ts]
import { convexLocal } from 'convex-vite-plugin'

const useLocalConvex = process.env.USE_LOCAL_CONVEX === 'true'
const localConvexUrl = 'http://127.0.0.1:3210'
const localConvexSiteUrl = 'http://127.0.0.1:3211'
const appUrl = process.env.SITE_URL || 'http://localhost:3000'

export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],
  convex: {
    url: useLocalConvex ? localConvexUrl : process.env.CONVEX_URL,
    siteUrl: useLocalConvex ? localConvexSiteUrl : process.env.CONVEX_SITE_URL,
  },
  hooks: {
    'vite:extendConfig': (config, { isClient }) => {
      if (!useLocalConvex || !isClient) return
      config.plugins = [
        ...(config.plugins ?? []),
        convexLocal({
          port: 3210,
          siteProxyPort: 3211,
          projectDir: '.',
          convexDir: 'convex',
          envVars: {
            SITE_URL: appUrl,
            AUTH_BASE_URL: process.env.AUTH_BASE_URL || appUrl,
            AUTH_TRUSTED_ORIGINS: process.env.AUTH_TRUSTED_ORIGINS || appUrl,
            BETTER_AUTH_SECRET:
              process.env.BETTER_AUTH_SECRET || 'local-dev-better-auth-secret-not-for-production',
          },
        }),
      ]
    },
  },
})
```

Register `convexLocal()` only on the client Vite config. In Nuxt, attaching it to both client and
server configs can try to start the same local backend twice.

## Mode Switch

- Hosted mode: `pnpm dev`
- Local mode: `USE_LOCAL_CONVEX=true pnpm dev`
- Reset local backend state: `USE_LOCAL_CONVEX=true RESET_LOCAL_BACKEND=true pnpm dev`

For the repo playground specifically, `pnpm dev:local` also enables the privileged reference-lane
demo with a playground-only bridge key. Plain `pnpm dev` leaves that backend-only example disabled
until you opt in deliberately.

## Required Env Contract

Hosted mode still reads the normal Nuxt-side values:

- `CONVEX_URL`
- `CONVEX_SITE_URL` when auto-derivation is not enough

Local mode should still provide or derive the auth-facing values:

- `SITE_URL`
- `AUTH_BASE_URL`
- `AUTH_TRUSTED_ORIGINS`
- `BETTER_AUTH_SECRET`

`CONVEX_URL` and `CONVEX_SITE_URL` are replaced at runtime by the fixed localhost values.

## Auth Token Exchange In Local Mode

The Nuxt auth proxy and `serverConvexQuery` / `serverConvexMutation` / `serverConvexAction`
always use `convex.siteUrl` for Better Auth token exchange.

That means:

- hosted mode exchanges tokens against your hosted `.convex.site` URL
- local mode exchanges tokens against the local HTTP Actions proxy, for example `http://127.0.0.1:3211`

The flow stays the same. Only the target host changes.

## Diagnostics

If local auth is broken, the most common causes are:

- `convex.siteUrl` is missing or points at the wrong host
- Better Auth routes are not registered in `convex/http.ts`
- `BETTER_AUTH_SECRET` is missing for the local backend
- your request has no Better Auth session cookie

The server helper errors now include helper name, function path, auth mode, and target Convex URL
to make those failures easier to identify.

Reference for `trellis.permissions`. One key points `usePermissions()` and `useAuthGuard()` at your backend-owned permission-context query.

## Two shapes

```ts [nuxt.config.ts]
trellis: {
  permissions: 'workspaces.getPermissionContext',
}
```

```ts [nuxt.config.ts]
trellis: {
  permissions: {
    query: 'workspaces.getPermissionContext',
  },
}
```

Both work. Use the shorthand for the common case; use the object when you need room to grow.

## What it wires

| Composable | Reads from |
| --- | --- |
| `usePermissions()` | The configured query's result |
| `useAuthGuard({ can })` | The `can` map in that result |

## Important constraint

::warning
The string must point at a real Convex query export path. If it doesn't, Trellis raises a *configured-function error* at setup — it won't silently do nothing. Fix the path; the error tells you exactly which export it couldn't find.
::

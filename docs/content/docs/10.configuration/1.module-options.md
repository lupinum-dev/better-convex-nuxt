Use this page to look up the top-level `trellis` config keys and their expected shapes.

## Root Shape

```ts
trellis: {
  url,
  auth,
  permissions,
  trustedCallers,
  mcp,
  query,
  upload,
  observability,
  validation,
}
```

Common top-level shape:

```ts
export default defineNuxtConfig({
  trellis: {
    url: process.env.CONVEX_URL,
    auth: false,
    permissions: { query: api.permissions.get },
    mcp: { enabled: true },
  },
})
```

## Options

- `url`
  Convex deployment URL. Usually sourced from `CONVEX_URL`.
- `auth`
  `boolean` or auth options object. `false` disables auth; an object enables and configures auth behavior.
- `permissions`
  Permission-query shorthand or object config for frontend permission state.
- `trustedCallers`
  Enables trusted caller infrastructure for server-to-server flows.
- `mcp`
  MCP runtime options.
- `query`
  Default query behavior such as SSR and subscriptions.
- `upload`
  Upload defaults such as `maxConcurrent`.
- `observability`
  Semantic observability config.
- `validation`
  Startup/build validation strictness.

## Shape Notes

`auth` supports two common shapes:

```ts
auth: false
```

```ts
auth: {
  enabled: true,
  routeProtection: {
    redirectTo: '/auth/signin',
  },
}
```

`permissions` supports both a compact app-facing shorthand and a fuller object:

```ts
permissions: api.permissions.get
```

```ts
permissions: {
  query: api.permissions.get,
}
```

## Query Defaults

`query.server` defaults to `true`.
`query.subscribe` defaults to `true`.

These set the default behavior for `useConvexQuery()` and `useConvexPaginatedQuery()`, but per-query options can still override them.

## Upload Defaults

`upload.maxConcurrent` controls the default queue concurrency for `useConvexUpload()`.

## Validation

`validation.strict` promotes module validation warnings into startup or build errors.

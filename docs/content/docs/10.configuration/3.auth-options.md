Reference for `trellis.auth`. Two common shapes: a boolean for enable/disable, or an object when you need to configure specific behavior.

```ts [nuxt.config.ts]
trellis: {
  auth: {
    enabled: true,
    route: '/api/auth',
    trustedOrigins: [],
    skipAuthRoutes: [],
    cache: { enabled: false, ttl: 60 },
    proxy: {
      maxRequestBodyBytes: 1_048_576,
      maxResponseBodyBytes: 1_048_576,
    },
  },
}
```

## Shorthand

| Value | Effect |
| --- | --- |
| `auth: true` | Enable with defaults |
| `auth: false` | Disable auth entirely |

## Option reference

| Option | Default | Purpose |
| --- | --- | --- |
| `enabled` | `true` | Master switch |
| `route` | `/api/auth` | Better Auth proxy route |
| `trustedOrigins` | `[]` | Extra origins allowed through proxy CORS validation |
| `skipAuthRoutes` | `[]` | Routes that skip auth token fetches |
| `cache.enabled` | `false` | SSR auth token cache on/off |
| `cache.ttl` | `60` | Cache TTL (seconds, clamped 1â€“60) |
| `proxy.maxRequestBodyBytes` | `1_048_576` | Request body size limit |
| `proxy.maxResponseBodyBytes` | `1_048_576` | Response body size limit |

## Important distinctions

::caution
`skipAuthRoutes` skips the *auth token fetch* on specific routes. It isn't a replacement for `definePageMeta({ skipConvexAuth: true })`, which skips auth checks at the page level. They solve different problems at different layers.
::

::note
At the module level, Trellis defaults to the auth-enabled path. Set `auth: false` when you want a truly public-only baseline.
::

::note
`cache` is opt-in. It trades a small correctness window (up to `ttl` seconds of stale permission state) for fewer token-exchange round-trips per SSR request. Right for traffic-heavy deployments, wrong for local dev where you want to see auth changes immediately.
::

`cache.ttl` is normalized into the safe 1â€“60 second range â€” passing `3600` clamps to `60`.

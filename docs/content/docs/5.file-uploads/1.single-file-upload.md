Use this page when a component needs to upload one file and react to the result directly.

The main composable is:

```ts
const {
  upload,
  pending,
  progress,
  data: storageId,
  error,
} = useConvexUpload(api.files.generateUploadUrl, {
  allowedTypes: ['image/*', 'text/*', 'application/pdf'],
  maxSizeBytes: 5_000_000,
})
```

## What The Upload Flow Does

`useConvexUpload(...)` handles the normal client-side path:

1. call your Convex mutation to get an upload URL
2. upload the file through XHR
3. expose progress and status
4. return the resulting `storageId`

That means your mutation must return a string upload URL. If it does not, the upload fails immediately.

## Minimal Browser Flow

```ts
async function handleFile(event: Event) {
  const input = event.target as HTMLInputElement
  const file = input.files?.[0]
  if (!file) return

  await upload(file)
}
```

Use the reactive state directly in the template:

- `pending`
- `progress`
- `data`
- `error`

## Client-Side Validation

The composable can reject files before upload starts:

- `allowedTypes`
- `maxSizeBytes`

Those checks are useful UX, but they are not your full security model. Server-side file handling still needs its own validation rules.

## Cancellation And Reset

Single-file mode supports aborting the in-flight upload through `reset()`.

That clears:

- current status
- current error
- current progress
- current `storageId`

If your component can be destroyed mid-upload, Trellis also aborts the in-flight upload on scope dispose.

## Common Mistakes

- Returning something other than a string upload URL from the mutation
- Treating client-side type checks as the whole security boundary
- Forgetting to persist the returned `storageId` into your document model

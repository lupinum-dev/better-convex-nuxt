description: "Upload files to Convex with progress tracking and validation."

## Overview

`useConvexUpload` handles file uploads to Convex storage with built-in progress tracking, file validation, and automatic cleanup.

## Basic Usage

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const upload = useConvexUpload(api.files.generateUploadUrl)

async function handleFileChange(event: Event) {
  const file = (event.target as HTMLInputElement).files?.[0]
  if (!file) return

  const storageId = await upload(file)
  console.log('Uploaded:', storageId)
}
</script>

<template>
  <div>
    <input type="file" @change="handleFileChange" :disabled="upload.pending.value" />
    <div v-if="upload.pending.value">
      Uploading: {{ upload.progress.value }}%
    </div>
    <div v-if="upload.status.value === 'error'">
      Error: {{ upload.error.value?.message }}
    </div>
  </div>
</template>
```

::note
You need a Convex mutation that generates upload URLs. See the [Convex docs on file storage](https://docs.convex.dev/file-storage/upload-files) for the backend setup.
::

## Return Value

The composable returns a callable function with reactive state properties:

::field-group
::field{name="upload(file, mutationArgs?)" type="(file: File) => Promise<string>"}
Upload a single file. Returns the `storageId`.
::
::field{name="data" type="Ref<string | undefined>"}
The `storageId` from the last successful upload.
::
::field{name="status" type="ComputedRef<UploadStatus>"}
`'idle'` | `'pending'` | `'success'` | `'error'`
::
::field{name="pending" type="ComputedRef<boolean>"}
`true` while an upload is in progress.
::
::field{name="progress" type="ComputedRef<number>"}
Upload progress from 0 to 100.
::
::field{name="error" type="Ref<Error | null>"}
Error from the last upload attempt.
::
::field{name="reset()" type="() => void"}
Reset state to idle. Aborts any in-progress upload.
::
::

## Options

::field-group
::field{name="allowedTypes" type="string[]"}
Allowed MIME types. Supports wildcards like `image/*`. Files not matching are rejected before upload.
::
::field{name="maxSizeBytes" type="number"}
Maximum file size in bytes. Files exceeding this limit are rejected before upload.
::
::field{name="onSuccess" type="(storageId: string, file: File) => void"}
Called when a file uploads successfully.
::
::field{name="onError" type="(error: Error, file: File) => void"}
Called when an upload fails.
::
::field{name="onProgress" type="(info: UploadProgressInfo, file: File) => void"}
Called for progress updates during upload. `info` contains `percent`, `loaded`, and `total`.
::
::

## Validation Example

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const upload = useConvexUpload(api.files.generateUploadUrl, {
  allowedTypes: ['image/*', 'application/pdf'],
  maxSizeBytes: 10 * 1024 * 1024, // 10 MB
  onSuccess: (storageId) => {
    console.log('Upload complete:', storageId)
  },
  onError: (error) => {
    console.error('Upload failed:', error.message)
  },
})
</script>
```

::tip
Validation errors are thrown immediately — the file is never sent to the server if it fails type or size checks.
::

## Cleanup

The upload is automatically aborted when the component unmounts (via `onScopeDispose`). You can also call `reset()` to manually abort and clear state.

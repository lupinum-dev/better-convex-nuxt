title: "Upload Queue"
description: "Upload multiple files with concurrency control and per-item status tracking."
---

## Overview

When you pass an array of files to `useConvexUpload`, it automatically switches to queue mode with concurrent uploads, per-item progress tracking, and cancellation support.

## Basic Usage

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const upload = useConvexUpload(api.files.generateUploadUrl, {
  maxConcurrent: 3,
  onQueueIdle: () => console.log('All uploads finished'),
})

async function handleFiles(event: Event) {
  const files = Array.from((event.target as HTMLInputElement).files ?? [])
  if (!files.length) return

  const storageIds = await upload(files)
  console.log('All uploaded:', storageIds)
}
</script>

<template>
  <div>
    <input type="file" multiple @change="handleFiles" />

    <div v-for="item in upload.items.value" :key="item.id">
      <span>{{ item.file.name }}</span>
      <span>{{ item.status }}</span>
      <progress v-if="item.status === 'pending'" :value="item.progress" max="100" />
      <button v-if="item.status === 'queued' || item.status === 'pending'" @click="upload.cancelItem(item.id)">
        Cancel
      </button>
      <span v-if="item.error">{{ item.error.message }}</span>
    </div>

    <div v-if="upload.pending.value">
      Overall: {{ upload.progress.value }}%
    </div>

    <button v-if="upload.items.value.length" @click="upload.clearFinished()">
      Clear finished
    </button>
  </div>
</template>
```

## Queue Item Properties

Each item in the `items` array has:

::field-group
::field{name="id" type="string"}
Unique identifier for the queue item.
::
::field{name="file" type="File"}
The file being uploaded.
::
::field{name="status" type="UploadQueueItemStatus"}
`'queued'` | `'pending'` | `'success'` | `'error'` | `'cancelled'`
::
::field{name="progress" type="number"}
Upload progress from 0 to 100.
::
::field{name="loadedBytes" type="number"}
Bytes uploaded so far.
::
::field{name="totalBytes" type="number"}
Total file size in bytes.
::
::field{name="storageId" type="string | undefined"}
The `storageId` after successful upload.
::
::field{name="error" type="Error | null"}
Error if upload failed.
::
::

## Queue Management

::field-group
::field{name="cancelItem(id)" type="(id: string) => void"}
Cancel a specific upload. Aborts in-progress uploads or removes queued items.
::
::field{name="cancelAll()" type="() => void"}
Cancel all queued items and abort in-progress uploads.
::
::field{name="clearFinished()" type="() => void"}
Remove completed, errored, and cancelled items from the list.
::
::field{name="reset()" type="() => void"}
Cancel everything and clear all items.
::
::

## Options

::field-group
::field{name="maxConcurrent" type="number"}
Maximum number of concurrent uploads. Default: `3`.
::
::field{name="continueOnError" type="boolean"}
Continue processing the queue after an item fails. Default: `true`. When `false`, the queue halts on the first error.
::
::field{name="onSuccess" type="(storageId: string, file: File) => void"}
Called when each individual file uploads successfully.
::
::field{name="onError" type="(error: Error, file: File) => void"}
Called when an individual upload fails.
::
::field{name="onQueueIdle" type="() => void"}
Called when all queued uploads finish (success, error, or cancelled).
::
::

## Awaitable Enqueue

The `upload(files)` call returns a promise that resolves with all storage IDs when every file finishes:

```ts
try {
  const storageIds = await upload([file1, file2, file3])
  // All succeeded
  console.log(storageIds) // ['storageId1', 'storageId2', 'storageId3']
} catch (err) {
  // At least one failed (AggregateError if multiple failures)
}
```

## Aggregate Progress

The `progress` computed ref shows overall progress across all items (0-100), weighted by file size.

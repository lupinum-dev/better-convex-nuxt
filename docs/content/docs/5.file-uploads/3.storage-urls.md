---
title: "Storage URLs"
description: "Generate URLs for files stored in Convex storage."
---

## Overview

`useConvexStorageUrl` generates a direct URL for a file in Convex storage. It automatically skips the query when no `storageId` is provided.

## Basic Usage

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const props = defineProps<{ storageId: string | null }>()

const imageUrl = useConvexStorageUrl(api.files.getUrl, () => props.storageId)
</script>

<template>
  <img v-if="imageUrl" :src="imageUrl" alt="Uploaded image" />
  <p v-else>No image</p>
</template>
```

## Parameters

::field-group
::field{name="query" type="FunctionReference<'query'>" required}
A Convex query that takes a `storageId` and returns a URL string.
::
::field{name="storageId" type="MaybeRef<string | null | undefined>" required}
The storage ID to look up. When `null` or `undefined`, the query is skipped automatically.
::
::

## Return Value

Returns a `ComputedRef<string | null>` — the URL when available, `null` when loading or skipped.

## Auto-Skip Behavior

When `storageId` is `null` or `undefined`, no query is sent to Convex. This is useful when the storage ID comes from another query that may not have loaded yet:

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const { data: profile } = await useConvexQuery(api.users.current, {})

// Automatically skips until profile loads and has an avatarId
const avatarUrl = useConvexStorageUrl(
  api.files.getUrl,
  () => profile.value?.avatarId
)
</script>
```

## Upload + Display Pattern

A common pattern is uploading a file and immediately displaying it:

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const upload = useConvexUpload(api.files.generateUploadUrl)
const imageUrl = useConvexStorageUrl(api.files.getUrl, () => upload.data.value)

async function handleFile(event: Event) {
  const file = (event.target as HTMLInputElement).files?.[0]
  if (file) await upload(file)
}
</script>

<template>
  <input type="file" accept="image/*" @change="handleFile" />
  <img v-if="imageUrl" :src="imageUrl" alt="Preview" />
</template>
```

## Convex Backend

Your Convex query needs to call `ctx.storage.getUrl()`:

```ts [convex/files.ts]
import { query } from './_generated/server'
import { v } from 'convex/values'

export const getUrl = query({
  args: { storageId: v.id('_storage') },
  handler: async (ctx, args) => {
    return await ctx.storage.getUrl(args.storageId)
  },
})
```

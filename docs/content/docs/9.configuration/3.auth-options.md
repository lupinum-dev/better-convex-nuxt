Use this page to look up `trellis.auth`.

## Shorthand

- `auth: true` enables auth with defaults
- `auth: false` disables auth

## Object Form

```ts
auth: {
  enabled: true,
  route: '/api/auth',
  trustedOrigins: [],
  skipAuthRoutes: [],
  cache: {
    enabled: true,
    ttl: 60,
  },
  proxy: {
    maxRequestBodyBytes: 1_048_576,
    maxResponseBodyBytes: 1_048_576,
  },
}
```

## Important Options

- `route`: Better Auth proxy route. Default `/api/auth`.
- `trustedOrigins`: extra trusted origins for proxy CORS validation.
- `skipAuthRoutes`: routes that should skip auth token fetches.
- `cache`: SSR auth token cache.
- `proxy`: request and response size limits for the auth proxy.

## Notes

- `skipAuthRoutes` is route-level skipping for auth token fetches, not a replacement for page metadata like `skipConvexAuth: true`.
- auth cache is opt-in and defaults to disabled.
- `cache.ttl` is normalized into a safe 1..60 second range.

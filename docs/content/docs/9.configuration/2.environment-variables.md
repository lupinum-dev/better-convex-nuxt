## Quick Setup

For most projects, you only need these:

```bash [.env.local]
# Created automatically by `npx convex dev`
CONVEX_URL=https://your-deployment-123.convex.cloud
```

If using auth, also set these in your [Convex Dashboard](https://dashboard.convex.dev) under Settings → Environment Variables:

```bash
BETTER_AUTH_SECRET=<run: openssl rand -hex 32>
SITE_URL=http://localhost:3000
```

That's it for local development. The sections below cover all available variables in detail.

## Nuxt Side

Set these in your `.env.local` file (loaded with `nuxt dev --dotenv .env.local`):

| Variable | Required | Description |
|----------|----------|-------------|
| `CONVEX_URL` | Yes | Convex deployment WebSocket URL (e.g., `https://your-app.convex.cloud`) |
| `NUXT_PUBLIC_CONVEX_URL` | Alt | Alternative to `CONVEX_URL` — uses Nuxt's public runtime config convention |
| `CONVEX_SITE_URL` | No | Convex HTTP Actions URL. Auto-derived from `CONVEX_URL` (`.cloud` → `.site`) |
| `NUXT_PUBLIC_CONVEX_SITE_URL` | No | Alternative to `CONVEX_SITE_URL` |
| `PRIVATE_BRIDGE_KEY` | No | Shared secret for privileged server calls |

::note
You only need one of `CONVEX_URL` or `NUXT_PUBLIC_CONVEX_URL`. The module checks both.
::

## Convex Dashboard

Set these in your [Convex Dashboard](https://dashboard.convex.dev) under Settings → Environment Variables:

| Variable | Required | Description |
|----------|----------|-------------|
| `BETTER_AUTH_SECRET` | Yes (auth) | Symmetric key for Better Auth token signing |
| `SITE_URL` | Yes (auth) | Your app's origin (e.g., `http://localhost:3000` or `https://myapp.com`) |
| `GITHUB_CLIENT_ID` | No | GitHub OAuth client ID |
| `GITHUB_CLIENT_SECRET` | No | GitHub OAuth client secret |
| `GOOGLE_CLIENT_ID` | No | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | No | Google OAuth client secret |

::tip
Add OAuth provider credentials only for the providers you've configured in your `convex/auth.ts`.
::

## Example `.env.local`

```bash [.env.local]
CONVEX_URL=https://happy-animal-123.convex.cloud
```

## Auto-Derived Site URL

The module automatically derives the Convex HTTP Actions URL from the WebSocket URL:

```
https://happy-animal-123.convex.cloud  →  https://happy-animal-123.convex.site
```

Override this by setting `CONVEX_SITE_URL` or `siteUrl` in your module config if you use a custom domain.

## Production

In production, set `CONVEX_URL` (or `NUXT_PUBLIC_CONVEX_URL`) as an environment variable in your hosting provider (Vercel, Netlify, etc.) rather than using `.env.local`.

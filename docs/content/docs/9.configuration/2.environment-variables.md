Use this page when Trellis is wired into a Nuxt app and you need to know which environment variables matter, which ones are optional, and which ones are launcher-owned versus app-owned.

The important distinction is this:

- some values are part of normal Trellis runtime setup
- some values are auth-specific
- some values only matter for trusted callers, MCP, or local development

## Core Convex Variables

Trellis needs a Convex deployment URL through either module config or environment:

- `CONVEX_URL`
- `NUXT_PUBLIC_CONVEX_URL`

The normal baseline is:

```ts
trellis: {
  url: process.env.CONVEX_URL,
}
```

If neither `convex.url` nor one of those environment variables is set, Trellis cannot talk to Convex.

## Convex Site URL Variables

These are the HTTP Actions host variables:

- `CONVEX_SITE_URL`
- `NUXT_PUBLIC_CONVEX_SITE_URL`

You usually do not need to set them when:

- `CONVEX_URL` points at a standard `.convex.cloud` deployment
- you are using the normal local Convex ports `127.0.0.1:3210` and `127.0.0.1:3211`

Trellis can derive the site URL for those cases.

Set them explicitly when:

- you use a custom HTTP Actions domain
- auth token exchange cannot be derived reliably from the deployment URL

## Auth Variables

These matter when `trellis.auth` is enabled:

- `SITE_URL`
- `BETTER_AUTH_SECRET`

`SITE_URL` should match the app origin that Better Auth expects for callbacks and trusted-origin checks.

`BETTER_AUTH_SECRET` must stay aligned between the local environment and the Convex Better Auth deployment.

If either one is missing, `npx trellis doctor` reports it.

## Trusted Caller Variables

These matter when the app uses trusted callers, webhook verification, or authenticated MCP flows:

- `CONVEX_TRUSTED_CALLER_KEY`

For the internal private bridge lane in local development, you may also need:

- `CONVEX_PRIVATE_BRIDGE_KEY`

Do not set these unless the app actually uses the corresponding surfaces.

## Local Development Defaults

Typical local values in this repo are:

- `CONVEX_URL=http://127.0.0.1:3210`
- `CONVEX_SITE_URL=http://127.0.0.1:3211`
- `SITE_URL=http://localhost:3000`

Examples often inject `CONVEX_URL` and `CONVEX_SITE_URL` for you during `pnpm dev`. In those cases, treat those as launcher-owned values rather than hand-edited app config.

## Launcher-Owned Versus App-Owned Values

Inside the repo examples, the local launcher owns:

- `CONVEX_URL`
- `CONVEX_SITE_URL`

Common app-owned values are:

- `SITE_URL`
- `BETTER_AUTH_SECRET`
- provider credentials such as `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET`
- trusted-caller keys

That distinction matters because overwriting launcher-owned values manually usually creates confusing local mismatches.

## Fast Verification

Use the doctor command to verify env expectations:

```bash
npx trellis doctor
```

The doctor checks:

- Convex URL source
- `SITE_URL`
- `CONVEX_SITE_URL`
- `BETTER_AUTH_SECRET`
- Better Auth route registration
- `CONVEX_TRUSTED_CALLER_KEY` when trusted-caller surfaces are detected

## Common Mistakes

- Setting auth env vars even though `auth: false` is the intended baseline.
- Forgetting that `BETTER_AUTH_SECRET` must match both locally and in the Convex deployment.
- Setting `CONVEX_SITE_URL` unnecessarily when Trellis can derive it.
- Editing launcher-owned local Convex URLs in example apps that already inject them automatically.

Use this page to look up the top-level `trellis` config keys.

## Root Shape

```ts
trellis: {
  url,
  auth,
  permissions,
  trustedCallers,
  mcp,
  query,
  upload,
  observability,
  validation,
}
```

## Options

- `url`: Convex deployment URL.
- `auth`: `boolean` or auth options object.
- `permissions`: string shorthand or `{ query }` object for configured permissions.
- `trustedCallers`: enable trusted caller infrastructure.
- `mcp`: MCP runtime options.
- `query`: default query behavior like SSR and subscriptions.
- `upload`: upload defaults such as `maxConcurrent`.
- `observability`: semantic observability config.
- `validation`: startup/build validation strictness.

## Query Defaults

`query.server` defaults to `true`.
`query.subscribe` defaults to `true`.

These set the default behavior for `useConvexQuery()` and `useConvexPaginatedQuery()`, but per-query options can still override them.

## Upload Defaults

`upload.maxConcurrent` controls the default queue concurrency for `useConvexUpload()`.

## Validation

`validation.strict` promotes module validation warnings into startup or build errors.

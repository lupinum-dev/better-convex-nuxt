## Overview

Configure better-convex-nuxt via the `convex` key in your `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],
  convex: {
    // options
  }
})
```

## Core Options

::field-group
::field{name="url" type="string"}
Convex deployment WebSocket URL (e.g., `https://your-app.convex.cloud`). Auto-detected from `CONVEX_URL` or `NUXT_PUBLIC_CONVEX_URL` environment variables.
::
::field{name="siteUrl" type="string"}
Convex HTTP Actions URL. Auto-derived from `url` by replacing `.convex.cloud` with `.convex.site`. Override if using a custom domain.
::
::

## Authentication

::field-group
::field{name="auth" type="boolean | AuthOptions"}
Enable authentication. Default: `true`. Pass `true` for defaults or an object for detailed configuration.
::
::field{name="auth.route" type="string"}
Auth proxy route path. Default: `'/api/auth'`.
::
::field{name="auth.trustedOrigins" type="string[]"}
CORS trusted origins for the auth proxy. Useful for preview deployments (e.g., `['https://*.vercel.app']`).
::
::field{name="auth.skipAuthRoutes" type="string[]"}
Glob patterns for routes that skip auth token fetch.
::
::

### Route Protection

::field-group
::field{name="auth.routeProtection.redirectTo" type="string"}
Default redirect path for protected pages. Default: `'/auth/signin'`.
::
::field{name="auth.routeProtection.preserveReturnTo" type="boolean"}
Add `?returnTo=/original-path` to the redirect URL. Default: `true`.
::
::

### Unauthorized Handling

::field-group
::field{name="auth.unauthorized.enabled" type="boolean"}
Automatically handle 401/403 errors from Convex.
::
::field{name="auth.unauthorized.redirectTo" type="string"}
Redirect path when an unauthorized error is detected.
::
::field{name="auth.unauthorized.includeQueries" type="boolean"}
Also handle auth errors from queries (not just mutations/actions).
::
::

### Token Caching

::field-group
::field{name="auth.cache.enabled" type="boolean"}
Enable server-side JWT token caching. Default: `false`.
::
::field{name="auth.cache.ttl" type="number"}
Cache time-to-live in seconds (1–60). Default: `60`.
::
::

### Proxy Limits

::field-group
::field{name="auth.proxy.maxRequestBodyBytes" type="number"}
Maximum request body size for auth proxy. Default: `1048576` (1 MB).
::
::field{name="auth.proxy.maxResponseBodyBytes" type="number"}
Maximum response body size from auth proxy. Default: `1048576` (1 MB).
::
::

## Query Defaults

::field-group
::field{name="query.server" type="boolean"}
Run queries server-side during SSR. Default: `true`.
::
::field{name="query.subscribe" type="boolean"}
Keep WebSocket subscriptions active. Default: `true`.
::
::

## Upload Defaults

::field-group
::field{name="upload.maxConcurrent" type="number"}
Maximum concurrent uploads in queue mode. Default: `3`.
::
::

## Permissions

::field-group
::field{name="permissions" type="boolean"}
Enable the `createPermissions` composable. Default: `false`.
::
::

## Logging

::field-group
::field{name="logging" type="false | 'info' | 'debug'"}
Module logging level. `false` disables all logs. `'info'` logs auth, mutations, actions, uploads, and connection events. `'debug'` adds query updates and timing.
::
::

## Debug Channels

::field-group
::field{name="debug.authFlow" type="boolean"}
Enable general auth flow tracing.
::
::field{name="debug.clientAuthFlow" type="boolean"}
Enable client-side auth flow tracing.
::
::field{name="debug.serverAuthFlow" type="boolean"}
Enable server-side auth flow tracing.
::
::

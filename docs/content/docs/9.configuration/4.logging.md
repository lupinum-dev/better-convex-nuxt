## Overview

Trellis no longer exposes `trellis.logging`.
The old debug/runtime logger surface was removed in favor of one coherent observability model.

`trellis.observability` now drives:

- semantic Trellis events
- `evlog` delivery and pretty output
- request/tool/invocation wide summaries
- production-critical event visibility

See the [Observability](/9.configuration/5.observability) page for the semantic event model, delivery behavior, and configuration surface.

## Configure Observability

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  trellis: {
    observability: {
      service: 'kanban-workspace',
      level: 'verbose',
    },
  },
})
```

Use `level: 'verbose'` when you want the highest event volume in development.
Use the production default `level: 'critical'` when you only want denials, destructive flows, trust-boundary usage, and failures.

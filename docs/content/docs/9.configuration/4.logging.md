## Overview

Trellis uses structured logging via [consola](https://github.com/unjs/consola) to help you debug queries, mutations, auth flows, and connection state.

This page is about debug/runtime logging only.
It is not the same thing as Trellis observability, which ships semantic events, correlation, redaction, and adapter delivery under `trellis.observability`.

## Log Levels

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  trellis: {
    logging: false, // No logs (default in production)
    // logging: 'info'  // Auth, mutations, actions, uploads, connection
    // logging: 'debug' // All info + query updates, timing, debug details
  },
})
```

| Level     | What's logged                                                           |
| --------- | ----------------------------------------------------------------------- |
| `false`   | Nothing                                                                 |
| `'info'`  | Auth events, mutation/action calls, upload progress, connection changes |
| `'debug'` | Everything in `info` + query subscribe/update/unsubscribe, timing data  |

## Observability vs Logging

Use `logging` when you want sink-level runtime/debug output.

Use `observability` when you want semantic Trellis events such as:

- principal and actor resolution
- guard and authorize outcomes
- `ctx.db.crossTenant` and `ctx.db.raw` usage
- service access checks and denials
- operation preview, drift, and execution
- MCP tool denial, confirmation, and execution

The built-in shipped adapter is `observability.adapter = 'dev'`.

## Auth Flow Tracing

Use `logging: 'debug'` when you need the highest verbosity for auth, query, and connection
tracing. The removed `convex.debug.*` channels are no longer part of the module config.

## Example Output

With `logging: 'info'`:

```
[convex] auth    → token refreshed (user: john@example.com)
[convex] mutation → posts:create success (42ms)
[convex] upload  → avatar.png success (1.2MB, 850ms)
```

With `logging: 'debug'`:

```
[convex] query   → tasks:list subscribe (args: {status: "active"})
[convex] query   → tasks:list update (3 items)
[convex] query   → tasks:list unsubscribe
```

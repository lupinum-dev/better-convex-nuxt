## Overview

Better Convex Nuxt uses structured logging via [consola](https://github.com/unjs/consola) to help you debug queries, mutations, auth flows, and connection state.

## Log Levels

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    logging: false    // No logs (default in production)
    // logging: 'info'  // Auth, mutations, actions, uploads, connection
    // logging: 'debug' // All info + query updates, timing, debug details
  }
})
```

| Level | What's logged |
|-------|---------------|
| `false` | Nothing |
| `'info'` | Auth events, mutation/action calls, upload progress, connection changes |
| `'debug'` | Everything in `info` + query subscribe/update/unsubscribe, timing data |

## Auth Debug Channels

For detailed auth flow tracing, use the `debug` channels:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    debug: {
      authFlow: true,        // General auth flow events
      clientAuthFlow: true,  // Client-side token fetch, refresh, hydration
      serverAuthFlow: true   // Server-side cookie exchange, token caching
    }
  }
})
```

These channels produce detailed step-by-step output showing exactly how auth tokens are resolved, cached, and refreshed.

## Example Output

With `logging: 'info'`:

```
[convex] auth    → token refreshed (user: john@example.com)
[convex] mutation → posts:create success (42ms)
[convex] upload  → avatar.png success (1.2MB, 850ms)
```

With `logging: 'debug'`:

```
[convex] query   → tasks:list subscribe (args: {status: "active"})
[convex] query   → tasks:list update (3 items)
[convex] query   → tasks:list unsubscribe
```

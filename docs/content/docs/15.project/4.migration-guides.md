Where ordered upgrade walkthroughs will live once a release needs more than a changelog entry can carry.

Right now Trellis has no standalone migration guides. The changelog carries the upgrade notes, and this page stays hidden until a release genuinely needs a step-by-step cutover.

## When a migration guide earns its own page

Write one when readers need all of these, not just some:

- An ordered upgrade path.
- Before-and-after examples.
- Explicit failure modes with names.
- A clear "you are done" endpoint.

If the change fits in a changelog bullet, keep it in the changelog.

## The shape every migration guide follows

| Section | Answers |
| --- | --- |
| Who needs to care | Which apps, versions, or configs are affected |
| What changed | One-sentence description plus link to the PR or RFC |
| What to do, in order | Numbered steps — stop-and-verify points called out |
| Before and after | Real code, not pseudo-code |
| Common breakages | What will fail and what the error looks like |
| Deeper reference | Links to related guide or API pages |

::note
If this page stays empty for long enough, it should be removed instead of lingering as public scaffolding.
::

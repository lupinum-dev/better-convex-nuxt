Use this page as the home for real migration guides when Trellis needs them.

Right now, the changelog carries most upgrade guidance. That is acceptable until one release introduces a breaking or high-friction migration that deserves its own ordered document.

## When To Write A Migration Guide

Write one when readers need:

- an ordered upgrade path
- before-and-after examples
- explicit failure modes
- a clear “you are done” endpoint

## Default Shape

1. who needs to care
2. what changed
3. what to do in order
4. before and after examples
5. common breakages
6. links to deeper reference

## Current State

No standalone migration guide is mandatory yet. This page exists so Trellis has a clean home ready when that boundary arrives.

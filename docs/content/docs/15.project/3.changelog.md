Find out what changed between versions and what you need to do on upgrade. The canonical record is the repo-root [CHANGELOG.md](https://github.com/lupinum-dev/trellis/blob/main/CHANGELOG.md) — this page is the policy around it.

## What belongs in an entry

| Keep | Skip |
| --- | --- |
| User-facing behavior changes | Raw commit dumps |
| Upgrade notes and breaking changes | Internal-only refactors with no user impact |
| Support-boundary shifts | Dependency bumps that don't change public shape |

The goal is something a reader can scan in 30 seconds and know whether an upgrade will cost them an hour or a day.

## Current state

The existing changelog already carries the important support-boundary notes, but older entries mix release styles. New entries should stay in one format with scannable upgrade notes — a one-line summary, then the "what you need to do" bullets.

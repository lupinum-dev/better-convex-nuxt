`useConvexQuery` connects a Convex query to Nuxt SSR and Vue reactivity.

```ts
import { api } from '#convex/api'

const { data, status, pending, error, isStale, refresh, clear } = await useConvexQuery(
  api.projects.list,
  { archived: false },
)
```

## Returned state

| Field       | Meaning                                                      |
| ----------- | ------------------------------------------------------------ |
| `data`      | Transformed query result or `null`                           |
| `status`    | `idle`, `pending`, `success`, or `error`                     |
| `pending`   | `true` while the current result is unresolved                |
| `error`     | Normalized `ConvexCallError` or `null`                       |
| `isStale`   | Previous arguments' data is visible while new arguments load |
| `refresh()` | Re-execute the current query                                 |
| `clear()`   | Clear visible data and error state                           |

## Options

```ts
const result = await useConvexQuery(api.projects.list, args, {
  server: true,
  subscribe: true,
  auth: 'optional',
  keepPreviousData: false,
  initialData: [],
  transform: (projects) => projects.filter((project) => !project.archived),
})
```

`server` and `subscribe` inherit module defaults. `auth` has a fixed per-query default of `optional` and cannot be changed globally.

## Skip execution

Pass the literal `'skip'` instead of arguments:

```ts
const args = computed(() =>
  selectedProjectId.value ? { projectId: selectedProjectId.value } : 'skip',
)

const project = await useConvexQuery(api.projects.get, args)
```

Skipped queries do not execute and are not pending.

## Transform data

A transform runs after each fetched or subscribed result:

```ts
const { data: names } = await useConvexQuery(
  api.projects.list,
  {},
  {
    transform: (projects) => projects.map((project) => project.name),
  },
)
```

The composable instance owns the transformed value. Convex still owns the raw query result.

## Refresh versus subscriptions

Live queries normally update after relevant mutations without `refresh()`. Use `refresh()` for one-shot queries, operator controls, or recovery from a known transient failure.

Do not refresh after every mutation. That duplicates work already driven by the subscription.

## Cleanup and identity

The composable stops its listener with the owning Vue scope. Identity-owned data clears synchronously when the stable user identity changes. `keepPreviousData` never carries results to another identity.

See the [composables reference](/docs/reference/composables#useconvexquery) for the exact type contract.

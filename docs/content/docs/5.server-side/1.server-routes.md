`serverConvex` is auto-imported in your `server/` directory.

::callout{icon="i-lucide-info"}
**Important naming distinction:** This page covers **Nitro server code** (`server/api/*`, `server/middleware/*`, webhooks, jobs). It does **not** cover Nuxt app route middleware (`defineNuxtRouteMiddleware`).
::

If you need Convex data inside `defineNuxtRouteMiddleware()` (for auth/permissions during navigation), use one-shot APIs:

- client navigation: `useConvex()`
- SSR navigation: `serverConvex(event)`

`useConvexQuery` and `useConvexPaginatedQuery` are setup-scope composables and should not be used inside route middleware/plugins.

## serverConvex

`serverConvex(event, options?)` returns a **request-scoped caller** — the single server-call API. It owns exactly one lazily-resolved auth token and one `ConvexHttpClient` for the request, then exposes `query`, `mutation`, and `action`. Every call passes an explicit args object; a no-argument function is called with `{}`.

```ts
function serverConvex(event: H3Event, options?: ServerConvexOptions): ServerConvexCaller

interface ServerConvexCaller {
  getToken: () => Promise<string | null>
  query: <Q>(query: Q, args: FunctionArgs<Q>) => Promise<FunctionReturnType<Q>>
  mutation: <M>(mutation: M, args: FunctionArgs<M>) => Promise<FunctionReturnType<M>>
  action: <A>(action: A, args: FunctionArgs<A>) => Promise<FunctionReturnType<A>>
}
```

| Parameter | Type                  | Description                                  |
| --------- | --------------------- | -------------------------------------------- |
| `event`   | `H3Event`             | Current Nitro event (used for config + auth) |
| `options` | `ServerConvexOptions` | Optional auth policy / token override        |

Reuse one caller for every call in a request — the token and client are resolved once and shared.

## Query

```ts [server/api/posts.get.ts]
import { api } from '#convex/api'

export default defineEventHandler(async (event) => {
  const posts = await serverConvex(event).query(api.posts.list, { status: 'published' })

  return posts
})
```

## ServerConvexOptions

```ts
interface ServerConvexOptions {
  auth?: 'optional' | 'required' | 'none'
  authToken?: string
}
```

- `auth: 'required'` — reject with an authentication error if no token is available.
- `auth: 'none'` — never attach a token; call anonymously.
- `auth: 'optional'` (default) — attach the request's token if present, otherwise call anonymously.
- `authToken` — supply a token explicitly instead of deriving it from the request.

::callout{icon="i-lucide-info" to="/docs/advanced/logging"}
Enable module-level logging via the `logging` config option for debugging server-side operations.
::

## Patterns

### Webhook Handler

```ts [server/api/webhooks/stripe.post.ts]
import { api } from '#convex/api'
import Stripe from 'stripe'

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!)

export default defineEventHandler(async (event) => {
  const body = await readRawBody(event)
  const signature = getHeader(event, 'stripe-signature')!

  // Verify webhook
  const stripeEvent = stripe.webhooks.constructEvent(
    body!,
    signature,
    process.env.STRIPE_WEBHOOK_SECRET!,
  )

  const convex = serverConvex(event)

  // Handle the event
  switch (stripeEvent.type) {
    case 'checkout.session.completed': {
      const session = stripeEvent.data.object
      await convex.mutation(api.subscriptions.activate, {
        userId: session.metadata!.userId,
        stripeSubscriptionId: session.subscription as string,
      })
      break
    }

    case 'customer.subscription.deleted': {
      const subscription = stripeEvent.data.object
      await convex.mutation(api.subscriptions.cancel, {
        stripeSubscriptionId: subscription.id,
      })
      break
    }
  }

  return { received: true }
})
```

### Authenticated API Route

Pass a token explicitly via `authToken`, or use `auth: 'required'` to reject unauthenticated requests. When you set `authToken`, the caller uses it directly instead of deriving one from the request:

```ts [server/api/user/profile.get.ts]
import { api } from '#convex/api'

export default defineEventHandler(async (event) => {
  const config = useRuntimeConfig()

  // Get auth token from session cookie
  const sessionCookie = getCookie(event, 'better-auth.session_token')
  if (!sessionCookie) {
    throw createError({ statusCode: 401, message: 'Unauthorized' })
  }

  // Exchange session for JWT token
  const tokenResponse = await $fetch<{ token?: string }>(
    `${config.public.convex.siteUrl}/api/auth/convex/token`,
    { headers: { Cookie: `better-auth.session_token=${sessionCookie}` } },
  )

  if (!tokenResponse?.token) {
    throw createError({ statusCode: 401, message: 'Invalid session' })
  }

  // Fetch with the supplied token
  const profile = await serverConvex(event, { authToken: tokenResponse.token }).query(
    api.users.getProfile,
    {},
  )

  return profile
})
```

### Background Job Trigger

```ts [server/api/cron/daily-digest.post.ts]
import { api } from '#convex/api'

export default defineEventHandler(async (event) => {
  // Verify cron secret
  const cronSecret = getHeader(event, 'x-cron-secret')
  if (cronSecret !== process.env.CRON_SECRET) {
    throw createError({ statusCode: 401 })
  }

  // Trigger digest generation
  const result = await serverConvex(event).action(api.notifications.sendDailyDigests, {})

  return { sent: result.count }
})
```

### Server Middleware

```ts [server/middleware/analytics.ts]
import { api } from '#convex/api'

export default defineEventHandler(async (event) => {
  // Only track page views
  if (!event.path.startsWith('/api/')) {
    // Fire and forget - don't await
    serverConvex(event)
      .mutation(api.analytics.trackPageView, {
        path: event.path,
        userAgent: getHeader(event, 'user-agent') ?? '',
        timestamp: Date.now(),
      })
      .catch(() => {
        // Ignore errors - analytics shouldn't break the request
      })
  }
})
```

### External API Integration

```ts [server/api/import/github.post.ts]
import { api } from '#convex/api'

export default defineEventHandler(async (event) => {
  const { repoUrl } = await readBody(event)

  // Fetch from external API
  const [owner, repo] = repoUrl.replace('https://github.com/', '').split('/')
  const issues = await $fetch(`https://api.github.com/repos/${owner}/${repo}/issues`, {
    headers: { Authorization: `Bearer ${process.env.GITHUB_TOKEN}` },
  })

  // Store in Convex
  const imported = await serverConvex(event).mutation(api.issues.importFromGithub, {
    repoUrl,
    issues: issues.map((i: any) => ({
      title: i.title,
      body: i.body,
      githubId: i.id,
    })),
  })

  return { imported: imported.count }
})
```

## Error Handling

```ts
export default defineEventHandler(async (event) => {
  try {
    const result = await serverConvex(event).mutation(api.tasks.create, { text: 'New task' })
    return result
  } catch (error) {
    // Convex errors have a message property
    if (error instanceof Error) {
      throw createError({
        statusCode: 400,
        message: error.message,
      })
    }
    throw error
  }
})
```

## Debugging

Enable logging in your `nuxt.config.ts` to debug server-side operations:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    url: process.env.CONVEX_URL,
    logging: {
      enabled: true, // Enable canonical log events
    },
  },
})
```

Server-side operations emit `operation:complete` events:

```
[better-convex-nuxt] ▷ query ✓ api.posts.list 45ms
[better-convex-nuxt] ▷ mutation ✓ api.tasks.create 67ms
```

See [Logging](/docs/advanced/logging) for full configuration options.

## TypeScript

Full type inference:

```ts
// Return type is inferred from the query
const posts = await serverConvex(event).query(api.posts.list, { status: 'published' })
// posts: Post[]

// Args are typed
await serverConvex(event).mutation(
  api.posts.create,
  { title: 123 }, // Type error: title should be string
)
```

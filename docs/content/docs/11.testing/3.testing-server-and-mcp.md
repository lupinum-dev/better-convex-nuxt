Use this page when the thing under test crosses the Nuxt server or MCP boundary.

## Server Helper Tests

Trellis has focused server-helper tests around:

- `serverConvexQuery`
- `serverConvexMutation`
- `serverConvexAction`

Those tests verify:

- request shape
- auth-mode handling
- error reporting
- current-event resolution

That is the right level for server helper contracts.

## MCP And Integration Flows

For MCP-oriented or integration-oriented flows, use the internal harness and example suites instead of inventing one-off mocks for the whole pipeline.

Good examples in this repo:

- MCP smoke coverage
- internal-harness server API tests
- example 07 MCP reference tests

## Testing Reset And Seed Helpers

The internal harness includes test-only Convex helpers for resetting and seeding state. That is useful for repeatable MCP and server integration tests where isolated setup matters more than tiny unit-level mocks.

## Rule Of Thumb

- use unit tests for helper logic
- use Trellis testing helpers for protected backend behavior
- use server helper tests for Nitro contracts
- use MCP or e2e-style suites for full transport flows

Do not flatten those layers into one giant “integration test” bucket.

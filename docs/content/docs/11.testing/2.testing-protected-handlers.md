Use this page when the thing under test is backend authorization, tenancy, or handler behavior.

## Seed Tenants And Users

The common path is:

```ts
const ctx = createTestContext({
  schema,
  modules,
  tenant: { table: 'organizations', field: 'organizationId' },
})

const team = await ctx.seedTenant({
  name: 'Acme',
  users: {
    alice: { role: 'member' },
    bob: { role: 'viewer' },
  },
})
```

That gives you authenticated callers with realistic tenant and role state.

## Test Real Protected Mutations

```ts
const postId = await team.users.alice.mutation(api.posts.create, {
  title: 'Seeded by helper',
  content: 'This goes through the real scoped mutation.',
})

await expect(team.users.bob.mutation(api.posts.publish, { id: postId })).rejects.toThrow(
  /Forbidden/,
)
```

This is the right level for permission rules. The test goes through the real handler path instead of a hand-built mock actor.

## Forward Principals Explicitly

When you need to prove parity between browser and forwarded callers:

```ts
const trustedCaller = ctx.asPrincipal({
  kind: 'user',
  userId: team.users.viewer.authId,
})
```

Then call the same protected handlers and assert that the permission result stays the same.

## What To Expect

The changelog is most useful when it emphasizes:

- user-facing changes
- upgrade notes
- support-boundary changes

Less useful patterns:

- raw commit dumps
- internal-only refactor lists with no user impact

## Current State

The existing changelog already contains important support-boundary notes, but it still mixes older release styles. Future entries should stay closer to one public format with upgrade notes that are easy to scan.

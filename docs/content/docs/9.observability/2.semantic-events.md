Use this page to look up what a Trellis observation event means.

## Core Dimensions

- `transport`: `browser`, `nuxt-server`, `convex`, `mcp`, `service`, `webhook`
- `status`: `success`, `deny`, `error`, `skip`
- `family`: identity, authorization, trust, operations, mcp, browser

## Identity And Authorization Events

- `principal.resolved`
- `actor.resolved`
- `actor.missing`
- `guard.allowed`
- `guard.denied`
- `authorize.allowed`
- `authorize.denied`
- `rls.denied`

These tell you who the runtime believes the caller is and where authorization failed.

## Trust-Boundary Events

- `db.cross_tenant.used`
- `db.raw.used`
- `service.access.checked`
- `service.access.denied`

These are the high-signal events for code-review-sensitive seams.

## Operation Events

- `operation.preview.started`
- `operation.preview.completed`
- `operation.confirm.validated`
- `operation.confirm.drifted`
- `operation.execute.completed`
- `operation.execute.failed`

These matter most for destructive flows and operation-backed MCP tools.

## MCP Events

- `tool.called`
- `tool.denied`
- `tool.confirmation.required`
- `tool.executed`
- `tool.failed`

These tell you whether the tool was visible, denied, awaiting confirmation, or actually executed.

## Browser And Runtime Events

- `auth.session.checked`
- `query.subscribed`
- `query.updated`
- `query.unsubscribed`
- `query.failed`
- `mutation.completed`
- `mutation.failed`
- `action.completed`
- `action.failed`
- `upload.completed`
- `upload.failed`
- `connection.lost`
- `connection.restored`

- `guard.auth_required`
- `guard.denied`

- `service.access.denied`
- `tool.capability_denied`
- `tool.disabled`

- `query.failed`
- `mutation.failed`
- `action.failed`
- `upload.failed`
- `operation.execute.failed`

## Denial Explanations

Trellis denial explanations can include:

- `decision`
- `message`
- `policy`
- `tenantId`
- `suggestedAction`

Suggested actions are intentionally small and concrete, such as:

- `sign_in`
- `invite_to_workspace`
- `grant_capability`
- `switch_tenant`
- `retry_with_confirmation`
- `contact_admin`

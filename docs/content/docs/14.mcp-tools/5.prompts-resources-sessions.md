This page covers the advanced MCP surfaces beyond tools.

## Prompts

Trellis re-exports MCP prompt helpers, so prompt files can be defined alongside tools.

Example prompt usage:

- planning a runbook workflow before mutating anything

Prompts are good for structured guidance, not for replacing backend authorization.

## Resources

Resources are good for exposing durable read-only context such as:

- app guides
- inventories
- policy descriptions

The MCP reference example exposes a resource with a Markdown guide for the available MCP capabilities.

## Sessions

When `mcp.sessions` is enabled, `useMcpSession()` gives you a storage-backed session namespace scoped to the MCP session ID.

That is useful for:

- saved focus or preference state
- per-session dynamic tool registration

It is not a replacement for durable app data in Convex.

## Server Registration

`useMcpServer()` exposes the active MCP server instance and lets you register or remove tools, prompts, and resources during a live MCP request context.

That is the advanced path for dynamic session-bound or request-bound server surfaces.

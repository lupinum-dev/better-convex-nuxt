Use this page when the MCP runtime exists and you need to add real tools.

## The Main Surface

Trellis gives you two related paths:

- `defineTool(...)`
- `tool(...)` / `tool.fromOperation(...)` from `defineMcpApp(...)`

Use the runtime factory when tool capability checks and principal handling depend on the app runtime.

## Common Tool Shape

Tool definition decides:

- schema
- underlying Convex ref
- operation type (`query`, `mutation`, `action`)
- capability gate
- summary or result mapping

## Start With Common-Case Tools

Good first tools:

- read-only query tools
- small mutation tools with stable args

Avoid starting the MCP surface with destructive actions or dynamic session registration unless the app actually needs them on day one.

## Capability Gates

Tool visibility and execution can be gated by the capability snapshot resolved in `defineMcpApp(...)`.

That keeps coarse tool availability outside the business authorization layer, while the backend still owns real authorization.

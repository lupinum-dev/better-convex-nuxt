Use this page for the narrowest safe path from zero to one working Trellis MCP tool.

## 1. Define The MCP Endpoint

The endpoint handler is small:

```ts
export default defineMcpHandler({
  name: 'mcp-reference',
  browserRedirect: '/',
})
```

## 2. Authenticate MCP Requests In Middleware

The example pattern is:

1. read `Authorization: Bearer ...`
2. hash the token
3. validate the hash in Convex
4. map the result into `event.context.mcpAuth`

That keeps bearer tokens hashed at rest and gives the runtime a clean transport identity.

## 3. Define The MCP Runtime

```ts
export const mcpRuntime = defineMcpApp({
  callConvex: async (event, principal) => createServerConvexCaller(event, { principal }),
  resolvePrincipal: async (event) => getMcpPrincipal(event),
  resolveCapabilities: async ({ principal, convex }) => {
    // derive tool-level capabilities
  },
})
```

## 4. Export A Tool

Use the runtime tool factory:

```ts
export const tool = mcpRuntime.tool
```

Then register one tool file under `server/mcp/tools/*`.

## 5. Start With A Read Tool

The safe first tool is a read-only query-backed tool. Destructive tools belong later because Trellis treats them differently on purpose.

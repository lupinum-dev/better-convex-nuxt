Use this page when an MCP tool can delete, archive, revoke, or otherwise perform destructive work.

The correct Trellis path is operation-backed tooling.

## Use `tool.fromOperation(...)`

Example:

```ts
export default tool.fromOperation(removeRunbookOp, {
  execute: remove,
  preview: previewRemove,
  capability: 'writeWorkspaceRunbooks',
  meta: {
    name: 'delete-runbook',
  },
})
```

## Why This Is Required

Trellis binds destructive tools to:

- an operation with a stable `id`
- an execute ref projected from that operation
- a preview ref projected from that same operation

That is how preview, confirmation, and execute stay tied to one protected business action.

## Confirmation Flow

The expected path is:

1. preview
2. present summary and confirmation state
3. confirm
4. execute

If confirmation drifts, Trellis denies execution instead of trusting stale preview output.

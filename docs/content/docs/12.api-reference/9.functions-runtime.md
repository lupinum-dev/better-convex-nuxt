Use this entrypoint when you are building the protected backend runtime for a principal-first Trellis app. This is where you define how handlers load actors, apply guards, and expose stable internal seams for non-browser callers.

Do not start here for ordinary Nuxt app work. Start with module setup, composables, auth, and server helpers first.

## Main APIs

### `defineTrellis(...)`

`defineTrellis(...)` is the main advanced runtime builder. It combines:

- principal resolution
- actor resolution
- structured guarded handlers
- optional tenant isolation / RLS / triggers
- raw and structured handler registration
- explicit `ctx.db.crossTenant` / `ctx.db.raw` access on query and mutation handlers

Use it when your app owns a real authorization model and you want one canonical protected runtime.

Public-access handlers use the same `query(...)` / `mutation(...)` builders with `guard: open`. The runtime does not ship a separate `publicQuery` / `publicMutation` builder family.

### `defineOperation(...)` and `previewOf(...)`

Use `defineOperation(...)` for reusable protected business definitions that can be registered in more than one place or exposed through more than one transport.

Use `previewOf(...)` when a destructive workflow needs a read-only confirmation phase before execution.

### `definePrincipal(...)`

Use `definePrincipal(...)` to model transport identity, not business role. Examples:

- anonymous browser caller
- authenticated end user
- service account or automation caller

Resolve the principal here, then derive the actor later from app data.

### `createComponentBridge(...)`

`createComponentBridge(...)` is an advanced seam for exposing a stable inventory of root refs that forward `{ ...args, principal }` into isolated Convex components.

Use it when server routes, MCP tools, or automations need a durable root surface that should stay stable even if internal component structure changes.

Do not use it as a substitute for authorization. The bridge forwards identity; your protected handlers still own the business checks.

## Recommended Boundary

- Public Nuxt app work: composables and server helpers
- Protected Convex business runtime: `defineTrellis(...)`
- Stable non-browser seam: `createComponentBridge(...)`

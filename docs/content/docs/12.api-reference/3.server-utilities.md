This page separates the **published package export** from **Nuxt-only server auto-imports**.

## Package Export

Import these from `better-convex-nuxt/server`:

```ts
import {
  serverConvexQuery,
  serverConvexMutation,
  serverConvexAction,
  type ServerConvexOptions,
} from 'better-convex-nuxt/server'
```

## Nuxt Server Auto-Imports

These are available in Nuxt server files from `#imports` when the module is installed:

- `serverConvexQuery`
- `serverConvexMutation`
- `serverConvexAction`
- `serverConvexClearAuthCache`
- `validateConvexArgs`

`serverConvexClearAuthCache` and `validateConvexArgs` are **not** package subpath exports.

## serverConvexQuery

```ts
await serverConvexQuery(event, api.tasks.list, { status: 'active' })
await serverConvexQuery(api.tasks.list, { status: 'active' })
```

The event is optional when Nitro async request context is available.

## serverConvexMutation

```ts
await serverConvexMutation(event, api.tasks.create, { text: body.text })
await serverConvexMutation(api.tasks.create, { text: body.text })
```

## serverConvexAction

```ts
await serverConvexAction(event, api.reports.generate, { reportId })
await serverConvexAction(api.reports.generate, { reportId })
```

## ServerConvexOptions

```ts
interface ServerConvexOptions {
  auth?: 'auto' | 'required' | 'none' | 'trusted'
  authToken?: string
  actor?: { userId: string }
  trustedCallerKey?: string
}
```

### Auth Modes

| Mode         | Behavior                                             |
| ------------ | ---------------------------------------------------- |
| `'auto'`     | Resolve auth from the request when possible          |
| `'required'` | Throw if no auth token can be resolved               |
| `'none'`     | Skip auth entirely                                   |
| `'trusted'`  | Inject trusted-caller transport into the Convex args |

### Trusted Caller Mode

Use `auth: 'trusted'` when a Nuxt server route is intentionally forwarding a server-owned actor into a Convex handler built with `createFunctions(...)` in `convex/functions.ts`.

```ts
await serverConvexMutation(event, api.posts.create, args, {
  auth: 'trusted',
  actor: { userId: actor.userId },
  trustedCallerKey: process.env.CONVEX_TRUSTED_CALLER_KEY,
})
```

## serverConvexClearAuthCache

Nuxt server auto-import only:

```ts
await serverConvexClearAuthCache(sessionToken)
```

Use this for custom revocation or logout flows. Ordinary `useConvexAuth().signOut()` already clears the current session cache automatically.

## validateConvexArgs

Nuxt server auto-import only:

```ts
const body = await readValidatedBody(event, validateConvexArgs(v.object(createPost.args)))
```

Use it when an H3 route should validate input with the same Convex validator shape used by your backend code.

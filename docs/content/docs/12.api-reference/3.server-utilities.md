This page separates the **published package export** from **Nuxt-only server auto-imports**.

## Package Export

Import these from `@lupinum/trellis/server`:

```ts
import {
  serverConvexQuery,
  serverConvexMutation,
  serverConvexAction,
  type ServerConvexOptions,
} from '@lupinum/trellis/server'
```

## Nuxt Server Auto-Imports

These are available in Nuxt server files from `#imports` when the module is installed:

- `serverConvexQuery`
- `serverConvexMutation`
- `serverConvexAction`
- `serverConvexClearAuthCache`
- `validateConvexArgs`

`serverConvexClearAuthCache` and `validateConvexArgs` are **not** package subpath exports.

## serverConvexQuery

```ts
await serverConvexQuery(event, api.tasks.list, { status: 'active' }, { auth: 'auto' })
await serverConvexQuery(api.tasks.list, { status: 'active' }, { auth: 'auto' })
```

The event is optional when Nitro async request context is available.

## serverConvexMutation

```ts
await serverConvexMutation(event, api.tasks.create, { text: body.text }, { auth: 'auto' })
await serverConvexMutation(api.tasks.create, { text: body.text }, { auth: 'auto' })
```

## serverConvexAction

```ts
await serverConvexAction(event, api.reports.generate, { reportId }, { auth: 'auto' })
await serverConvexAction(api.reports.generate, { reportId }, { auth: 'auto' })
```

## ServerConvexOptions

```ts
interface ServerConvexOptions {
  auth?: 'auto' | 'required' | 'none' | 'trusted'
  authToken?: string
  actor?: { userId: string }
  trustedCallerKey?: string
}
```

### Auth Modes

| Mode         | Behavior                                                  |
| ------------ | --------------------------------------------------------- |
| `'auto'`     | Resolve auth from the request when possible               |
| `'required'` | Throw if no auth token can be resolved                    |
| `'none'`     | Skip auth entirely                                        |
| `'trusted'`  | Low-level compatibility mode for trusted-caller transport |

### Preferred Pattern

For principal-first Trellis apps, prefer `auth: 'none'` and forward an explicit `principal` into a root ref:

```ts
await serverConvexMutation(
  event,
  internal.posts.publishPost,
  {
    id: postId,
    principal: {
      kind: 'service',
      service: 'content-webhook',
    },
  },
  {
    auth: 'none',
  },
)
```

### Low-Level Trusted Mode

`auth: 'trusted'` still exists as a low-level compatibility mode for code that still uses trusted-caller transport.

```ts
await serverConvexMutation(event, api.posts.create, args, {
  auth: 'trusted',
  actor: { userId: actor.userId },
  trustedCallerKey: process.env.CONVEX_TRUSTED_CALLER_KEY,
})
```

## serverConvexClearAuthCache

Nuxt server auto-import only:

```ts
await serverConvexClearAuthCache(sessionToken)
```

Use this for custom revocation or logout flows. Ordinary `useConvexAuth().signOut()` already clears the current session cache automatically.

## validateConvexArgs

Nuxt server auto-import only:

```ts
const body = await readValidatedBody(event, validateConvexArgs(v.object(createPost.args)))
```

Use it when an H3 route should validate input with the same Convex validator shape used by your backend code.

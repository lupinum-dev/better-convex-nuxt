## Overview

The module emits a small set of Nuxt runtime hooks for cross-cutting concerns like global error
handling, connection banners, analytics, and auth reactions.

```ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('trellis:connection:changed', ({ state }) => {
    console.log('Convex connection:', state)
  })

  nuxtApp.hook('trellis:auth:changed', ({ isAuthenticated, user }) => {
    console.log('Auth changed:', isAuthenticated, user?.id)
  })
})
```

## Hook Matrix

| Hook                         | Scope    | Fires When                                                                 | Payload                                                            |
| ---------------------------- | -------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| `trellis:mutation:success`   | Public   | A mutation completes successfully                                          | `{ functionPath, operation: 'mutation', args, result, duration }`  |
| `trellis:mutation:error`     | Public   | A mutation fails                                                           | `{ functionPath, operation: 'mutation', args, error, duration }`   |
| `trellis:action:success`     | Public   | An action completes successfully                                           | `{ functionPath, operation: 'action', args, result, duration }`    |
| `trellis:action:error`       | Public   | An action fails                                                            | `{ functionPath, operation: 'action', args, error, duration }`     |
| `trellis:unauthorized`       | Public   | Unauthorized recovery is triggered for a Convex query, mutation, or action | `{ error, source, functionName?, redirectTo }`                     |
| `trellis:connection:changed` | Public   | The derived connection phase changes                                       | `{ state, previousState, connection, previousConnection }`         |
| `trellis:auth:changed`       | Public   | The effective authenticated user changes                                   | `{ isAuthenticated, previousIsAuthenticated, user, previousUser }` |
| `trellis:auth:refresh`       | Internal | A Convex auth refresh is requested                                         | No payload                                                         |
| `trellis:auth:invalidate`    | Internal | The live Convex auth transport is de-authenticated                         | No payload                                                         |

## Hook Details

### `trellis:mutation:success`

Fires after a successful `useConvexMutation()` call. **Use for:** analytics events, success toasts, audit logging.

| Field          | Type                      |
| -------------- | ------------------------- |
| `functionPath` | `string`                  |
| `operation`    | `'mutation'`              |
| `args`         | `Record<string, unknown>` |
| `result`       | `unknown`                 |
| `duration`     | `number`                  |

### `trellis:mutation:error`

Fires after a failed `useConvexMutation()` call. **Use for:** global error reporting (Sentry, LogRocket), error toasts, retry logic.

| Field          | Type                      |
| -------------- | ------------------------- |
| `functionPath` | `string`                  |
| `operation`    | `'mutation'`              |
| `args`         | `Record<string, unknown>` |
| `error`        | `ConvexCallError`         |
| `duration`     | `number`                  |

### `trellis:action:success`

Fires after a successful `useConvexAction()` call. **Use for:** analytics, success notifications.

| Field          | Type                      |
| -------------- | ------------------------- |
| `functionPath` | `string`                  |
| `operation`    | `'action'`                |
| `args`         | `Record<string, unknown>` |
| `result`       | `unknown`                 |
| `duration`     | `number`                  |

### `trellis:action:error`

Fires after a failed `useConvexAction()` call. **Use for:** global error reporting, error toasts.

| Field          | Type                      |
| -------------- | ------------------------- |
| `functionPath` | `string`                  |
| `operation`    | `'action'`                |
| `args`         | `Record<string, unknown>` |
| `error`        | `ConvexCallError`         |
| `duration`     | `number`                  |

### `trellis:unauthorized`

Fires when the module detects an unauthorized Convex call and starts configured recovery behavior. **Use for:** custom redirect logic, session expiry modals, logging auth failures.

| Field          | Type                  |
| -------------- | --------------------- |
| `error`        | `unknown`             |
| `source`       | `string`              |
| `functionName` | `string \| undefined` |
| `redirectTo`   | `string`              |

### `trellis:connection:changed`

Fires only when the derived connection phase changes, not when only raw counters like inflight
requests change. **Use for:** offline/reconnecting UI banners, connection-aware retry logic.

| Field                | Type                                            |
| -------------------- | ----------------------------------------------- |
| `state`              | `'connecting' \| 'connected' \| 'reconnecting'` |
| `previousState`      | `'connecting' \| 'connected' \| 'reconnecting'` |
| `connection`         | `ConnectionState`                               |
| `previousConnection` | `ConnectionState`                               |

::note
This hook does not fire on initial plugin boot.
::

### `trellis:auth:changed`

Fires when the effective auth identity changes on the client. **Use for:** analytics identity tracking, resetting app state on sign-out, navigation after auth changes.

| Field                     | Type                 |
| ------------------------- | -------------------- |
| `isAuthenticated`         | `boolean`            |
| `previousIsAuthenticated` | `boolean`            |
| `user`                    | `ConvexUser \| null` |
| `previousUser`            | `ConvexUser \| null` |

::note
This hook is identity-based. Same-user token refreshes do not emit it.
::

### `trellis:auth:refresh`

Internal hook used by the auth controller to trigger a fresh token exchange. You usually should not
listen to this directly unless you are extending module internals.

### `trellis:auth:invalidate`

Internal hook used by the auth controller during `useConvexAuth().signOut()`. It immediately
rebinds the live Convex client to an unauthenticated token source before the upstream Better Auth
logout request completes.

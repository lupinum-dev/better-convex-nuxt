## Overview

The module emits a small set of Nuxt runtime hooks for cross-cutting concerns like global error
handling, connection banners, analytics, and auth reactions.

```ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('convex:connection:changed', ({ state }) => {
    console.log('Convex connection:', state)
  })

  nuxtApp.hook('convex:auth:changed', ({ isAuthenticated, user }) => {
    console.log('Auth changed:', isAuthenticated, user?.id)
  })
})
```

## Hook Matrix

| Hook | Scope | Fires When | Payload |
|------|-------|------------|---------|
| `convex:mutation:success` | Public | A mutation completes successfully | `{ functionPath, operation: 'mutation', args, result, duration }` |
| `convex:mutation:error` | Public | A mutation fails | `{ functionPath, operation: 'mutation', args, error, duration }` |
| `convex:action:success` | Public | An action completes successfully | `{ functionPath, operation: 'action', args, result, duration }` |
| `convex:action:error` | Public | An action fails | `{ functionPath, operation: 'action', args, error, duration }` |
| `convex:unauthorized` | Public | Unauthorized recovery is triggered for a Convex query, mutation, or action | `{ error, source, functionName?, redirectTo }` |
| `convex:connection:changed` | Public | The derived connection phase changes | `{ state, previousState, connection, previousConnection }` |
| `convex:auth:changed` | Public | The effective authenticated user changes | `{ isAuthenticated, previousIsAuthenticated, user, previousUser }` |
| `better-convex:auth:refresh` | Internal | A Convex auth refresh is requested | No payload |
| `better-convex:auth:invalidate` | Internal | The live Convex auth transport is de-authenticated | No payload |

## Hook Details

### `convex:mutation:success`

Fires after a successful `useConvexMutation()` call.

| Field | Type |
|-------|------|
| `functionPath` | `string` |
| `operation` | `'mutation'` |
| `args` | `Record<string, unknown>` |
| `result` | `unknown` |
| `duration` | `number` |

### `convex:mutation:error`

Fires after a failed `useConvexMutation()` call.

| Field | Type |
|-------|------|
| `functionPath` | `string` |
| `operation` | `'mutation'` |
| `args` | `Record<string, unknown>` |
| `error` | `ConvexCallError` |
| `duration` | `number` |

### `convex:action:success`

Fires after a successful `useConvexAction()` call.

| Field | Type |
|-------|------|
| `functionPath` | `string` |
| `operation` | `'action'` |
| `args` | `Record<string, unknown>` |
| `result` | `unknown` |
| `duration` | `number` |

### `convex:action:error`

Fires after a failed `useConvexAction()` call.

| Field | Type |
|-------|------|
| `functionPath` | `string` |
| `operation` | `'action'` |
| `args` | `Record<string, unknown>` |
| `error` | `ConvexCallError` |
| `duration` | `number` |

### `convex:unauthorized`

Fires when the module detects an unauthorized Convex call and starts configured recovery behavior.

| Field | Type |
|-------|------|
| `error` | `unknown` |
| `source` | `string` |
| `functionName` | `string \| undefined` |
| `redirectTo` | `string` |

### `convex:connection:changed`

Fires only when the derived connection phase changes, not when only raw counters like inflight
requests change.

| Field | Type |
|-------|------|
| `state` | `'connecting' \| 'connected' \| 'reconnecting'` |
| `previousState` | `'connecting' \| 'connected' \| 'reconnecting'` |
| `connection` | `ConnectionState` |
| `previousConnection` | `ConnectionState` |

::note
This hook does not fire on initial plugin boot.
::

### `convex:auth:changed`

Fires when the effective auth identity changes on the client.

| Field | Type |
|-------|------|
| `isAuthenticated` | `boolean` |
| `previousIsAuthenticated` | `boolean` |
| `user` | `ConvexUser \| null` |
| `previousUser` | `ConvexUser \| null` |

::note
This hook is identity-based. Same-user token refreshes do not emit it.
::

### `better-convex:auth:refresh`

Internal hook used by the auth controller to trigger a fresh token exchange. You usually should not
listen to this directly unless you are extending module internals.

### `better-convex:auth:invalidate`

Internal hook used by the auth controller during `useConvexAuth().signOut()`. It immediately
rebinds the live Convex client to an unauthenticated token source before the upstream Better Auth
logout request completes.

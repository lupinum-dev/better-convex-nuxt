This page separates **published package imports** from **Nuxt-generated auto-imports**.

Start with [Which API Do I Use?](/docs/api-reference/which-api) if you are choosing between Nuxt auto-imports, server utilities, package exports, and generated aliases.

For the generated source-of-truth list, see [API Surface](/docs/api-reference/api-surface).

## Package Import

Import these from `@lupinum/trellis/composables`:

- `useConvex`
- `useConvexQuery`
- `useCachedQuery`
- `useConvexPaginatedQuery`
- `useConvexMutation`
- `useConvexAction`
- `useConvexUpload`
- `useConvexStorageUrl`
- `useConvexConnectionState`
- `useConvexAuth`
- `useConvexAuthActions`
- `getQueryKey`
- `prependTo`
- `appendTo`
- `removeFrom`
- `updateIn`

The same entrypoint also re-exports `ConvexCallError` and the public runtime types used by these composables.

## Nuxt Auto-Imports

The module also auto-imports those composables in Nuxt apps.

Additional Nuxt-only auto-imports:

- `executeConvexQuery`
- `usePermissions()` when `trellis.permissions.query` is configured
- `useAuthGuard()` when `trellis.permissions.query` is configured

## Query Composables

### useConvexQuery

See guide: [Queries](/docs/data-fetching/queries)

```ts
const { data, status, pending, error, refresh, clear, isStale } = await useConvexQuery(
  api.tasks.list,
  { status: 'active' },
  options,
)
```

Core options:

| Option             | Type                      | Default |
| ------------------ | ------------------------- | ------- |
| `server`           | `boolean`                 | `true`  |
| `subscribe`        | `boolean`                 | `true`  |
| `default`          | `() => RawT \| undefined` | —       |
| `transform`        | `(input) => DataT`        | —       |
| `keepPreviousData` | `boolean`                 | `false` |

### useCachedQuery

See guide: [Cached Queries](/docs/data-fetching/cached-queries)

```ts
const result = await useCachedQuery(api.posts.get, () => ({ id: postId.value }), {
  from: {
    query: api.posts.list,
    args: {},
    find: (items) => items.find((item) => item._id === postId.value),
  },
})
```

`useCachedQuery()` wraps `useConvexQuery()` and adds `isFromCache`. Await it when you want the SSR-blocking first result.

### useConvexPaginatedQuery

See guide: [Paginated Queries](/docs/data-fetching/paginated-queries)

```ts
const { results, status, loadMore, hasNextPage, isLoading, isExhausted, refetch, restart } =
  await useConvexPaginatedQuery(api.posts.listPaginated, {}, options)
```

Core options:

| Option             | Type                           | Default |
| ------------------ | ------------------------------ | ------- |
| `initialNumItems`  | `number`                       | `10`    |
| `server`           | `boolean`                      | `true`  |
| `subscribe`        | `boolean`                      | `true`  |
| `default`          | `() => Item[]`                 | —       |
| `transform`        | `(items) => TransformedItem[]` | —       |
| `keepPreviousData` | `boolean`                      | `false` |

Status values:

- `'skipped'`
- `'loading-first-page'`
- `'ready'`
- `'loading-more'`
- `'exhausted'`
- `'error'`

### executeConvexQuery

Nuxt auto-import only:

```ts
const result = await executeConvexQuery(api.tasks.list, { status: 'active' })
```

Use this for one-shot query execution when you do not want composable state.

## Mutations, Actions, Uploads

### useConvexMutation

See guide: [Mutations](/docs/mutations/mutations), [Optimistic Updates](/docs/mutations/optimistic-updates)

```ts
const createTask = useConvexMutation(api.tasks.create, {
  optimisticUpdate: (ctx, args) => {
    prependTo(ctx, api.tasks.list, {}, { _id: 'temp', text: args.text, completed: false })
  },
})
```

Returns a callable function with `data`, `status`, `pending`, `error`, and `reset()`.

### useConvexAction

See guide: [Actions](/docs/mutations/actions)

Same state shape as `useConvexMutation()`, without optimistic updates.

### useConvexUpload

See guide: [Single File Upload](/docs/file-uploads/single-file-upload), [Upload Queue](/docs/file-uploads/upload-queue)

```ts
const upload = useConvexUpload(api.files.generateUploadUrl, { maxConcurrent: 3 })
```

Returns a callable upload function plus queue state:

- `data`
- `status`
- `pending`
- `progress`
- `error`
- `items`
- `cancelItem()`
- `cancelAll()`
- `clearFinished()`
- `reset()`

### useConvexStorageUrl

See guide: [Storage URLs](/docs/file-uploads/storage-urls)

```ts
const url = useConvexStorageUrl(api.files.getUrl, storageId)
```

This wraps `useConvexQuery()` and automatically skips when `storageId` is nullish.

## Auth And Connection State

### useConvexAuth

See guide: [Authentication](/docs/auth-security/authentication), [Sign In & Sign Out](/docs/auth-security/sign-in-and-sign-out)

```ts
const {
  user,
  isAuthenticated,
  isPending,
  isAnonymous,
  isSessionExpired,
  client,
  refreshAuth,
  authError,
  signOut,
} = useConvexAuth()
```

Available only when `trellis.auth` is enabled.

### useConvexAuthActions

```ts
const { execute, status, pending, data, error, reset } = useConvexAuthActions()
```

Wrap Better Auth client calls when you want Convex auth refresh and redirect handling after success. This is for auth-side flows; it does not replace your app's own permission or tenant model.

### useConvexConnectionState

See guide: [Connection State](/docs/real-time/connection-state)

```ts
const { state, isConnected, isReconnecting, pendingMutations, shouldShowOfflineUi } =
  useConvexConnectionState()
```

## Permissions Auto-Imports

These are Nuxt-generated and only exist when `trellis.permissions.query` is configured.

### usePermissions

See guide: [Checking Permissions](/docs/permissions/checking-permissions)

```ts
const { ctx, role, plan, userId, tenantId, ready, pending, can } = usePermissions()
```

- `ctx` is the raw permission context query result
- `can(key)` returns a `ComputedRef<boolean>`
- `ready` means the permission context resolved to a non-null actor

### useAuthGuard

See guide: [Route Protection](/docs/auth-security/route-protection)

```ts
useAuthGuard({
  can: 'posts.create',
  redirectTo: '/',
  loginPath: '/auth/signin',
})
```

Options:

| Option       | Type               | Notes                          |
| ------------ | ------------------ | ------------------------------ |
| `can`        | permission key     | Checks `ctx.can[key] === true` |
| `check`      | `(ctx) => boolean` | Custom guard predicate         |
| `redirectTo` | route location     | Default `/`                    |
| `loginPath`  | route location     | Default `/auth/signin`         |

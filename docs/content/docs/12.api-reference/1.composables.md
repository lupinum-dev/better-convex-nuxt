This page separates **published package imports** from **Nuxt-generated auto-imports**.

For the generated source-of-truth list, see [API Surface](/docs/api-reference/api-surface).

## Package Import

Import these from `better-convex-nuxt/composables`:

- `useConvex`
- `useConvexQuery`
- `useCachedQuery`
- `useConvexPaginatedQuery`
- `useConvexMutation`
- `useConvexAction`
- `useConvexUpload`
- `useConvexStorageUrl`
- `useConvexConnectionState`
- `useConvexAuth`
- `useConvexAuthActions`
- `getQueryKey`
- `prependTo`
- `appendTo`
- `removeFrom`
- `updateIn`

The same entrypoint also re-exports `ConvexCallError` and the public runtime types used by these composables.

## Nuxt Auto-Imports

The module also auto-imports those composables in Nuxt apps.

Additional Nuxt-only auto-imports:

- `executeConvexQuery`
- `usePermissions()` when `convex.permissions.query` is configured
- `useAuthGuard()` when `convex.permissions.query` is configured

## Query Composables

### useConvexQuery

```ts
const { data, status, pending, error, refresh, clear, isStale } =
  await useConvexQuery(api.tasks.list, { status: 'active' }, options)
```

Core options:

| Option | Type | Default |
| ------ | ---- | ------- |
| `server` | `boolean` | `true` |
| `subscribe` | `boolean` | `true` |
| `default` | `() => RawT \| undefined` | — |
| `transform` | `(input) => DataT` | — |
| `keepPreviousData` | `boolean` | `false` |
| `shared` | `string` | — |
| `onData` | `(data) => void` | — |
| `onError` | `(error) => void` | — |

### useCachedQuery

```ts
const result = useCachedQuery(api.posts.get, () => ({ id: postId.value }), {
  from: {
    query: api.posts.list,
    args: {},
    find: (items) => items.find((item) => item._id === postId.value),
  },
})
```

`useCachedQuery()` wraps `useConvexQuery()` and adds `isFromCache`.

### useConvexPaginatedQuery

```ts
const { results, status, loadMore, hasNextPage, isLoading, isExhausted, refetch, restart } =
  await useConvexPaginatedQuery(api.posts.listPaginated, {}, options)
```

Core options:

| Option | Type | Default |
| ------ | ---- | ------- |
| `initialNumItems` | `number` | `10` |
| `server` | `boolean` | `true` |
| `subscribe` | `boolean` | `true` |
| `default` | `() => Item[]` | — |
| `transform` | `(items) => TransformedItem[]` | — |
| `keepPreviousData` | `boolean` | `false` |

Status values:

- `'skipped'`
- `'loading-first-page'`
- `'ready'`
- `'loading-more'`
- `'exhausted'`
- `'error'`

### executeConvexQuery

Nuxt auto-import only:

```ts
const result = await executeConvexQuery(api.tasks.list, { status: 'active' })
```

Use this for one-shot query execution when you do not want composable state.

## Mutations, Actions, Uploads

### useConvexMutation

```ts
const createTask = useConvexMutation(api.tasks.create, {
  optimisticUpdate: (ctx, args) => {
    prependTo(ctx, api.tasks.list, {}, { _id: 'temp', text: args.text, completed: false })
  },
})
```

Returns a callable function with `data`, `status`, `pending`, `error`, and `reset()`.

### useConvexAction

Same state shape as `useConvexMutation()`, without optimistic updates.

### useConvexUpload

```ts
const upload = useConvexUpload(api.files.generateUploadUrl, { maxConcurrent: 3 })
```

Returns a callable upload function plus queue state:

- `data`
- `status`
- `pending`
- `progress`
- `error`
- `items`
- `cancelItem()`
- `cancelAll()`
- `clearFinished()`
- `reset()`

### useConvexStorageUrl

```ts
const url = useConvexStorageUrl(api.files.getUrl, storageId)
```

This wraps `useConvexQuery()` and automatically skips when `storageId` is nullish.

## Auth And Connection State

### useConvexAuth

```ts
const { user, isAuthenticated, isPending, isAnonymous, isSessionExpired, client, refreshAuth, authError, signOut } =
  useConvexAuth()
```

Available only when `convex.auth` is enabled.

### useConvexAuthActions

```ts
const { execute, status, pending, data, error, reset } = useConvexAuthActions()
```

Wrap Better Auth client calls when you want Convex auth refresh and redirect handling after success.

### useConvexConnectionState

```ts
const { state, isConnected, isReconnecting, pendingMutations, shouldShowOfflineUi } =
  useConvexConnectionState()
```

## Permissions Auto-Imports

These are Nuxt-generated and only exist when `convex.permissions.query` is configured.

### usePermissions

```ts
const { ctx, role, plan, userId, tenantId, ready, pending, can } = usePermissions()
```

- `ctx` is the raw permission context query result
- `can(key)` returns a `ComputedRef<boolean>`
- `ready` means the permission context resolved to a non-null actor

### useAuthGuard

```ts
useAuthGuard({
  can: 'posts.create',
  redirectTo: '/',
  loginPath: '/auth/signin',
})
```

Options:

| Option | Type | Notes |
| ------ | ---- | ----- |
| `can` | permission key | Checks `ctx.can[key] === true` |
| `check` | `(ctx) => boolean` | Custom guard predicate |
| `redirectTo` | route location | Default `/` |
| `loginPath` | route location | Default `/auth/signin` |

Use this page for the **published MCP package export** and the **generated `#convex/mcp` alias**.

## Import Surfaces

Published package import:

```ts
import { defineTool, wrapSuccess, wrapError, useMcpServer, useMcpSession } from 'better-convex-nuxt/mcp'
```

Generated Nuxt alias:

```ts
import { defineTool, defineMcpHandler, defineMcpPrompt, defineMcpResource } from '#convex/mcp'
```

`#convex/mcp` is generated inside the Nuxt app by the module. It is not a bare npm specifier.

## Generic MCP Re-Exports

The entrypoint re-exports the generic Nuxt MCP Toolkit server helpers:

- `defineMcpHandler`
- `defineMcpPrompt`
- `defineMcpResource`
- `defineMcpTool`
- `useMcpServer`
- `useMcpSession`
- `completable`
- `imageResult`
- `extractToolNames`

Use `defineTool()` when the tool is Convex-backed and should reuse shared args, auth gating, result envelopes, and trusted-caller-aware scoping.

## defineTool

```ts
import { defineTool } from '#convex/mcp'
```

Core options:

| Option | Type | Default | Notes |
| ------ | ---- | ------- | ----- |
| `schema` | shared args definition | required | From `defineArgs()` |
| `handler` | `(args, ctx) => unknown` | required | Can return plain data or `ctx.ok(...)` |
| `name` | `string` | filename-derived | Required for rate limiting |
| `description` | `string` | schema description | Tool description |
| `operation` | `'query' \| 'mutation' \| 'action'` | `'mutation'` | Drives annotation defaults |
| `annotations` | partial MCP annotations | auto | Override annotation defaults |
| `auth` | `'required' \| 'optional' \| 'none'` | `'none'` | Actor requirement |
| `check` | `(actor) => boolean \| Promise<boolean>` | — | Visibility and execution guard |
| `scoped` | `boolean` | `false` | Trusted-caller-aware scoped path |
| `resolveAuth` | `(event) => actor \| null` | context resolver | Override auth resolution |
| `destructive` | `boolean` | `false` | Enables preview/confirmation flow |
| `maxItems` | `{ field, limit }` | — | Caps bulk array size |
| `rateLimit` | `{ max, window }` | — | In-memory limiter |
| `preview` | `(args, ctx) => PreviewResult \| string` | — | First-call preview for destructive tools |
| `group` | `string` | directory-derived | Grouping hint |
| `tags` | `string[]` | — | Free-form metadata |
| `outputSchema` | `ZodRawShape` | — | Output schema metadata |
| `inputExamples` | partial args array | auto | Example inputs |
| `middleware` | `(args, ctx, next) => result` | — | One custom middleware chain entry |
| `enabled` | `(event) => boolean` | — | Per-request visibility guard |
| `cache` | `McpToolCache` | — | Passed through to the toolkit |

## Handler Context

`defineTool()` handlers receive:

```ts
interface ConvexToolHandlerCtx {
  event: H3Event
  actor: McpAuthIdentity | null
  query(fn, args?)
  mutation(fn, args?)
  action(fn, args?)
  ok(data, summary?)
  error(category, message, issues?)
  preview(preview)
  blocked(preview)
}
```

## Visibility Semantics

`defineTool()` applies visibility in this order:

1. `enabled(event)`
2. `auth`
3. `scoped`
4. `check(actor)`

`scoped: true` hides the tool when no actor exists or when `actor.tenantId` is missing.

## Generic MCP Exports

```ts
import {
  defineMcpHandler,
  defineMcpPrompt,
  defineMcpResource,
  defineMcpTool,
  useMcpServer,
  useMcpSession,
} from '#convex/mcp'
```

Use `defineTool()` for Convex-backed tool ergonomics. Use the generic exports from the same entrypoint when you need prompts, resources, custom handlers, or session/dynamic server features directly.

## defineTool

```ts
import { defineTool } from '#convex/mcp'
```

Define an MCP tool from a shared args definition. Returns a `McpToolDefinition` for `@nuxtjs/mcp-toolkit`.

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `schema` | `SchemaDefinition` | **required** | Shared args definition from `defineArgs` |
| `handler` | `(args, ctx) => unknown` | **required** | Tool handler. Return data or use `ctx.ok(...)`. |
| `name` | `string` | auto (filename) | Tool name |
| `description` | `string` | `schema.meta.description` | Tool description |
| `operation` | `'query' \| 'mutation' \| 'action'` | `'mutation'` | Convex operation type (drives annotations) |
| `annotations` | `Partial<McpToolAnnotations>` | auto-derived | Override MCP annotations |
| `auth` | `'required' \| 'optional' \| 'none'` | `'none'` | Auth requirement |
| `check` | `(actor) => boolean \| Promise<boolean>` | тАФ | Optional actor check before the handler runs and in default visibility |
| `destructive` | `boolean` | `false` | Enable two-call confirmation flow |
| `maxItems` | `{ field: string; limit: number }` | тАФ | Cap array field size |
| `rateLimit` | `{ max: number; window: string }` | тАФ | In-memory rate limit (needs `name`) |
| `preview` | `(args, ctx) => PreviewResult` | тАФ | Preview for destructive tools |
| `group` | `string` | auto (directory) | Tool group |
| `tags` | `string[]` | тАФ | Free-form tags |
| `outputSchema` | `ZodRawShape` | тАФ | Output schema (auto-wrapped in envelope) |
| `inputExamples` | `Partial<Args>[]` | auto from metadata | Explicit input examples |
| `middleware` | `(args, ctx, next) => Result` | тАФ | Custom middleware |
| `enabled` | `(event) => boolean` | тАФ | Per-request visibility guard layered on top of auth/scoped/check visibility |
| `cache` | `McpToolCache` | тАФ | Cache config (passthrough) |

## Result Envelope Helpers

```ts
import { wrapSuccess, wrapError, wrapPreview, withSummary } from 'better-convex-nuxt/mcp'
```

### wrapSuccess(value)

Wrap a value in the success envelope. Called automatically by the framework тАФ you rarely need this directly.

### wrapError(category, message, issues?)

```ts
wrapError('not_found', 'Note not found.')
wrapError('validation', 'Invalid input.', [{ field: 'email', message: 'Invalid email' }])
```

| Param | Type | Description |
|-------|------|-------------|
| `category` | `ConvexErrorCategory` | Error category |
| `message` | `string` | Human/agent-readable message |
| `issues` | `ConvexErrorIssue[]` | Optional field-level issues |

### wrapPreview(preview)

Wrap a `PreviewResult` for the confirmation flow. Called by the framework for destructive tools тАФ you rarely need this directly.

### withSummary(data, summary)

```ts
return withSummary({ id, title }, `Created note "${title}"`)
```

Marks handler return with a human-readable summary. The summary becomes the text content; data becomes structured content.

```ts

  | ConvexToolSuccessResult<T>    // { ok: true, data: T }
  | ConvexToolPreviewResult       // { ok: true, preview: PreviewResult, awaitingConfirmation: true }
  | ConvexToolErrorResult         // { ok: false, error: { category, message, retryable, issues? } }
```

### Visibility Semantics

`defineTool()` applies the same actor-level rules to discovery and execution:

1. explicit `enabled(event)` can hide the tool
2. `auth: 'required'` hides the tool for anonymous callers
3. `scoped: true` hides the tool when the actor has no `tenantId`
4. `check(actor)` hides the tool when the actor fails the check

Resource-aware decisions are still explicit: use `enabled`, middleware, or the handler when a lookup is required.

### Annotation Derivation

| Operation | readOnlyHint | destructiveHint | idempotentHint | openWorldHint |
|-----------|-------------|-----------------|----------------|--------------|
| `query` | `true` | `false` | `true` | `false` |
| `mutation` | `false` | `destructive` flag | `false` | `false` |
| `action` | `false` | `destructive` flag | `false` | `true` |

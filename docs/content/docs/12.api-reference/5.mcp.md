## defineConvexTool

```ts
import { defineConvexTool } from 'better-convex-nuxt/mcp'
```

Define an MCP tool from a shared Convex schema. Returns a `McpToolDefinition` for `@nuxtjs/mcp-toolkit`.

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `schema` | `ConvexSchemaDefinition` | **required** | Shared schema from `defineConvexSchema` |
| `handler` | `(args, extra) => unknown` | **required** | Tool handler. Return data — the framework wraps it. |
| `name` | `string` | auto (filename) | Tool name |
| `description` | `string` | `schema.meta.description` | Tool description |
| `operation` | `'query' \| 'mutation' \| 'action'` | `'mutation'` | Convex operation type (drives annotations) |
| `annotations` | `Partial<McpToolAnnotations>` | auto-derived | Override MCP annotations |
| `auth` | `'required' \| 'optional' \| 'none'` | `'none'` | Auth requirement |
| `require` | `string` | — | Permission to check (needs factory) |
| `destructive` | `boolean` | `false` | Enable two-call confirmation flow |
| `maxItems` | `{ field: string; limit: number }` | — | Cap array field size |
| `rateLimit` | `{ max: number; window: string }` | — | In-memory rate limit (needs `name`) |
| `preview` | `(args, ctx) => PreviewResult` | — | Preview for destructive tools |
| `group` | `string` | auto (directory) | Tool group |
| `tags` | `string[]` | — | Free-form tags |
| `outputSchema` | `ZodRawShape` | — | Output schema (auto-wrapped in envelope) |
| `inputExamples` | `Partial<Args>[]` | auto from metadata | Explicit input examples |
| `middleware` | `(args, ctx, next) => Result` | — | Custom middleware |
| `enabled` | `(event) => boolean` | — | Per-request visibility guard |
| `cache` | `McpToolCache` | — | Cache config (passthrough) |

## createConvexTools

```ts
import { createConvexTools } from 'better-convex-nuxt/mcp'
```

Create a tool factory with shared auth and permission config.

```ts
const { defineConvexTool } = createConvexTools({
  checkPermission,             // from your permissions config
  resolveAuth?: (event) => McpAuthIdentity | null,  // custom auth resolver
})
```

### CreateConvexToolsOptions

| Option | Type | Description |
|--------|------|-------------|
| `checkPermission` | `CheckPermissionFn<P>` | Permission check function |
| `resolveAuth` | `(event: H3Event) => McpAuthIdentity \| null` | Custom auth resolver (default: `event.context.mcpAuth`) |

## Result Envelope Helpers

```ts
import { wrapSuccess, wrapError, wrapPreview, withSummary } from 'better-convex-nuxt/mcp'
```

### wrapSuccess(value)

Wrap a value in the success envelope. Called automatically by the framework — you rarely need this directly.

### wrapError(category, message, issues?)

```ts
wrapError('not_found', 'Note not found.')
wrapError('validation', 'Invalid input.', [{ field: 'email', message: 'Invalid email' }])
```

| Param | Type | Description |
|-------|------|-------------|
| `category` | `ConvexErrorCategory` | Error category |
| `message` | `string` | Human/agent-readable message |
| `issues` | `ConvexErrorIssue[]` | Optional field-level issues |

### wrapPreview(preview)

Wrap a `PreviewResult` for the confirmation flow. Called by the framework for destructive tools — you rarely need this directly.

### withSummary(data, summary)

```ts
return withSummary({ id, title }, `Created note "${title}"`)
```

Marks handler return with a human-readable summary. The summary becomes the text content; data becomes structured content.

### McpAuthIdentity

```ts
interface McpAuthIdentity {
  readonly role: string
  readonly userId: string
}
```

### PreviewResult

```ts
interface PreviewResult {
  summary: string
  warn?: string
  affects?: Record<string, number>
  blocked?: boolean
}
```

### ConvexToolMiddlewareCtx

```ts
interface ConvexToolMiddlewareCtx<P extends string = string> {
  event: H3Event
  mcpAuth: McpAuthIdentity | null
  can: (permission: P, resource?: Resource) => boolean
}
```

### ConvexToolResult

```ts
type ConvexToolResult<T = unknown> =
  | ConvexToolSuccessResult<T>    // { ok: true, data: T }
  | ConvexToolPreviewResult       // { ok: true, preview: PreviewResult, awaitingConfirmation: true }
  | ConvexToolErrorResult         // { ok: false, error: { category, message, retryable, issues? } }
```

## ConvexToolOperation

```ts
type ConvexToolOperation = 'query' | 'mutation' | 'action'
```

### Annotation Derivation

| Operation | readOnlyHint | destructiveHint | idempotentHint | openWorldHint |
|-----------|-------------|-----------------|----------------|--------------|
| `query` | `true` | `false` | `true` | `false` |
| `mutation` | `false` | `destructive` flag | `false` | `false` |
| `action` | `false` | `destructive` flag | `false` | `true` |

Use this page when you need the built-in auth boundary components.

## Components

- `<ConvexAuthenticated>`: render only when auth is resolved and authenticated.
- `<ConvexUnauthenticated>`: render only when auth is resolved and unauthenticated.
- `<ConvexAuthLoading>`: render only while auth state is still pending.
- `<ConvexAuthError>`: global auth error state surface when enabled in the app flow.

## Scope

These components are globally registered only when `trellis.auth` is enabled.

## Use

They are best for page-level auth boundaries and login-page shells. They are not a replacement for backend permissions or route protection.

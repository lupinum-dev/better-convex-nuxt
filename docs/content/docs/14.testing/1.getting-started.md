## Overview

`better-convex-nuxt/testing` keeps the test story small on purpose:

- `convexTestConfig()` sets Vitest up for Convex-friendly execution
- `createTestContext()` wraps `convex-test`
- `seedTenant()` creates a workspace tenant plus authenticated users
- `asTrustedCaller()` exercises the same hidden trusted-caller path used by MCP tools and server calls

## Vitest Setup

```ts [vitest.config.ts]
import { defineConfig } from 'vitest/config'
import { convexTestConfig } from 'better-convex-nuxt/testing'

export default defineConfig(convexTestConfig())
```

## Create a Test Context

```ts [posts.test.ts]
import { createTestContext } from 'better-convex-nuxt/testing'
import schema from '~/convex/schema'
import { modules } from '~/convex/test.setup'

const ctx = createTestContext({ schema, modules })
```

Keep `convex/test.setup.ts` in your app. The Vite module glob and `vi.mock('./_generated/server', ...)`
must live in consumer code, but the boilerplate is now standardized:

```ts [convex/test.setup.ts]
/// <reference types="vite/client" />

import { vi } from 'vitest'

import { convexServerMock, createConvexTestModules } from 'better-convex-nuxt/testing'

export const modules = createConvexTestModules(
  import.meta.glob('./**/*.ts', {
    eager: false,
  }),
)

vi.mock('./_generated/server', async () => await convexServerMock())
```

## Seed Workspaces and Users

```ts [posts.test.ts]
const team = await ctx.seedTenant({
  name: 'Acme',
  users: {
    alice: { role: 'owner' },
    bob: { role: 'member' },
    carol: { role: 'viewer' },
  },
})

await team.users.alice.mutation(api.posts.create, {
  title: 'Ship the release',
  content: '...',
})
```

Each seeded user gets ready-to-use `query`, `mutation`, and `action` helpers backed by a real
authenticated identity.

## Trusted Callers

```ts [posts.test.ts]
const trustedCaller = ctx.asTrustedCaller({
  userId: team.users.carol.authId,
})

await expect(
  trustedCaller.mutation(api.posts.create, {
    title: 'Nope',
    content: 'Viewers should not create posts.',
  }),
).rejects.toThrow('Forbidden: post.create')
```

Use this when you want to prove that browser calls, MCP calls, and server-to-server calls all hit
the same Convex permission checks.

## Canonical Defaults

`createTestContext({ schema, modules })` is the default path for the repo's canonical
single-workspace shape:

- `workspaces` table with `ownerId`, `createdAt`, and `updatedAt`
- `users` table with `authId`, `email`, `displayName`, `role`, and `workspaceId`
- indexes named `by_auth_id`, `by_email`, and `by_workspace`
- ownership fields that store the auth subject string, not `Id<'users'>`

The optional `tenant` and `users` overrides still exist for non-canonical schemas such as the
playground's organization-based model, but they are now the escape hatch, not the default setup.

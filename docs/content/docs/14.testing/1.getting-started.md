## Overview

`better-convex-nuxt/testing` keeps the test story small on purpose:

- `convexTestConfig()` sets Vitest up for Convex-friendly execution
- `createTestContext()` wraps `convex-test`
- `seedTenant()` creates a tenant plus authenticated users
- `asService()` exercises the same hidden service-auth path used by MCP tools and server calls

## Vitest Setup

```ts [vitest.config.ts]
import { defineConfig } from 'vitest/config'
import { convexTestConfig } from 'better-convex-nuxt/testing'

export default defineConfig(convexTestConfig())
```

## Create a Test Context

```ts [posts.test.ts]
import { createTestContext } from 'better-convex-nuxt/testing'
import schema from '~/convex/schema'

const ctx = createTestContext({ schema })
```

By default the helper looks for Convex modules in your app's `convex/` directory. Pass `modules`
explicitly only if your tests use a non-standard function layout.

## Seed Tenants and Users

```ts [posts.test.ts]
const team = await ctx.seedTenant({
  name: 'Acme',
  users: {
    alice: { role: 'owner' },
    bob: { role: 'member' },
    carol: { role: 'viewer' },
  },
})

await team.users.alice.mutation(api.posts.create, {
  title: 'Ship the release',
  content: '...'
})
```

Each seeded user gets ready-to-use `query`, `mutation`, and `action` helpers backed by a real
authenticated identity.

## Service Auth

```ts [posts.test.ts]
const service = ctx.asService({
  userId: team.users.carol.authId,
  role: 'viewer',
  tenantId: team.id,
})

await expect(
  service.mutation(api.posts.create, {
    title: 'Nope',
    content: 'Viewers should not create posts.',
  }),
).rejects.toThrow('Forbidden: post.create')
```

Use this when you want to prove that browser calls, MCP calls, and server-to-server calls all hit
the same Convex permission checks.

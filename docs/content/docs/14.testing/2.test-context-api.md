## createTestContext

Wraps `convex-test` with higher-level helpers for seeding tenants, users, and trusted callers.

```ts
import { createTestContext } from '@lupinum/trellis/testing'
import schema from '~/convex/schema'
import { modules } from '~/convex/test.setup'

const ctx = createTestContext({ schema, modules })
```

### Options

```ts
interface CreateTestContextOptions<TSchema> {
  schema: TSchema
  modules?: Record<string, () => Promise<unknown>>
  trustedCallerKey?: string
  tenant?: { table?: string; field?: string }
  users?: {
    table?: string
    authField?: string
    roleField?: string
    tenantField?: string
    nameField?: string
    emailField?: string
  }
}
```

| Option             | Default                                                                | Purpose                                               |
| ------------------ | ---------------------------------------------------------------------- | ----------------------------------------------------- |
| `schema`           | **required**                                                           | Your Convex schema definition                         |
| `modules`          | Auto-detected via `import.meta.glob`                                   | Convex function modules for the test runner           |
| `trustedCallerKey` | `process.env.CONVEX_TRUSTED_CALLER_KEY` or `'test-trusted-caller-key'` | Key for trusted caller auth                           |
| `tenant.table`     | `'workspaces'`                                                         | Table name for tenant records                         |
| `tenant.field`     | `'workspaceId'`                                                        | Field name on user records that references the tenant |
| `users.table`      | `'users'`                                                              | Table name for user records                           |
| `users.authField`  | `'authId'`                                                             | Field storing the auth subject string                 |
| `users.roleField`  | `'role'`                                                               | Field storing the user's role                         |

The `tenant` and `users` overrides are escape hatches for non-canonical schemas. Omit them if your schema follows the default workspace model.

### Returned Methods

#### `ctx.raw`

Direct access to the underlying `convex-test` `TestConvex` instance. Use this for low-level operations like `ctx.raw.run()` or `ctx.raw.withIdentity()`.

#### `ctx.seed(table, data)`

Insert a single row into a table. Returns the document `_id`.

```ts
const postId = await ctx.seed('posts', {
  title: 'Hello',
  workspaceId: team.id,
  ownerId: team.users.alice.authId,
})
```

#### `ctx.readAll(table)`

Read all rows from a table. Returns the full document array.

```ts
const allPosts = await ctx.readAll('posts')
expect(allPosts).toHaveLength(1)
```

#### `ctx.seedTenant(options)`

Create a workspace with named, authenticated users in one call. This is the primary setup helper for multi-tenant tests.

```ts
const team = await ctx.seedTenant({
  name: 'Acme',
  users: {
    alice: { role: 'owner' },
    bob: { role: 'member' },
    carol: { role: 'viewer' },
  },
})
```

**What it creates:**

1. A row in the tenant table (`workspaces` by default) with `name`, `slug`, `ownerId`, `createdAt`, `updatedAt`
2. One row per user in the users table with `authId`, `role`, `workspaceId`, `displayName`, `email`, `createdAt`, `updatedAt`

**Canonical defaults per user:**

| Field         | Default                                |
| ------------- | -------------------------------------- |
| `authId`      | `'{slug}-{key}'` (e.g. `'acme-alice'`) |
| `displayName` | Key name (e.g. `'alice'`)              |
| `email`       | `'{slug}-{key}@example.test'`          |

Override any default by passing it explicitly:

```ts
alice: { role: 'owner', email: 'alice@acme.com', authId: 'custom-auth-id' }
```

**Return value:**

```ts
{
  id: string // tenant document _id
  users: {
    alice: {
      id: string // user document _id
      authId: string // auth subject string
      role: 'owner' // the role passed in
      query: Function // authenticated query caller
      mutation: Function // authenticated mutation caller
      action: Function // authenticated action caller
    }
  }
}
```

Each user gets `query`, `mutation`, and `action` helpers backed by `convex-test`'s `withIdentity({ subject })`. Call them like the real API:

```ts
await team.users.alice.mutation(api.posts.create, { title: 'Ship it' })
const posts = await team.users.alice.query(api.posts.list, {})
```

#### `ctx.asTrustedCaller(actor)`

Create a test client that injects trusted-caller fields into every call. Use this to verify that server-to-server and MCP calls hit the same permission checks as browser calls.

```ts
const trustedCaller = ctx.asTrustedCaller({
  userId: team.users.carol.authId,
})

// This goes through the same guards as a browser call
await expect(trustedCaller.mutation(api.posts.create, { title: 'Nope' })).rejects.toThrow(
  'Forbidden',
)
```

The trusted caller client automatically adds `_trustedCallerKey` and `_trustedCaller` fields to every call's args. Your handler's `withTrustedCallerHandler` wrapper strips these and resolves the actor identity.

## convexTestConfig

Merges Convex-friendly defaults into your Vitest config. Sets `environment: 'edge-runtime'` and inlines Convex dependencies.

```ts
import { defineConfig } from 'vitest/config'
import { convexTestConfig } from '@lupinum/trellis/testing'

export default defineConfig(convexTestConfig())
```

Pass your own Vitest config to merge with the defaults:

```ts
export default defineConfig(
  convexTestConfig({
    test: {
      include: ['convex/**/*.test.ts'],
    },
  }),
)
```

## convexServerMock

Factory for `vi.mock('./_generated/server', ...)`. Maps Convex's generated server module to the generic function builders so `convex-test` can intercept calls.

```ts
import { vi } from 'vitest'
import { convexServerMock } from '@lupinum/trellis/testing'

vi.mock('./_generated/server', async () => await convexServerMock())
```

## createConvexTestModules

Wraps `import.meta.glob` results to ensure the `_generated/api` module is included. Required by `convex-test`.

```ts
import { createConvexTestModules } from '@lupinum/trellis/testing'

export const modules = createConvexTestModules(import.meta.glob('./**/*.ts', { eager: false }))
```

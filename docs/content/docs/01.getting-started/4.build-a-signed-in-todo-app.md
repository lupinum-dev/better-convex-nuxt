Add sign-in, one protected query, one protected mutation. At the end you'll have a todo app where each user sees only their own list — and if you sign in as a second account, the first account's todos stay invisible.

The scope stays narrow on purpose. No tenancy, no permission context, no MCP — just the smallest protected app that proves the boundary works. The code follows [`examples/02-auth-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/02-auth-todo).

::note
Do the [First live query](/docs/getting-started/first-live-query) before this page. Protection is easier to understand when you've already seen the unprotected baseline.
::

::tip
If you want the paved Trellis shape, run `npx trellis init my-app --template personal` first. The generated starter puts feature code under `convex/domain/` and permission projection under `convex/permissions/`. This page keeps the flatter example layout because it mirrors `02-auth-todo` line for line.
::

## Before you start

You should already have the installation baseline. For the auth-enabled path, these values must all be set:

- `CONVEX_URL`
- `SITE_URL`
- `BETTER_AUTH_SECRET`

If `npx trellis doctor` isn't clean, stop and fix it first — debugging an auth app on top of a broken baseline is a bad time.

## Build it

::steps{level="3"}

### Enable the module auth path

Start from the auth-ready module config:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

That puts Trellis on the signed-in path from the first run.

### Register Better Auth in Convex

Three files here: install the Better Auth component, create the auth bridge, and register the HTTP routes so Better Auth can serve its callbacks.

```ts [convex/convex.config.ts]
import betterAuth from '@convex-dev/better-auth/convex.config'
import { defineApp } from 'convex/server'

const app = defineApp()

app.use(betterAuth, { name: 'betterAuth' })

export default app
```

```ts [convex/auth.ts]
import { defineAuth } from '@lupinum/trellis/auth'

import { components, internal } from './_generated/api'
import { mutation } from './_generated/server'
import authConfig from './auth.config'

export const { authComponent, createAuth, createUserIfNeeded } = defineAuth(
  { components, internal, mutation, authConfig },
  {
    emailPassword: true,
  },
)

export const { onCreate, onUpdate, onDelete } = authComponent.triggersApi()
```

```ts [convex/http.ts]
import { httpRouter } from 'convex/server'

import { authComponent, createAuth } from './auth'

const http = httpRouter()

authComponent.registerRoutes(http, createAuth)

export default http
```

The `triggersApi()` export is what keeps the `users` table in sync — when Better Auth creates a user, Trellis inserts a matching row; when it updates or deletes one, the row follows. Without this line, sign-in looks fine but the actor resolver below fails with an explicit setup error because the `users` record never appears.

::tip
The official starters scaffold the default auth files if you'd rather not type them by hand.
::

### Resolve the current signed-in user

For this first protected app, Trellis needs one small piece of app-owned identity: "which signed-in user is this?" Later docs call that shape the `actor`. On this page, keep it simpler: resolve the current app user from the mirrored `users` table and return the smallest shape the handlers need.

```ts [convex/auth/actor.ts]
import { getAuth } from '@lupinum/trellis/auth'
import type { GenericActionCtx, GenericMutationCtx, GenericQueryCtx } from 'convex/server'

import type { DataModel } from '../_generated/dataModel'

export type Actor = { kind: 'user'; userId: string } | null

type AuthTodoCtx =
  | GenericQueryCtx<DataModel>
  | GenericMutationCtx<DataModel>
  | GenericActionCtx<DataModel>

export async function getActor(ctx: AuthTodoCtx): Promise<Actor> {
  const auth = await getAuth(ctx)
  if (!auth) return null
  if (!('db' in ctx)) {
    throw new Error('AuthTodo actor resolution requires a query or mutation context.')
  }

  const user = await ctx.db
    .query('users')
    .withIndex('by_auth_id', (q: any) => q.eq('authId', auth.subject))
    .first()

  if (!user) {
    throw new Error(
      [
        \`Expected a Trellis users row for auth subject "\${auth.subject}", but none was found.\`,
        'Ensure convex/auth.ts exports onCreate, onUpdate, and onDelete from authComponent.triggersApi().',
        'If the auth wiring is already correct, ensure auth:createUserIfNeeded has run for this user.',
      ].join(' '),
    )
  }

  return {
    kind: 'user',
    userId: user.authId,
  }
}
```

Then wire that user resolver into Trellis:

```ts [convex/functions.ts]
import { defineTrellis } from '@lupinum/trellis/functions'

import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'
import { getActor } from './auth/actor'

export const { mutation, query, raw } = defineTrellis(
  { query: generatedQuery, mutation: generatedMutation },
  {
    actor: getActor,
  },
)
```

This is the first meaningful boundary: Trellis owns the protected function pipeline, but your app decides what a valid signed-in user looks like.

### Add one signed-in check, one table, two handlers

Define the smallest useful outer check: "is this caller signed in?" Then add a schema and two protected handlers.

```ts [convex/auth/checks.ts]
import { defineGuard } from '@lupinum/trellis/auth'

import type { Actor } from './actor'

export const isAuthenticated = defineGuard<Actor>('Authenticated', (actor) => actor !== null)
```

```ts [convex/schema.ts]
import { defineSchema, defineTable } from 'convex/server'
import { v } from 'convex/values'

export default defineSchema({
  users: defineTable({
    authId: v.string(),
    email: v.optional(v.string()),
    displayName: v.optional(v.string()),
    createdAt: v.number(),
    updatedAt: v.number(),
  }).index('by_auth_id', ['authId']),

  todos: defineTable({
    ownerId: v.string(),
    title: v.string(),
    completed: v.boolean(),
    createdAt: v.number(),
  }).index('by_owner', ['ownerId']),
})
```

```ts [convex/todos.ts]
import { createTodo } from '../shared/schemas/todo'
import { isAuthenticated } from './auth/checks'
import { mutation, query } from './functions'

export const list = query({
  args: {},
  guard: isAuthenticated,
  handler: async (ctx) => {
    const actor = await ctx.actor()

    return await ctx.db
      .query('todos')
      .withIndex('by_owner', (q) => q.eq('ownerId', actor.userId))
      .order('desc')
      .collect()
  },
})

export const create = mutation({
  args: createTodo.args,
  guard: isAuthenticated,
  handler: async (ctx, args) => {
    const actor = await ctx.actor()

    return await ctx.db.insert('todos', {
      ownerId: actor.userId,
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})
```

One guard protects both handlers. Ownership is explicit in the stored row — which is what makes the isolation check below possible.

### Build the Nuxt page

Use `useConvexAuth` for session state, `useConvexAuthActions` for sign-in transitions, `useConvexQuery` for the protected list, and `useConvexMutation` for creation. Pass `undefined` as query args when the user is not signed in yet. That tells Trellis to skip the subscription entirely.

```vue [pages/index.vue]
<script setup lang="ts">
import * as z from 'zod'

import { api } from '#trellis/api'

const { client, isAuthenticated, isPending, user, signOut } = useConvexAuth()
const authAction = useConvexAuthActions()

const title = ref('')
const createTodo = useConvexMutation(api.todos.create)

const todoArgs = computed(() => (isAuthenticated.value ? {} : undefined))

const { data: todos, pending: todosPending, error: todosError } = await useConvexQuery(api.todos.list, todoArgs)

const signInSchema = z.object({
  email: z.string().email(),
  password: z.string().min(1),
})

async function handleSignIn(payload: { email: string; password: string }) {
  if (!client) throw new Error('Auth client unavailable.')
  await authAction.execute(() => client.signIn.email(payload), { redirectTo: '/' })
}

async function handleCreateTodo() {
  await createTodo({ title: title.value })
  title.value = ''
}
</script>
```

::note
The complete page in the example also covers sign-up, sign-out, loading states, toggle, and delete. Keep your first pass smaller if you don't need those yet — adding them later is mechanical.

## What success looks like

You're done when you can:

1. create an account;
2. sign in without writing auth glue code in the page;
3. see an empty personal list;
4. create a todo and see it appear;
5. sign in as a different account and confirm the first account's todo **does not appear**.

::caution
That last check is the whole point of this tutorial. If the second account can see the first account's todos, protection is happening in the UI only — which isn't protection. The rest of this tutorial assumes you ran it.
::

## Common pitfalls

- Auth enabled but `SITE_URL` or `BETTER_AUTH_SECRET` missing.
- Better Auth routes defined in `convex/auth.ts` but never registered in `convex/http.ts`.
- Forgetting the `triggersApi()` export — sign-in works, but the `users` row never lands and current-user resolution fails with a setup error.
- The page subscribes to the protected query before auth is ready (fix: gate the args on `isAuthenticated`).
- The handler writes rows but doesn't store `ownerId`, so the list query can't scope correctly.

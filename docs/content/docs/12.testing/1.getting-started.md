Use this page when you need the supported testing entry points, not just the repo’s raw test tree.

## Main Helpers

- `convexTestConfig(...)`
- `createConvexTestModules(...)`
- `createTestContext(...)`

## Vitest Setup

The normal Trellis-friendly Vitest setup is:

```ts
import { convexTestConfig } from '@lupinum/trellis/testing'

export default convexTestConfig({
  test: {
    include: ['convex/**/*.test.ts'],
  },
})
```

That gives you the Convex-friendly edge runtime setup plus the generated-server mock bridge used by Trellis apps.

## Create A Test Context

The high-value helper for protected backend behavior is:

```ts
const ctx = createTestContext({
  schema,
  modules,
})
```

From there you can:

- seed data
- seed tenants
- get authenticated user callers
- forward explicit principals

## What This Is For

Use these helpers when you want realistic tests against actual protected handlers instead of mocks that bypass the Trellis runtime model.

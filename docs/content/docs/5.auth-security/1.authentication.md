Authentication in Trellis starts with Better Auth for transport identity and then refreshes Convex auth state so protected handlers can resolve the real caller.

Trellis auth is the runtime path that keeps Better Auth, Convex, and Nuxt in sync. It is not only a login form concern. It covers the auth proxy route, SSR-aware auth state, auth refresh, and the boundary between signed-in and signed-out UI.

## Start With The Auth-Enabled Module Config

The baseline config is:

```ts
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

When auth is enabled, Trellis expects the Better Auth bridge to exist on the Convex side and the core auth environment variables to be configured.

## Wire Better Auth Through Convex

Create the auth bridge in `convex/auth.ts`:

```ts
import { defineAuth } from '@lupinum/trellis/auth'

import { components, internal } from './_generated/api'
import { mutation } from './_generated/server'
import authConfig from './auth.config'

export const { authComponent, createAuth, createUserIfNeeded } = defineAuth(
  { components, internal, mutation, authConfig },
  {
    emailPassword: true,
  },
)
```

Then register the routes in `convex/http.ts`:

```ts
import { httpRouter } from 'convex/server'

import { authComponent, createAuth } from './auth'

const http = httpRouter()

authComponent.registerRoutes(http, createAuth)

export default http
```

If you want the default file set instead of writing them by hand, run:

```bash
npx trellis init auth
```

## Keep The Required Environment In Place

For the normal auth path, keep these configured:

- `CONVEX_URL`
- `SITE_URL`
- `BETTER_AUTH_SECRET`

If auth behavior is failing, run:

```bash
npx trellis doctor
```

That is the fastest way to catch missing env, missing route registration, or origin mismatches before debugging page code.

## Use `useConvexAuth()` For Runtime State

`useConvexAuth()` is the main auth composable:

```ts
const { user, isAuthenticated, isPending, isAnonymous, client, signOut, refreshAuth } =
  useConvexAuth()
```

It gives you:

- the current user
- authenticated and anonymous state
- pending state during auth initialization
- the Better Auth client
- `signOut()`
- `refreshAuth()` when you need to re-synchronize Convex auth state

The important behavior is that auth state is SSR-aware and hydrated into the client without forcing every page to invent its own auth bootstrap logic.

## Use Auth Boundary Components In Templates

Trellis ships three simple auth boundary components:

- `ConvexAuthLoading`
- `ConvexUnauthenticated`
- `ConvexAuthenticated`

They are the cleanest way to split UI by auth state:

```vue
<ConvexAuthLoading>
  <p>Checking authentication...</p>
</ConvexAuthLoading>

<ConvexUnauthenticated>
  <LoginForm />
</ConvexUnauthenticated>

<ConvexAuthenticated>
  <Dashboard />
</ConvexAuthenticated>
```

Use them when template readability matters more than repeating `v-if="isAuthenticated && !isPending"` in multiple places.

## Prefer `useConvexAuthActions()` For Sign-In And Sign-Up

The recommended path for auth flows is `useConvexAuthActions()`:

```ts
const { client } = useConvexAuth()
const authAction = useConvexAuthActions()

async function handleSignIn(email: string, password: string) {
  if (!client) throw new Error('Auth client unavailable.')

  await authAction.execute(() => client.signIn.email({ email, password }), {
    redirectTo: '/',
  })
}
```

That wrapper:

- executes the Better Auth client call
- refreshes Convex auth state afterwards
- tracks status, pending, data, and error
- handles redirect behavior

That is a better default than calling the Better Auth client directly and remembering to refresh Convex auth state yourself every time.

## Basic Usage

`useConvexMutation` returns a **callable function** with reactive state properties attached. Call it directly to execute the mutation.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const createPost = useConvexMutation(api.posts.create)

// The return value IS the function — call it directly
await createPost({ title: 'Hello', body: 'World' })
</script>
```

The composable accepts a Convex mutation function reference and an optional options object. It returns a function you invoke with the mutation arguments.

## Reactive State

Every mutation callable exposes reactive state properties so you can bind directly in templates without managing your own refs.

::field-group
  ::field{name="data" type="Ref<Result | undefined>"}
  Result data from the last successful call. Resets to `undefined` when `reset()` is called.
  ::

  ::field{name="status" type="ComputedRef<MutationStatus>"}
  Current lifecycle status. One of `'idle'`, `'pending'`, `'success'`, or `'error'`.
  ::

  ::field{name="pending" type="ComputedRef<boolean>"}
  Shorthand for `status === 'pending'`. Useful for disabling buttons and showing spinners.
  ::

  ::field{name="error" type="Ref<Error | null>"}
  Error from the last call attempt. Set automatically when the mutation throws. The error is a `ConvexCallError` instance with additional properties like `code` and `status`.
  ::

  ::field{name="reset()" type="() => void"}
  Reset all state back to idle. Clears `data`, `error`, and sets `status` to `'idle'`.
  ::
::

## Options

Pass an options object as the second argument to configure callbacks and optimistic updates.

::field-group
  ::field{name="optimisticUpdate" type="(ctx: OptimisticContext, args: Args) => void"}
  Called immediately before the mutation is sent to the server. Use the `ctx` object to update local query results for instant UI feedback. Automatically rolled back when the server response arrives. See [Optimistic Updates](/docs/mutations/optimistic-updates) for the full API.
  ::

  ::field{name="onSuccess" type="(result: Result, args: Args) => void"}
  Called after the mutation completes successfully. Receives the return value and the original arguments. Errors thrown inside this callback are logged and ignored.
  ::

  ::field{name="onError" type="(error: Error, args: Args) => void"}
  Called after the mutation fails. Receives the error and the original arguments. Errors thrown inside this callback are logged and ignored.
  ::
::

## Complete Example

A form that creates a post with loading state, error display, and success feedback.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const title = ref('')
const body = ref('')

const createPost = useConvexMutation(api.posts.create, {
  onSuccess: (result, args) => {
    // Clear the form after success
    title.value = ''
    body.value = ''
  },
  onError: (error, args) => {
    console.error('Failed to create post:', error.message)
  },
})

async function handleSubmit() {
  try {
    await createPost({ title: title.value, body: body.value })
  } catch {
    // Error is already tracked in createPost.error
  }
}
</script>

<template>
  <form @submit.prevent="handleSubmit">
    <input v-model="title" placeholder="Post title" />
    <textarea v-model="body" placeholder="Write something..." />

    <button type="submit" :disabled="createPost.pending.value">
      {{ createPost.pending.value ? 'Creating...' : 'Create Post' }}
    </button>

    <p v-if="createPost.status.value === 'error'" class="text-red-500">
      {{ createPost.error.value?.message }}
    </p>

    <p v-if="createPost.status.value === 'success'" class="text-green-500">
      Post created!
    </p>
  </form>
</template>
```

## Resetting State

After a successful mutation, the status remains `'success'` until the next call or an explicit reset. Use `reset()` to clear the state, for example when navigating away or dismissing a success message.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const deleteTask = useConvexMutation(api.tasks.remove)

async function handleDelete(id: string) {
  await deleteTask({ id })

  // Reset after a brief delay so the success message shows
  setTimeout(() => deleteTask.reset(), 2000)
}
</script>
```

## Multiple Mutations

Each `useConvexMutation` call creates an independent instance with its own state. You can use multiple mutations in the same component without them interfering.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const createTask = useConvexMutation(api.tasks.create)
const updateTask = useConvexMutation(api.tasks.update)
const deleteTask = useConvexMutation(api.tasks.remove)
</script>

<template>
  <button :disabled="createTask.pending.value" @click="createTask({ text: 'New task' })">
    Add Task
  </button>
</template>
```

::note
Mutations are **client-only**. They do not execute during SSR. The `useConvexMutation` composable must be called within a client-side Vue component setup context.
::

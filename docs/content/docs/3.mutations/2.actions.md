Actions differ from mutations: they can call external APIs, run longer computations, and perform side effects that aren't possible in queries or mutations.

## Basic Usage

```vue
<script setup lang="ts">
import { api } from '#convex/api'

const sendEmail = useConvexAction(api.emails.send)

async function handleSend() {
  try {
    await sendEmail({
      to: 'user@example.com',
      subject: 'Welcome!',
      body: 'Thanks for signing up.',
    })
  } catch {
    // error is automatically tracked
  }
}
</script>

<template>
  <button :disabled="sendEmail.pending.value" @click="handleSend">
    {{ sendEmail.pending.value ? 'Sending...' : 'Send Email' }}
  </button>
  <p v-if="sendEmail.status.value === 'error'" class="error">
    {{ sendEmail.error.value?.message }}
  </p>
  <p v-if="sendEmail.status.value === 'success'" class="success">Email sent!</p>
</template>
```

## API Reference

### Parameters

| Parameter | Type                          | Description                      |
| --------- | ----------------------------- | -------------------------------- |
| `action`  | `FunctionReference<"action">` | Convex action function reference |
| `options` | `UseConvexActionOptions`      | Optional configuration           |

### Options

| Option      | Type                     | Default | Description                  |
| ----------- | ------------------------ | ------- | ---------------------------- |
| `onSuccess` | `(result, args) => void` | -       | Called after action succeeds |
| `onError`   | `(error, args) => void`  | -       | Called after action fails    |

::callout{icon="i-lucide-info" to="/docs/advanced/logging"}
Enable module-level logging via the `logging` config option for debugging actions.
::

### Returns

| Property  | Type                                    | Description                                         |
| --------- | --------------------------------------- | --------------------------------------------------- |
| callable  | `(args) => Promise<Result>`             | Execute the action                                  |
| `safe`    | `(args) => Promise<CallResult<Result>>` | Execute without throwing (`{ ok, data/error }`)     |
| `data`    | `Ref<Result \| undefined>`              | Result from last successful action                  |
| `status`  | `ComputedRef<ConvexCallStatus>`         | `'idle'` \| `'pending'` \| `'success'` \| `'error'` |
| `pending` | `ComputedRef<boolean>`                  | True while action is in flight                      |
| `error`   | `Ref<Error \| null>`                    | Error from last attempt                             |
| `reset`   | `() => void`                            | Clear error and status                              |

### Callback Hooks

```ts
const generateReport = useConvexAction(api.reports.generate, {
  onSuccess: (result, args) => {
    console.log('Report generated', result, args)
  },
  onError: (error, args) => {
    console.error('Report generation failed', error.message, args)
  },
})
```

## When to Use Actions

Use actions when you need to:

| Use Case                         | Example                                             |
| -------------------------------- | --------------------------------------------------- |
| **Call external APIs**           | Payment processing, email sending, AI services      |
| **Long computations**            | Data processing, file generation, reports           |
| **Non-deterministic operations** | Generating random values, getting current time      |
| **Side effects**                 | Sending notifications, logging to external services |

::callout{icon="i-lucide-info"}
Actions run in a Node.js environment and can use `fetch`, npm packages, and other Node APIs. They don't have direct database access like mutations.
::

## Patterns

### API Integration

```vue
<script setup lang="ts">
const generateImage = useConvexAction(api.ai.generateImage)

const prompt = ref('')

async function handleGenerate() {
  await generateImage({ prompt: prompt.value })
}
</script>

<template>
  <div>
    <input v-model="prompt" placeholder="Describe an image..." />
    <button :disabled="generateImage.pending.value" @click="handleGenerate">
      {{ generateImage.pending.value ? 'Generating...' : 'Generate' }}
    </button>

    <img v-if="generateImage.data.value" :src="generateImage.data.value" alt="Generated image" />
  </div>
</template>
```

### File Processing

```vue
<script setup lang="ts">
const processFile = useConvexAction(api.files.process)

async function handleUpload(file: File) {
  const base64 = await fileToBase64(file)
  await processFile({ fileName: file.name, content: base64 })
}
</script>

<template>
  <input type="file" @change="handleUpload($event.target.files[0])" />
  <div v-if="processFile.pending.value">Processing file...</div>
  <div v-if="processFile.status.value === 'success'">
    Processed! Result: {{ processFile.data.value }}
  </div>
</template>
```

### Payment Processing

The action must authenticate the caller, accept only a server-defined product key, map that key to a provider price, and build success and cancellation URLs from the validated server application origin. Never accept an amount, provider price ID, or callback URL from the browser. Validate the action result again at the navigation sink so a compromised response cannot redirect to another origin.

```ts [convex/payments.ts]
'use node'

import Stripe from 'stripe'
import { v } from 'convex/values'
import { action } from './_generated/server'

const stripeSecret = process.env.STRIPE_SECRET_KEY
const monthlyPriceId = process.env.STRIPE_PRO_MONTHLY_PRICE_ID
const applicationOrigin = process.env.APPLICATION_ORIGIN

if (!stripeSecret || !monthlyPriceId || !applicationOrigin) {
  throw new Error('Checkout server configuration is incomplete')
}

const parsedApplicationOrigin = new URL(applicationOrigin)
if (
  parsedApplicationOrigin.protocol !== 'https:' ||
  parsedApplicationOrigin.origin !== applicationOrigin
) {
  throw new Error('APPLICATION_ORIGIN must be one exact HTTPS origin')
}

const stripe = new Stripe(stripeSecret)
const plans = {
  'pro-monthly': monthlyPriceId,
} as const

export const createCheckoutSession = action({
  args: { plan: v.literal('pro-monthly') },
  handler: async (ctx, args) => {
    const identity = await ctx.auth.getUserIdentity()
    if (!identity) throw new Error('Not authenticated')

    const session = await stripe.checkout.sessions.create({
      mode: 'subscription',
      line_items: [{ price: plans[args.plan], quantity: 1 }],
      success_url: new URL('/billing/success', applicationOrigin).href,
      cancel_url: new URL('/billing/cancel', applicationOrigin).href,
      client_reference_id: identity.subject,
    })

    if (!session.url) throw new Error('Checkout is unavailable')

    const target = new URL(session.url)
    if (
      target.protocol !== 'https:' ||
      target.origin !== 'https://checkout.stripe.com' ||
      target.username ||
      target.password
    ) {
      throw new Error('Checkout provider returned an invalid destination')
    }

    return { url: target.href }
  },
})
```

```vue
<script setup lang="ts">
const createCheckout = useConvexAction(api.payments.createCheckoutSession)

async function handleCheckout() {
  try {
    const { url } = await createCheckout({ plan: 'pro-monthly' })
    const target = new URL(url)

    if (
      target.protocol !== 'https:' ||
      target.origin !== 'https://checkout.stripe.com' ||
      target.username ||
      target.password
    ) {
      throw new Error('Checkout returned an invalid destination')
    }

    window.location.assign(target.href)
  } catch {
    // Show error
  }
}
</script>

<template>
  <button :disabled="createCheckout.pending.value" @click="handleCheckout">
    {{ createCheckout.pending.value ? 'Redirecting...' : 'Checkout' }}
  </button>
  <p v-if="createCheckout.error.value" class="error">{{ createCheckout.error.value.message }}</p>
</template>
```

### With Status Display

```vue
<script setup lang="ts">
const generateReport = useConvexAction(api.reports.generate)

const downloadPath = computed(() => {
  const reportId = generateReport.data.value?.reportId
  return reportId ? `/api/reports/${encodeURIComponent(reportId)}` : null
})
</script>

<template>
  <div class="status-card">
    <template v-if="generateReport.status.value === 'idle'">
      <button @click="generateReport({ month: 'january' })">Generate Report</button>
    </template>

    <template v-else-if="generateReport.status.value === 'pending'">
      <div class="spinner" />
      <p>Generating report...</p>
    </template>

    <template v-else-if="generateReport.status.value === 'success'">
      <p>Report ready!</p>
      <a v-if="downloadPath" :href="downloadPath" download>Download</a>
      <button @click="generateReport.reset">Generate Another</button>
    </template>

    <template v-else-if="generateReport.status.value === 'error'">
      <p class="error">Failed: {{ generateReport.error.value?.message }}</p>
      <button @click="generateReport.reset">Try Again</button>
    </template>
  </div>
</template>
```

The action returns an opaque `reportId`, not a remote or signed download URL. The same-origin `/api/reports/:reportId` route must authenticate the current request, reauthorize access to that exact report, and only then stream the file. Authorization is checked again on every download; possession of an action result is not authorization.

## Comparison with Mutations

| Aspect                 | useConvexMutation   | useConvexAction            |
| ---------------------- | ------------------- | -------------------------- |
| **Database access**    | Direct via `ctx.db` | Via internal mutations     |
| **External APIs**      | Not allowed         | Allowed                    |
| **Execution time**     | Should be fast      | Can be longer              |
| **Optimistic updates** | Supported           | Not supported              |
| **Use case**           | Data changes        | Side effects, integrations |

### Example: Using Both

```vue
<script setup lang="ts">
import type { Id } from '#convex/dataModel'

const props = defineProps<{ cartId: Id<'carts'> }>()
const purchase = useConvexAction(api.orders.purchase)

async function handlePurchase() {
  await purchase({ cartId: props.cartId })
}
</script>
```

The purchase action authenticates the caller, loads the canonical cart and server-owned prices through an internal query, makes an idempotent provider request for the amount it calculated, and records the order through an internal mutation. The browser supplies only the cart identifier; it does not orchestrate the charge/write invariant or choose the amount.

## TypeScript

Full type inference from your Convex schema:

```ts
// Args and return type are inferred
const sendEmail = useConvexAction(api.emails.send)

// Type error if args don't match
await sendEmail({ wrong: 'args' })

// data is typed based on action return
console.log(sendEmail.data.value?.messageId) // typed correctly
```

## Common Mistakes

### 1. Using actions for database operations

```ts
// WRONG: Use mutation for database writes
const updateProfileAction = useConvexAction(api.users.updateProfile)

// RIGHT: Use mutation
const updateProfile = useConvexMutation(api.users.updateProfile)
```

### 2. Expecting real-time updates

```ts
// Actions don't trigger query subscriptions automatically
// If your action affects data, call a mutation to update the database

// In your Convex action:
export const processAndSave = action({
  handler: async (ctx, args) => {
    const result = await processExternally(args)
    // Call mutation to save and trigger subscriptions
    await ctx.runMutation(internal.data.save, { result })
  },
})
```

### 3. Not handling long-running operations

```vue
<!-- WRONG: No feedback for long operations -->
<script setup>
const processData = useConvexAction(api.processing.run)
</script>

<template>
  <button @click="processData({ data: largeData })">Process</button>
</template>

<!-- RIGHT: Show progress -->
<script setup>
const processData = useConvexAction(api.processing.run)
</script>

<template>
  <button :disabled="processData.pending.value" @click="processData({ data: largeData })">
    <span v-if="processData.pending.value">Processing... This may take a minute</span>
    <span v-else>Process Data</span>
  </button>
</template>
```

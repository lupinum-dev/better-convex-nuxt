## Basic Usage

`useConvexAction` has the same API shape as `useConvexMutation` but is designed for Convex actions -- server-side functions that can call third-party APIs, run longer computations, and perform side effects.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const sendEmail = useConvexAction(api.emails.send)

await sendEmail({ to: 'user@example.com', subject: 'Welcome!' })
</script>
```

Like mutations, the return value is a **callable function** with reactive state properties attached.

## Reactive State

The action callable exposes the same reactive state properties as mutations.

::field-group
  ::field{name="data" type="Ref<Result | undefined>"}
  Result data from the last successful call.
  ::

  ::field{name="status" type="ComputedRef<MutationStatus>"}
  Current lifecycle status: `'idle'` | `'pending'` | `'success'` | `'error'`.
  ::

  ::field{name="pending" type="ComputedRef<boolean>"}
  Shorthand for `status === 'pending'`.
  ::

  ::field{name="error" type="Ref<Error | null>"}
  Error from the last call attempt, or `null`.
  ::

  ::field{name="reset()" type="() => void"}
  Reset all state back to idle.
  ::
::

## Options

::field-group
  ::field{name="onSuccess" type="(result: Result, args: Args) => void"}
  Called after the action completes successfully. Errors thrown inside this callback are logged and ignored.
  ::

  ::field{name="onError" type="(error: Error, args: Args) => void"}
  Called after the action fails. Errors thrown inside this callback are logged and ignored.
  ::
::

::warning
Actions do **not** support the `optimisticUpdate` option. Since actions run arbitrary server-side code (not transactional database operations), there is no query result to optimistically update. Use `useConvexMutation` when you need optimistic updates.
::

## When to Use Actions vs Mutations

| Use Case | Composable |
|---|---|
| Create, update, or delete database records | `useConvexMutation` |
| Call a third-party API (Stripe, SendGrid, etc.) | `useConvexAction` |
| Process or transform files | `useConvexAction` |
| Run longer computations | `useConvexAction` |
| Fan out to multiple mutations | `useConvexAction` |
| Operations needing optimistic updates | `useConvexMutation` |

Convex mutations are transactional and deterministic. Actions are for everything else -- any work that involves external services, non-deterministic logic, or long-running processes.

## Examples

### Sending an Email

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const sendEmail = useConvexAction(api.emails.send)

async function handleSend() {
  try {
    await sendEmail({
      to: 'user@example.com',
      subject: 'Your invoice',
      body: 'Please find your invoice attached.',
    })
  } catch {
    // Error tracked in sendEmail.error
  }
}
</script>

<template>
  <button :disabled="sendEmail.pending.value" @click="handleSend">
    {{ sendEmail.pending.value ? 'Sending...' : 'Send Email' }}
  </button>

  <p v-if="sendEmail.status.value === 'success'" class="text-green-500">
    Email sent!
  </p>
  <p v-if="sendEmail.status.value === 'error'" class="text-red-500">
    {{ sendEmail.error.value?.message }}
  </p>
</template>
```

### Generating a PDF

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const generatePdf = useConvexAction(api.reports.generatePdf)

async function handleGenerate(reportId: string) {
  const result = await generatePdf({ reportId })
  // result.url contains the download URL
  window.open(result.url, '_blank')
}
</script>

<template>
  <button :disabled="generatePdf.pending.value" @click="handleGenerate(report._id)">
    {{ generatePdf.pending.value ? 'Generating...' : 'Download PDF' }}
  </button>
</template>
```

### Calling an External API

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const geocode = useConvexAction(api.maps.geocode, {
  onSuccess: (result) => {
    map.value?.flyTo({ center: [result.lng, result.lat] })
  },
  onError: (error) => {
    toast.error(`Geocoding failed: ${error.message}`)
  },
})

async function handleSearch(address: string) {
  await geocode({ address })
}
</script>
```

::note
Actions are **client-only**. They do not execute during SSR. The `useConvexAction` composable must be called within a client-side Vue component setup context.
::

## Error Flow

When a mutation or action throws on the server, the following happens automatically:

1. The error is caught and normalized into a `ConvexCallError` instance
2. The `error` ref is set to the error object
3. The `status` becomes `'error'`
4. The `onError` callback fires (if provided)
5. The error is re-thrown from the `await` call so `try/catch` also works

## The `error` Ref

Every mutation and action callable exposes a reactive `error` ref. This is the simplest way to show errors in templates.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const createPost = useConvexMutation(api.posts.create)

async function handleSubmit(title: string) {
  try {
    await createPost({ title })
  } catch {
    // Error is already tracked in createPost.error
  }
}
</script>

<template>
  <form @submit.prevent="handleSubmit('My Post')">
    <button type="submit" :disabled="createPost.pending.value">Create</button>

    <div v-if="createPost.error.value" class="text-red-500">
      <p>{{ createPost.error.value.message }}</p>
      <button @click="createPost.reset()">Dismiss</button>
    </div>
  </form>
</template>
```

The `error` ref is cleared automatically when the next call starts, or when you call `reset()`.

## The `onError` Callback

Use the `onError` option for side effects like showing a toast notification. The callback receives the error and the original arguments.

```ts
const createPost = useConvexMutation(api.posts.create, {
  onError: (error, args) => {
    toast.error(`Failed to create "${args.title}": ${error.message}`)
  },
})
```

::note
Errors thrown inside `onError` are caught, logged, and ignored. This prevents callback errors from masking the original error.
::

## try/catch Pattern

Since the callable re-throws errors, you can use standard `try/catch` for imperative error handling.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const createPost = useConvexMutation(api.posts.create)

async function handleSubmit(title: string) {
  try {
    const postId = await createPost({ title })
    navigateTo(`/posts/${postId}`)
  } catch (err) {
    if (err instanceof ConvexCallError) {
      // Access structured error properties
      console.error('Code:', err.code)
      console.error('Status:', err.status)
      console.error('Function:', err.functionPath)
    }
  }
}
</script>
```

::tip
Both the `error` ref and `try/catch` work simultaneously. The `error` ref is reactive and ideal for template binding. `try/catch` is better for control flow decisions like navigation or retries.
::

## ConvexCallError

All errors from mutations and actions are normalized into `ConvexCallError` instances. This class extends the native `Error` with additional properties from Convex.

::field-group
  ::field{name="message" type="string"}
  The human-readable error message from Convex.
  ::

  ::field{name="code" type="string | undefined"}
  Convex error code, if available. Examples: `'LIMIT_CALLS'`, `'UNAUTHENTICATED'`.
  ::

  ::field{name="status" type="number | undefined"}
  HTTP status code, if available. For example, `401` or `403` for auth errors.
  ::

  ::field{name="functionPath" type="string | undefined"}
  The Convex function path that raised the error. For example, `'posts:create'`.
  ::

  ::field{name="operation" type="string | undefined"}
  The operation type: `'mutation'` or `'action'`.
  ::

  ::field{name="isConvexCallError" type="true"}
  Type brand for `instanceof` checks without importing the class. Always `true`.
  ::
::

You can check for `ConvexCallError` with either `instanceof` or the type brand:

::code-group
```ts [instanceof]
import { ConvexCallError } from '#imports'

try {
  await createPost({ title })
} catch (err) {
  if (err instanceof ConvexCallError) {
    console.log(err.code, err.status)
  }
}
```

```ts [Type brand]
try {
  await createPost({ title })
} catch (err) {
  if (err && typeof err === 'object' && 'isConvexCallError' in err) {
    console.log(err.code, err.status)
  }
}
```
::

## Unauthorized Error Hook

When a mutation or action returns a `401` or `403` error (or an error with an `UNAUTHENTICATED` / `FORBIDDEN` code), the module automatically fires the `convex:unauthorized` Nuxt hook. This happens in addition to the normal error flow.

You can listen for this hook in a Nuxt plugin to handle unauthorized errors globally -- for example, redirecting to a login page or refreshing an expired token.

```ts
// plugins/convex-auth.client.ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('convex:unauthorized', ({ error, source, functionName, redirectTo }) => {
    // source: 'mutation' | 'action' | 'query'
    // functionName: the Convex function that failed
    // redirectTo: configured redirect path from module options
    console.warn(`Unauthorized ${source} call to ${functionName}`)
    navigateTo(redirectTo)
  })
})
```

::note
`convex:unauthorized` is one of several runtime hooks exposed by the module. Other hooks include mutation/action success and error hooks, `convex:connection:changed`, and `convex:auth:changed`.
See the full matrix in [Runtime Hooks](/docs/api-reference/runtime-hooks).
::

::note
The `convex:unauthorized` hook fires automatically based on module configuration. You do not need to manually detect auth errors -- the module handles the detection for errors with status `401`/`403` or codes containing `UNAUTH` or `FORBIDDEN`.
::

## Template Error Display Pattern

A reusable pattern for showing mutation errors in templates with a dismiss action.

```vue
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const updateProfile = useConvexMutation(api.users.updateProfile)

async function handleSave() {
  try {
    await updateProfile({
      name: name.value,
      bio: bio.value,
    })
  } catch {
    // Handled by template binding
  }
}
</script>

<template>
  <form @submit.prevent="handleSave">
    <!-- form fields -->

    <button type="submit" :disabled="updateProfile.pending.value">
      {{ updateProfile.pending.value ? 'Saving...' : 'Save Changes' }}
    </button>

    <div
      v-if="updateProfile.status.value === 'error'"
      class="bg-red-50 border border-red-200 rounded p-4 mt-4"
    >
      <p class="text-red-800 font-medium">Something went wrong</p>
      <p class="text-red-600 text-sm mt-1">
        {{ updateProfile.error.value?.message }}
      </p>
      <button
        class="text-red-600 underline text-sm mt-2"
        @click="updateProfile.reset()"
      >
        Dismiss
      </button>
    </div>

    <div
      v-if="updateProfile.status.value === 'success'"
      class="bg-green-50 border border-green-200 rounded p-4 mt-4"
    >
      <p class="text-green-800">Profile updated!</p>
    </div>
  </form>
</template>
```

## Prerequisites

Before you begin, make sure you have the following:

::field-group
  ::field{name="Node.js" type="required"}
  Node.js 18 or later. You can check your version with `node -v`.
  ::
  ::field{name="Convex Account" type="required"}
  A free Convex account. [Sign up at convex.dev](https://www.convex.dev/) if you do not have one.
  ::
  ::field{name="Nuxt Project" type="required"}
  A Nuxt 4+ project. If you need to create one, run `npx nuxi@latest init my-app`.
  ::
::

## Installation

::steps

### Install the module

Add `better-convex-nuxt` and `convex` to your project:

::code-group

```bash [pnpm]
pnpm add better-convex-nuxt convex
```

```bash [yarn]
yarn add better-convex-nuxt convex
```

```bash [npm]
npm install better-convex-nuxt convex
```

```bash [bun]
bun add better-convex-nuxt convex
```

::

### Add the module to your Nuxt config

Register the module in `nuxt.config.ts` and configure the `convex` block. At minimum, you only need to add the module -- the Convex URL is read from environment variables automatically.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    // Auth is enabled by default. Set to false if you
    // want to use Convex without authentication.
    auth: false,
  },
})
```

::note
When `auth` is omitted or set to `true`, the module sets up a full authentication flow with [Better Auth](https://www.better-auth.com/) including SSR token exchange, route protection, and sign-in/sign-out composables. See the [Authentication guide](/docs/auth-security/authentication) for details.
::

### Initialize Convex

If this is a new project, initialize Convex in your project root:

```bash
npx convex dev
```

This command:
- Creates a `convex/` directory with your backend functions
- Generates a `.env.local` file containing your `CONVEX_URL`
- Starts the Convex dev server and watches for changes

::tip
If you already have a Convex project, run `npx convex dev` to connect to it and generate the `.env.local` file.
::

### Configure environment variables

The `npx convex dev` command creates a `.env.local` file with your deployment URL. Verify it contains:

```bash [.env.local]
CONVEX_URL=https://your-deployment-123.convex.cloud
```

The module reads this value automatically -- it checks both `CONVEX_URL` and `NUXT_PUBLIC_CONVEX_URL` environment variables at startup.

::warning
Do not commit `.env.local` to version control. It is already in `.gitignore` by default for Nuxt projects.
::

### Start the dev server

Run the Nuxt dev server with the `--dotenv` flag to load your `.env.local` file:

```bash
nuxt dev --dotenv .env.local
```

::tip
For convenience, add a script to your `package.json`:

```json [package.json]
{
  "scripts": {
    "dev": "nuxt dev --dotenv .env.local"
  }
}
```
::

You also need Convex running in a separate terminal:

```bash
npx convex dev
```

::

## Verify the setup

To confirm everything is working, create a simple query. Add a file to your `convex/` directory:

```ts [convex/hello.ts]
import { query } from './_generated/server'

export const greet = query({
  args: {},
  handler: async () => {
    return 'Hello from Convex!'
  },
})
```

Then use it in a page:

```vue [app/pages/index.vue]
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const { data: greeting } = await useConvexQuery(api.hello.greet)
</script>

<template>
  <div>
    <p>{{ greeting }}</p>
  </div>
</template>
```

If you see "Hello from Convex!" rendered on the page, you are all set.

## Module options reference

The `convex` block in `nuxt.config.ts` accepts the following top-level options:

::field-group
  ::field{name="url" type="string"}
  Convex deployment URL. Auto-read from `CONVEX_URL` or `NUXT_PUBLIC_CONVEX_URL` environment variables.
  ::
  ::field{name="siteUrl" type="string"}
  Convex site URL for HTTP Actions. Auto-derived from `url` by replacing `.convex.cloud` with `.convex.site`.
  ::
  ::field{name="auth" type="boolean | AuthOptions" default="true"}
  Enable authentication. Pass `true` for defaults, `false` to disable, or an object for fine-grained control.
  ::
  ::field{name="query" type="QueryDefaults"}
  Default behavior for query composables. `{ server: true, subscribe: true }` by default.
  ::
  ::field{name="logging" type="false | 'info' | 'debug'" default="false"}
  Enable module logging. Set to `'info'` for everyday use or `'debug'` for detailed timing information.
  ::
::

Start here if you are evaluating Trellis or wiring it into a Nuxt app for the first time.

Trellis is not a replacement for Convex or Better Auth. It is the module that keeps Nuxt, Convex, auth, permissions, operations, and MCP on one app-owned backend model.

## Pick The Right First Path

Choose the smallest path that matches the app you are actually building.

| Starting point | Use it when | What to read first |
| --- | --- | --- |
| Public-only app | You want SSR, live queries, and mutations with no sign-in flow yet. | [`01-public-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/01-public-todo) |
| Personal auth app | You need signed-in users and app-owned actors, but no tenant model yet. | [`02-auth-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/02-auth-todo) |
| Workspace app | You need roles, permissions, tenant isolation, or MCP tools. | [`03-team-workspace`](https://github.com/lupinum-dev/trellis/tree/main/examples/03-team-workspace) |

If you are unsure, start with the public-only path and add auth later. That keeps the first success path narrow and makes each new layer obvious.

## What Trellis Owns

Trellis gives you one execution model across browser, server, and agent callers:

- principal resolution
- optional `authenticated` gate
- actor resolution
- guard and authorization boundaries
- operation flows for destructive work
- correlated observability events around those decisions

Your data model, checks, actor rules, and business behavior still live in app code.

## First Success Goal

The first success should be small and visible:

1. Install the module.
2. Point it at a Convex deployment.
3. Render one query in a Nuxt page.
4. Confirm the page updates when the underlying data changes.

Do not start with auth, permissions, and MCP all at once unless the app genuinely needs all three on day one.

## Before You Add More Surface

Add each layer only when the app needs it:

- Add auth when the app needs sign-in and session-backed callers.
- Add permissions when the app needs backend-owned checks or tenant-scoped access.
- Add operations when destructive flows need preview and confirm.
- Add MCP when the same backend model needs to be exposed to an agent client.

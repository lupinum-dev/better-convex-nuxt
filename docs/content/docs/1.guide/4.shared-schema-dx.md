title: 'Shared Schema DX'
description: 'Use one args definition across Convex handlers, Nuxt server code, and MCP tools.'
---

`@lupinum/trellis/args` is the server-safe entrypoint for shared args definitions.

## Define Once

```ts [shared/schemas/post.ts]
import { defineArgs } from '@lupinum/trellis/args'
import { v } from 'convex/values'

export const createPost = defineArgs({
  description: 'Create a post',
  args: {
    title: v.string(),
    content: v.string(),
  },
  meta: {
    title: { label: 'Title', description: 'Post title' },
    content: { label: 'Content', description: 'Post body' },
  },
})
```

## Reuse In Convex

```ts [convex/posts.ts]
import { app } from './functions'

export const create = app.mutation({
  guard: canCreatePost,
  args: createPost.args,
  handler: async (ctx, args) => {
    const actor = await ctx.actor()
    return await ctx.db.insert('posts', {
      title: args.title,
      content: args.content,
      ownerId: actor.userId,
      workspaceId: actor.tenantId,
    })
  },
})
```

## Reuse In Nitro Or MCP

```ts [server/mcp/tools/create-post.ts]
import { defineTool } from '#trellis/mcp'

export default defineTool({
  schema: createPost,
  auth: 'required',
  handler: async (args, ctx) => {
    const id = await ctx.mutation(api.posts.create, args)
    return ctx.ok({ id }, `Created post "${args.title}"`)
  },
})
```

## Why `shared/` Exists

`shared/` is a runtime-boundary folder, not a Nuxt convention. Use it when both:

- `convex/` files
- Nuxt `server/` files

need to import the same args definitions.

## Trusted Caller Transport

- `schema.args` is the shared business shape used by MCP tools, docs, forms, and runtime parsing.
- `createApp(...)` owns the hidden trusted-caller transport in `convex/functions.ts`.
- `app.query(...)` / `app.mutation(...)` are the canonical handler authoring surface.

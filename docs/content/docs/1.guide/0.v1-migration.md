This release is a **hard cutover**. Legacy APIs are removed to keep the surface clean for production-scale apps.

## Breaking Changes

1. `useConvexMutation` now uses `execute()` instead of `mutate()`.
2. `useConvexMutation` and `useConvexAction` now expose `executeSafe()`.
3. Query disable semantics removed `'skip'`.
4. `useConvexQuery` and `useConvexPaginatedQuery` now disable with `args = null/undefined` or `enabled: false`.
5. Server helpers are now **event-first**:
   `serverConvexQuery(event, query, args, options)` (same for mutation/action).
6. `FetchOptions` is replaced by `ServerConvexOptions` with auth policy:
   `auth: 'auto' | 'required' | 'none'`.

## New APIs

- `useConvexOnce()` for one-shot client calls (`query/mutation/action` + safe variants).
- `createSharedConvexQuery()` for explicit app/request-scoped shared query state.
- `useConvexAuth().awaitAuthReady()` for middleware/startup auth stabilization.
- `keepPreviousData` and deep-unref args support in query composables.
- Pagination booleans: `hasNextPage`, `isLoadingFirstPage`, `isLoadingMore`, `isRefreshing`.

## Old to New Mapping

```ts
// Mutation
mutation.mutate(args)
mutation.execute(args)

// Mutation safe call
try {
  await mutation.execute(args)
} catch (e) {}
const result = await mutation.executeSafe(args)

// Query disable
useConvexQuery(api.users.get, condition ? {} : 'skip')
useConvexQuery(api.users.get, () => (condition ? {} : undefined))
// or
useConvexQuery(api.users.get, {}, { enabled: () => condition })

// Server helpers
serverConvexQuery(convexUrl, api.posts.list, {})
serverConvexQuery(event, api.posts.list, {})
```

## Anti-Patterns to Avoid

- Don’t wrap module calls in app-specific `withErrorHandling` wrappers unless behavior is truly app-only.
- Don’t store shared query state in custom `nuxtApp._foo` fields; use `createSharedConvexQuery()`.
- Don’t reintroduce `'skip'` helpers. Use nullable args + `enabled` for clarity.
- Don’t add RBAC route policy into core data/auth primitives unless required by broad use-cases.

This guide walks you through building a fully functional real-time task list. You will define a schema, write backend functions, and connect them to a Vue component using `useConvexQuery` and `useConvexMutation`.

::note
This guide assumes you have completed the [Get Started](/docs/guide/get-started) setup. Make sure `npx convex dev` and `nuxt dev --dotenv .env.local` are both running before you begin.
::

::steps

### Define your schema

Convex uses a schema to define your database tables. Create `convex/schema.ts` to describe a `tasks` table:

```ts [convex/schema.ts]
import { defineSchema, defineTable } from 'convex/server'
import { v } from 'convex/values'

export default defineSchema({
  tasks: defineTable({
    text: v.string(),
    completed: v.boolean(),
  }),
})
```

This creates a `tasks` table with two fields: `text` (a string) and `completed` (a boolean). Convex automatically adds `_id` and `_creationTime` to every document.

### Write a query function

Queries are read-only functions that fetch data from your database. Create `convex/tasks.ts` with a query to list all tasks:

```ts [convex/tasks.ts]
import { query } from './_generated/server'

export const list = query({
  args: {},
  handler: async (ctx) => {
    return await ctx.db.query('tasks').order('desc').collect()
  },
})
```

This query returns all tasks sorted by creation time (newest first). Convex queries are automatically cached and subscribe to real-time updates -- when any client adds or modifies a task, every subscribed client receives the update instantly.

### Write a mutation function

Mutations are functions that modify data. Add a `create` mutation to the same file:

```ts [convex/tasks.ts]
import { mutation, query } from './_generated/server'
import { v } from 'convex/values'

export const list = query({
  args: {},
  handler: async (ctx) => {
    return await ctx.db.query('tasks').order('desc').collect()
  },
})

export const create = mutation({
  args: {
    text: v.string(),
  },
  handler: async (ctx, args) => {
    await ctx.db.insert('tasks', {
      text: args.text,
      completed: false,
    })
  },
})

export const toggle = mutation({
  args: {
    id: v.id('tasks'),
  },
  handler: async (ctx, args) => {
    const task = await ctx.db.get(args.id)
    if (!task) throw new Error('Task not found')
    await ctx.db.patch(args.id, { completed: !task.completed })
  },
})
```

We have added two mutations: `create` inserts a new task, and `toggle` flips the `completed` status of an existing task.

### Display tasks with `useConvexQuery`

Now connect the backend to your Vue component. The `useConvexQuery` composable fetches data with full SSR support and keeps it updated in real time via WebSocket:

```vue [app/pages/tasks.vue]
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const { data: tasks, status } = await useConvexQuery(api.tasks.list)
</script>

<template>
  <div>
    <h1>Tasks</h1>

    <div v-if="status === 'pending'">Loading tasks...</div>

    <div v-else-if="status === 'error'">
      Something went wrong. Please try again.
    </div>

    <ul v-else-if="tasks && tasks.length > 0">
      <li v-for="task in tasks" :key="task._id">
        <span :class="{ 'line-through': task.completed }">
          {{ task.text }}
        </span>
      </li>
    </ul>

    <p v-else>No tasks yet. Add one below!</p>
  </div>
</template>
```

Key things to notice:

- **`await useConvexQuery`** -- the `await` enables SSR. The query runs on the server during the initial page load, so the HTML is rendered with data already present. No loading spinner on first paint.
- **`api.tasks.list`** -- fully typed reference to your Convex function. TypeScript knows the return type.
- **`data`** -- a reactive `Ref` that updates automatically when the underlying data changes on the server.
- **`status`** -- tracks the query lifecycle: `'pending'`, `'success'`, or `'error'`.

::tip
You can also destructure `pending` and `error` directly:

```ts
const { data: tasks, pending, error } = await useConvexQuery(api.tasks.list)
```
::

### Add task creation with `useConvexMutation`

The `useConvexMutation` composable returns a callable function with reactive state properties:

```vue [app/pages/tasks.vue]
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

const { data: tasks, status } = await useConvexQuery(api.tasks.list)

const newTaskText = ref('')
const createTask = useConvexMutation(api.tasks.create)

async function addTask() {
  const text = newTaskText.value.trim()
  if (!text) return

  await createTask({ text })
  newTaskText.value = ''
}
</script>

<template>
  <div>
    <h1>Tasks</h1>

    <!-- Add task form -->
    <form @submit.prevent="addTask">
      <input
        v-model="newTaskText"
        placeholder="What needs to be done?"
        :disabled="createTask.pending.value"
      />
      <button type="submit" :disabled="createTask.pending.value">
        {{ createTask.pending.value ? 'Adding...' : 'Add Task' }}
      </button>
    </form>

    <p v-if="createTask.error.value" style="color: red;">
      {{ createTask.error.value.message }}
    </p>

    <!-- Task list -->
    <div v-if="status === 'pending'">Loading tasks...</div>

    <ul v-else-if="tasks && tasks.length > 0">
      <li v-for="task in tasks" :key="task._id">
        <span :class="{ 'line-through': task.completed }">
          {{ task.text }}
        </span>
      </li>
    </ul>

    <p v-else>No tasks yet. Add one above!</p>
  </div>
</template>
```

The mutation function:
- **Is callable directly** -- `await createTask({ text })` sends the mutation to Convex.
- **Tracks its own state** -- `createTask.pending`, `createTask.error`, `createTask.status`, and `createTask.data` are all reactive refs.
- **Does not need manual loading flags** -- the composable manages pending/error state automatically.

### The complete working component

Here is the full component with task creation, toggling, and real-time updates:

```vue [app/pages/tasks.vue]
<script setup lang="ts">
import { api } from '~~/convex/_generated/api'

// Fetch tasks with SSR + real-time subscription
const { data: tasks, status } = await useConvexQuery(api.tasks.list)

// Mutations
const createTask = useConvexMutation(api.tasks.create)
const toggleTask = useConvexMutation(api.tasks.toggle)

// Form state
const newTaskText = ref('')

async function addTask() {
  const text = newTaskText.value.trim()
  if (!text) return

  await createTask({ text })
  newTaskText.value = ''
}

async function handleToggle(id: string) {
  await toggleTask({ id })
}
</script>

<template>
  <div style="max-width: 500px; margin: 2rem auto;">
    <h1>Task List</h1>

    <!-- Add task form -->
    <form @submit.prevent="addTask" style="display: flex; gap: 0.5rem; margin-bottom: 1rem;">
      <input
        v-model="newTaskText"
        placeholder="What needs to be done?"
        :disabled="createTask.pending.value"
        style="flex: 1; padding: 0.5rem;"
      />
      <button type="submit" :disabled="createTask.pending.value || !newTaskText.trim()">
        {{ createTask.pending.value ? 'Adding...' : 'Add' }}
      </button>
    </form>

    <p v-if="createTask.error.value" style="color: red;">
      Failed to add task: {{ createTask.error.value.message }}
    </p>

    <!-- Task list -->
    <div v-if="status === 'pending'">Loading tasks...</div>

    <ul v-else-if="tasks && tasks.length > 0" style="list-style: none; padding: 0;">
      <li
        v-for="task in tasks"
        :key="task._id"
        style="display: flex; align-items: center; gap: 0.5rem; padding: 0.5rem 0;"
      >
        <input
          type="checkbox"
          :checked="task.completed"
          @change="handleToggle(task._id)"
        />
        <span :style="{ textDecoration: task.completed ? 'line-through' : 'none' }">
          {{ task.text }}
        </span>
      </li>
    </ul>

    <p v-else>No tasks yet. Add one above!</p>
  </div>
</template>
```

::

## What just happened

With roughly 30 lines of backend code and a single Vue component, you built an app that:

1. **Renders on the server** -- the task list is included in the initial HTML response. No loading spinner.
2. **Hydrates without refetching** -- the client picks up the SSR data and starts a WebSocket subscription.
3. **Updates in real time** -- open the page in two browser tabs. Add a task in one tab and watch it appear instantly in the other.
4. **Is fully type-safe** -- TypeScript knows the shape of your tasks, mutation arguments, and return values.

## Basic protection

Protect any page from unauthenticated access by adding `convexAuth` to the page meta:

```vue [app/pages/dashboard.vue]
<script setup lang="ts">
definePageMeta({ convexAuth: true })

const { user } = useConvexAuth()
</script>

<template>
  <div>
    <h1>Dashboard</h1>
    <p>Welcome, {{ user?.name }}</p>
  </div>
</template>
```

When an unauthenticated user navigates to this page, they are automatically redirected to the sign-in page.

## Custom redirect

Override the default redirect destination for a specific page:

```vue [app/pages/admin/settings.vue]
<script setup lang="ts">
definePageMeta({
  convexAuth: {
    redirectTo: '/admin/login',
  },
})
</script>

<template>
  <div>
    <h1>Admin Settings</h1>
  </div>
</template>
```

The `redirectTo` value can be a string path or a Vue Router `RouteLocationRaw` object:

```ts
// String path
definePageMeta({ convexAuth: { redirectTo: '/login' } })

// Named route
definePageMeta({ convexAuth: { redirectTo: { name: 'auth-signin' } } })
```

## Default redirect configuration

Configure the default redirect for all protected routes in `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['better-convex-nuxt'],

  convex: {
    auth: {
      routeProtection: {
        redirectTo: '/login',        // default: '/auth/signin'
        preserveReturnTo: true,      // default: true
      },
    },
  },
})
```

::field-group
  ::field{name="redirectTo" type="string" default="'/auth/signin'"}
  The default path to redirect unauthenticated users to when a page has `convexAuth: true` without a custom `redirectTo`.
  ::
  ::field{name="preserveReturnTo" type="boolean" default="true"}
  When `true`, appends `?redirect=/original-path` to the redirect URL. This allows the sign-in page to redirect users back to the page they originally wanted to visit.
  ::
::

## Return-to behavior

When `preserveReturnTo` is enabled (the default), the redirect URL includes the original path as a query parameter:

```
/dashboard -> /auth/signin?redirect=%2Fdashboard
```

You can use this in your sign-in page to redirect back after successful authentication:

```vue [app/pages/auth/signin.vue]
<script setup lang="ts">
const { $auth } = useNuxtApp()
const { refreshAuth } = useConvexAuthInternal()
const route = useRoute()

async function handleSignIn() {
  const { error } = await $auth.signIn.email({
    email: email.value,
    password: password.value,
  })

  if (!error) {
    await refreshAuth()
    // Redirect to the original page, or home
    const redirect = route.query.redirect as string | undefined
    navigateTo(redirect || '/')
  }
}
</script>
```

::tip
To disable return-to behavior globally, set `preserveReturnTo: false` in your config. Per-page custom `redirectTo` values are used as-is without appending the return path when the value is a `RouteLocationRaw` object (non-string).
::

## How it works

When auth is enabled, the module registers a global middleware (`convex-auth.global`) that runs on every navigation. For each page:

1. The middleware reads the `convexAuth` page meta.
2. If `convexAuth` is not set or is `false`, the navigation proceeds without checks.
3. If `convexAuth` is `true` or an object, the middleware resolves the redirect destination.
4. It checks if the user is already on the redirect page to avoid infinite loops.
5. On the client, if auth is still pending (`isPending` is `true`), it waits for auth to settle using `awaitAuthReady()` with a 5-second timeout.
6. If the user is authenticated, the navigation proceeds.
7. If the user is not authenticated, they are redirected to the configured destination.

::note
On the server, SSR auth resolution happens before the middleware runs, so there is no waiting. If server-side auth is still pending (which should not happen under normal operation), the middleware falls through to the `isAuthenticated` check and redirects as a secure default.
::

## Example: protected dashboard with public landing page

```vue [app/pages/index.vue]
<script setup lang="ts">
// Public page -- no protection needed
</script>

<template>
  <div>
    <h1>Welcome to MyApp</h1>
    <NuxtLink to="/dashboard">Go to Dashboard</NuxtLink>
  </div>
</template>
```

```vue [app/pages/dashboard.vue]
<script setup lang="ts">
definePageMeta({ convexAuth: true })

import { api } from '~~/convex/_generated/api'

const { user } = useConvexAuth()
const { data: tasks } = await useConvexQuery(api.tasks.list)
</script>

<template>
  <div>
    <h1>{{ user?.name }}'s Dashboard</h1>
    <ul>
      <li v-for="task in tasks" :key="task._id">
        {{ task.text }}
      </li>
    </ul>
  </div>
</template>
```

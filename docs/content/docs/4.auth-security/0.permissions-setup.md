This is the complete setup for organization permissions. **Better Auth owns
roles, organizations, members, and invitations** (via the Organization plugin);
your app reads the role at request time and enforces it with
`auth.api.hasPermission`. You never store a `role` column or an `organizations`
table.

- **Owners** have full control over their organization
- **Admins** can manage members and content
- **Members** can create and edit their own content
- **Viewers** have read-only access

::callout{icon="i-lucide-zap" to="/docs/auth-security/standard-role-template"}
**Want the fastest start?** Use the [Standard Role Template](/docs/auth-security/standard-role-template) for a copy-pasteable baseline, then customize.
::

::callout{icon="i-lucide-alert-triangle" color="amber"}
**Better Auth is the single source of truth for role/org/member state.** Do not
mirror those tables into your app schema. See
[Authentication → source of truth](/docs/auth-security/authentication).
::

## Step 1: Enable the Organization Plugin

Roles and membership live in Better Auth. Add the Organization plugin and an
access-control policy to `convex/auth.ts` (the exact pattern `starters/team`
uses):

```ts [convex/auth.ts]
import { createAccessControl, organization } from 'better-auth/plugins'

const accessControl = createAccessControl({
  organization: ['update'],
  member: ['create', 'update', 'delete'],
  invitation: ['create', 'cancel'],
  post: ['create', 'read', 'update', 'delete'],
})

const owner = accessControl.newRole({
  organization: ['update'],
  member: ['create', 'update', 'delete'],
  invitation: ['create', 'cancel'],
  post: ['create', 'read', 'update', 'delete'],
})
const admin = accessControl.newRole({
  organization: ['update'],
  member: ['create', 'update', 'delete'],
  invitation: ['create', 'cancel'],
  post: ['create', 'read', 'update', 'delete'],
})
const member = accessControl.newRole({
  organization: [],
  member: [],
  invitation: [],
  post: ['create', 'read', 'update'],
})
const viewer = accessControl.newRole({
  organization: [],
  member: [],
  invitation: [],
  post: ['read'],
})

// ... inside betterAuth({ plugins: [...] }):
organization({
  ac: accessControl,
  roles: { owner, admin, member, viewer },
})
```

Product tables key off Better Auth **string IDs** — the organization id and the
Better Auth user id:

```ts [convex/schema.ts]
posts: defineTable({
  title: v.string(),
  content: v.string(),
  organizationId: v.string(), // Better Auth org id
  ownerId: v.string(), // Better Auth user id of the creator
  createdAt: v.number(),
})
  .index('by_organization', ['organizationId'])
  .index('by_owner', ['ownerId']),
```

## Step 2: Backend Authorization Helpers

Enforcement delegates to Better Auth. Throw **structured** `ConvexError` values
so the client can distinguish authentication from authorization failures.

```ts [convex/lib/permissions.ts]
import { ConvexError } from 'convex/values'

import { components } from '../_generated/api'
import type { MutationCtx, QueryCtx } from '../_generated/server'
import { authComponent, createAuth } from '../auth'

type Ctx = QueryCtx | MutationCtx
type Role = 'owner' | 'admin' | 'member' | 'viewer'
type PostPermission = 'create' | 'read' | 'update' | 'delete'

// The caller's role lives on their Better Auth member row.
export async function getBetterAuthMember(
  ctx: Ctx,
  args: { organizationId: string; userId: string },
) {
  return (await ctx.runQuery(components.betterAuth.adapter.findOne, {
    model: 'member',
    where: [
      { field: 'organizationId', value: args.organizationId },
      { field: 'userId', value: args.userId },
    ],
  })) as { role: Role } | null
}

// Require the caller to hold post:<permission> in the given organization.
export async function requirePostPermission(
  ctx: Ctx,
  args: { organizationId: string; permission: PostPermission },
) {
  const { auth, headers } = await authComponent.getAuth(createAuth, ctx)
  const session = await auth.api.getSession({ headers })
  if (!session) {
    throw new ConvexError({ code: 'UNAUTHENTICATED', message: 'Not authenticated' })
  }

  const allowed = await auth.api.hasPermission({
    headers,
    body: { organizationId: args.organizationId, permissions: { post: [args.permission] } },
  })
  if (!allowed.success) {
    throw new ConvexError({
      code: 'FORBIDDEN',
      message: `Missing post:${args.permission} permission`,
    })
  }

  return { userId: session.user.id }
}
```

## Step 3: Add the Permission Context Query

`createPermissions` calls a **no-argument** query that returns the caller's role
for UX gating, read from Better Auth. It matches the query type
`FunctionReference<'query', 'public', Record<string, never>, TContext | null>`.

```ts [convex/auth.ts]
import { query } from './_generated/server'
import { getBetterAuthMember } from './lib/permissions'

export const getPermissionContext = query({
  args: {},
  handler: async (ctx) => {
    const { auth, headers } = await authComponent.getAuth(createAuth, ctx)
    const session = await auth.api.getSession({ headers })
    if (!session) return null

    const organizationId = session.session.activeOrganizationId
    if (!organizationId) return null

    const member = await getBetterAuthMember(ctx, {
      organizationId,
      userId: session.user.id,
    })
    if (!member) return null

    return { role: member.role, userId: session.user.id, orgId: organizationId }
  },
})
```

## Step 4: UX Permission Map

`createPermissions` needs a `checkPermission` function to turn a permission
string into a boolean for `can()`. This mirrors the Better Auth policy for the
UI — it never enforces anything.

```ts [shared/permissions.ts]
export type Role = 'owner' | 'admin' | 'member' | 'viewer'

export const permissions = {
  'org.settings': { roles: ['owner'] },
  'org.invite': { roles: ['owner', 'admin'] },
  'org.members': { roles: ['owner', 'admin'] },
  'post.create': { roles: ['owner', 'admin', 'member'] },
  'post.read': { roles: ['owner', 'admin', 'member', 'viewer'] },
  'post.update': { own: ['member'], any: ['owner', 'admin'] },
  'post.delete': { own: ['member'], any: ['owner', 'admin'] },
  'post.publish': { roles: ['owner', 'admin'] },
} as const

export type Permission = keyof typeof permissions

export interface PermissionContext {
  role: Role
  userId: string
  orgId: string
}

export interface Resource {
  ownerId?: string
}

type Rule = { roles: readonly Role[] } | { own: readonly Role[]; any: readonly Role[] }

export function checkPermission(
  ctx: PermissionContext | null,
  permission: Permission,
  resource?: Resource,
): boolean {
  if (!ctx) return false
  const rule = permissions[permission] as Rule

  if ('roles' in rule) {
    return (rule.roles as readonly string[]).includes(ctx.role)
  }
  if ((rule.any as readonly string[]).includes(ctx.role)) return true
  if (!resource?.ownerId) return false
  return (rule.own as readonly string[]).includes(ctx.role) && resource.ownerId === ctx.userId
}
```

## Step 6: Create the Frontend Composable

```ts [composables/usePermissions.ts]
import { createPermissions } from '#imports'
import { api } from '#convex/api'
import {
  checkPermission,
  type Permission,
  type PermissionContext,
  type Resource,
} from '~~/shared/permissions'

export const { usePermissions, usePermissionRedirect } = createPermissions<
  Permission,
  PermissionContext,
  Resource
>({
  query: api.auth.getPermissionContext,
  checkPermission,
})
```

## Usage

### In Components

::callout{icon="i-lucide-alert-triangle" color="amber"}
**Frontend checks are UX only.** `can()` decides what to render; enforcement
happens in your Convex functions via `auth.api.hasPermission`.
::

```vue
<script setup lang="ts">
import { usePermissions } from '~/composables/usePermissions'

const { can, role, isAuthenticated } = usePermissions()
const { data: posts } = await useConvexQuery(api.posts.list, {})
</script>

<template>
  <button v-if="can('org.settings')">Settings</button>
  <button v-if="can('org.invite')">Invite Member</button>
  <button v-if="can('post.create')">New Post</button>

  <div v-for="post in posts" :key="post._id">
    <h3>{{ post.title }}</h3>
    <button v-if="can('post.update', post)">Edit</button>
    <button v-if="can('post.delete', post)">Delete</button>
    <button v-if="can('post.publish')">Publish</button>
  </div>
</template>
```

### Client-Side Redirect

This is a UX redirect helper. Enforcement stays in your Convex functions.

```vue [pages/settings.vue]
<script setup lang="ts">
import { usePermissionRedirect } from '~/composables/usePermissions'

usePermissionRedirect({ permission: 'org.settings', redirectTo: '/dashboard' })
</script>

<template>
  <h1>Organization Settings</h1>
</template>
```

### In Backend Mutations

Every mutation re-checks the caller's Better Auth permission and re-checks
ownership where the policy uses `own`. Bind `organizationId` from the target row,
not from a client argument.

```ts [convex/posts.ts]
import { mutation, query } from './_generated/server'
import { ConvexError, v } from 'convex/values'
import { getBetterAuthMember, requirePostPermission } from './lib/permissions'

export const create = mutation({
  args: { organizationId: v.string(), title: v.string(), content: v.string() },
  handler: async (ctx, args) => {
    const { userId } = await requirePostPermission(ctx, {
      organizationId: args.organizationId,
      permission: 'create',
    })

    return ctx.db.insert('posts', {
      title: args.title,
      content: args.content,
      organizationId: args.organizationId,
      ownerId: userId,
      createdAt: Date.now(),
    })
  },
})

export const update = mutation({
  args: { id: v.id('posts'), title: v.string() },
  handler: async (ctx, args) => {
    const post = await ctx.db.get(args.id)
    if (!post) throw new ConvexError({ code: 'NOT_FOUND', message: 'Not found' })

    const { userId } = await requirePostPermission(ctx, {
      organizationId: post.organizationId,
      permission: 'update',
    })

    // Ownership: members may only edit their own posts.
    const member = await getBetterAuthMember(ctx, {
      organizationId: post.organizationId,
      userId,
    })
    if (member?.role === 'member' && post.ownerId !== userId) {
      throw new ConvexError({ code: 'FORBIDDEN', message: 'You can only edit your own posts' })
    }

    await ctx.db.patch(args.id, { title: args.title })
  },
})
```

## Verification Checklist

- [ ] Owner can access org settings; admin cannot
- [ ] Member can create posts and edit only their own
- [ ] Viewer cannot create posts
- [ ] Backend rejects unauthorized mutations with `ConvexError({ code: 'FORBIDDEN' })`
- [ ] No `role` column and no `organizations` table exist in your schema

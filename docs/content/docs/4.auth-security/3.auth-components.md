---
title: "Auth Components"
description: "Conditionally render content based on authentication state."
---

The module provides four components for conditionally rendering content based on the current authentication state. These components prevent flash of wrong content during SSR hydration by waiting for auth state to settle before showing or hiding content.

::note
Auth components are auto-imported when `auth` is enabled in your Nuxt config (the default). No manual imports are needed.
::

## `<ConvexAuthenticated>`

Renders its slot content only when the user is authenticated **and** auth state has settled (not pending).

```vue
<ConvexAuthenticated>
  <p>Welcome back! You are signed in.</p>
</ConvexAuthenticated>
```

Internally, this component checks `isAuthenticated && !isPending` before rendering, so content is hidden during the initial auth check and only appears once the user is confirmed authenticated.

## `<ConvexUnauthenticated>`

Renders its slot content only when the user is **not** authenticated **and** auth state has settled.

```vue
<ConvexUnauthenticated>
  <p>Please sign in to continue.</p>
  <NuxtLink to="/auth/signin">Go to sign in</NuxtLink>
</ConvexUnauthenticated>
```

This component checks `!isAuthenticated && !isPending`, ensuring it does not flash on initial load while auth is still being determined.

## `<ConvexAuthLoading>`

Renders its slot content only while auth state is being determined (`isPending` is `true`).

```vue
<ConvexAuthLoading>
  <div class="loading-spinner">
    <p>Checking authentication...</p>
  </div>
</ConvexAuthLoading>
```

Use this to show loading indicators during the brief period between page load and auth resolution. On SSR-rendered pages where the server has already resolved auth, this component renders nothing.

## `<ConvexAuthError>`

Renders its slot content when authentication has failed or encountered an error. Provides `retry` and `error` via the scoped slot.

```vue
<ConvexAuthError v-slot="{ retry, error }">
  <div class="auth-error">
    <p>Authentication failed: {{ error }}</p>
    <button @click="retry">Retry</button>
  </div>
</ConvexAuthError>
```

The component detects error state when:
- Auth is no longer pending (the check has completed)
- The user is not authenticated
- An explicit auth error exists (e.g., 401/403 from the token endpoint), or a token is present but could not be decoded into user data

### Scoped slot props

::field-group
  ::field{name="error" type="string | null"}
  The error message from the auth failure, or `null` if the error is a token decode issue.
  ::
  ::field{name="retry" type="() => void"}
  Retries authentication by reloading the page.
  ::
::

## Combining all four components

Use all four components together to handle every possible auth state without any content flashes:

```vue [app/pages/index.vue]
<script setup lang="ts">
const { user } = useConvexAuth()
</script>

<template>
  <ConvexAuthLoading>
    <div class="loading">
      <p>Checking authentication...</p>
    </div>
  </ConvexAuthLoading>

  <ConvexAuthenticated>
    <h1>Dashboard</h1>
    <p>Welcome, {{ user?.name }}!</p>
    <Dashboard />
  </ConvexAuthenticated>

  <ConvexUnauthenticated>
    <h1>Welcome</h1>
    <p>Please sign in to access the dashboard.</p>
    <NuxtLink to="/auth/signin">Sign in</NuxtLink>
  </ConvexUnauthenticated>

  <ConvexAuthError v-slot="{ retry, error }">
    <div class="error">
      <h2>Something went wrong</h2>
      <p>{{ error ?? 'Authentication failed. Please try again.' }}</p>
      <button @click="retry">Retry</button>
    </div>
  </ConvexAuthError>
</template>
```

::tip
These components are mutually exclusive in practice -- only one renders at any given time based on the current auth state. You can safely place them all at the same level in your template.
::

## Usage with layouts

Auth components work well in layouts to provide app-wide authentication UI:

```vue [app/layouts/default.vue]
<script setup lang="ts">
const { user, signOut } = useConvexAuth()
</script>

<template>
  <div>
    <header>
      <nav>
        <NuxtLink to="/">Home</NuxtLink>
        <ConvexAuthenticated>
          <span>{{ user?.email }}</span>
          <button @click="signOut">Sign out</button>
        </ConvexAuthenticated>
        <ConvexUnauthenticated>
          <NuxtLink to="/auth/signin">Sign in</NuxtLink>
        </ConvexUnauthenticated>
      </nav>
    </header>

    <main>
      <slot />
    </main>
  </div>
</template>
```

## Overview

better-convex-nuxt manages Convex JWT tokens automatically. Tokens are fetched during SSR, hydrated to the client, reused briefly on the client to avoid duplicate requests, and refreshed when the current token is missing or too close to expiry.

## Token Lifecycle

::steps{level="4"}

#### Server-Side Token Exchange

During SSR, the server plugin reads the Better Auth session cookie and exchanges it for a Convex JWT via the auth proxy.

#### Client Hydration

The JWT is passed to the client via `useState`, so the client starts authenticated without an extra network request.

#### Automatic Refresh

On the client, auth refresh requests are deduplicated. A recent token can be reused for a short window, but a token near expiry is rejected and replaced with a fresh one.

::

## Key Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `TOKEN_CACHE_MS` | 10,000ms | Client-side fetch cache duration |
| `TOKEN_EXPIRY_SAFETY_BUFFER_MS` | 30,000ms | Refresh this long before expiry |
| `AUTH_REFRESH_TIMEOUT_MS` | 5,000ms | Timeout for refresh requests |
| `SERVER_FETCH_TIMEOUT_MS` | 8,000ms | Server-side fetch timeout |

## Manual Token Refresh

Use `refreshAuth()` from `useConvexAuth()` to manually trigger a token refresh after a direct Better Auth client action:

```vue
<script setup lang="ts">
const { client, refreshAuth } = useConvexAuth()

async function handleLogin(email: string, password: string) {
  await client!.signIn.email({ email, password })
  await refreshAuth() // Update Convex auth state
  navigateTo('/dashboard')
}
</script>
```

For the common email/password or email/sign-up flows, prefer `useConvexAuthActions()` instead. It wraps the Better Auth client call, refreshes Convex auth state, normalizes Better Auth errors, and applies redirect handling.

## Server-Side Token Caching

For high-traffic applications, enable server-side token caching to avoid repeated token exchanges:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    auth: {
      cache: {
        enabled: true,
        ttl: 60 // Cache for 60 seconds
      }
    }
  }
})
```

### Clearing the Cache

Ordinary `useConvexAuth().signOut()` automatically clears the cached JWT for the current Better Auth session when the upstream logout response removes the session cookie.

You only need manual cache clearing for custom server-side revocation flows outside the standard module-owned sign-out path:

```ts [server/api/logout.post.ts]
export default defineEventHandler(async (event) => {
  const sessionToken =
    getCookie(event, '__Secure-better-auth.session_token')
    ?? getCookie(event, 'better-auth.session_token')
  if (sessionToken) {
    await serverConvexClearAuthCache(sessionToken)
  }
})
```

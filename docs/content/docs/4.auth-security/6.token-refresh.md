## Overview

Better Convex Nuxt manages JWT tokens automatically. Tokens are fetched during SSR, hydrated to the client, and refreshed before they expire.

## Token Lifecycle

::steps{level="4"}

#### Server-Side Token Exchange

During SSR, the server plugin reads the `better-auth.session_token` cookie and exchanges it for a Convex JWT via the auth proxy.

#### Client Hydration

The JWT is passed to the client via `useState`, so the client starts authenticated without an extra network request.

#### Automatic Refresh

The client monitors token expiry and refreshes 30 seconds before it expires. A 10-second cache prevents redundant fetch requests.

::

## Key Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `TOKEN_CACHE_MS` | 10,000ms | Client-side fetch cache duration |
| `TOKEN_EXPIRY_SAFETY_BUFFER_MS` | 30,000ms | Refresh this long before expiry |
| `AUTH_REFRESH_TIMEOUT_MS` | 5,000ms | Timeout for refresh requests |
| `SERVER_FETCH_TIMEOUT_MS` | 8,000ms | Server-side fetch timeout |

## Manual Token Refresh

Use `refreshAuth()` from `useConvexAuthInternal()` to manually trigger a token refresh — for example, after a sign-in action:

```vue
<script setup lang="ts">
import { createAuthClient } from 'better-auth/vue'

const authClient = createAuthClient({ baseURL: '/api/auth' })
const { refreshAuth } = useConvexAuthInternal()

async function handleLogin(email: string, password: string) {
  await authClient.signIn.email({ email, password })
  await refreshAuth() // Update Convex auth state
  navigateTo('/dashboard')
}
</script>
```

## Server-Side Token Caching

For high-traffic applications, enable server-side token caching to avoid repeated token exchanges:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    auth: {
      cache: {
        enabled: true,
        ttl: 60 // Cache for 60 seconds
      }
    }
  }
})
```

### Clearing the Cache

When a user signs out, clear their cached token to prevent stale auth:

```ts [server/api/logout.post.ts]
export default defineEventHandler(async (event) => {
  const sessionToken = getCookie(event, 'better-auth.session_token')
  if (sessionToken) {
    await serverConvexClearAuthCache(sessionToken)
  }
})
```

## Nuxt Hook

The module emits a `better-convex:auth:refresh` hook when a token refresh occurs:

```ts [plugins/auth-watcher.ts]
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('better-convex:auth:refresh', () => {
    console.log('Auth token was refreshed')
  })
})
```

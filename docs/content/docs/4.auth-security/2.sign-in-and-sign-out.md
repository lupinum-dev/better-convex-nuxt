## Reading auth state

::note
Start with [Authentication](/docs/auth-security/authentication) if you want the full mental model first. This page focuses on the app-level sign-in and sign-out API once auth is already set up.
::

The `useConvexAuth()` composable provides reactive authentication state:

```vue [app/pages/profile.vue]
<script setup lang="ts">
const { user, isAuthenticated, isPending, signOut } = useConvexAuth()
</script>

<template>
  <div v-if="isPending">Loading...</div>
  <div v-else-if="isAuthenticated">
    <p>Welcome, {{ user?.name }}</p>
    <p>{{ user?.email }}</p>
    <button @click="signOut">Sign out</button>
  </div>
  <div v-else>
    <p>Not signed in</p>
  </div>
</template>
```

### Return values

::field-group
  ::field{name="user" type="Readonly<Ref<ConvexUser | null>>"}
  The authenticated user data. `null` when not authenticated.
  ::
  ::field{name="isAuthenticated" type="ComputedRef<boolean>"}
  `true` when both a valid token and user data are present.
  ::
  ::field{name="isPending" type="Readonly<Ref<boolean>>"}
  `true` while auth state is still being determined (initial load or refresh). `false` on the server since SSR auth is resolved before render.
  ::
  ::field{name="isAnonymous" type="ComputedRef<boolean>"}
  `true` when auth has settled and no authenticated user is present.
  ::
  ::field{name="isSessionExpired" type="ComputedRef<boolean>"}
  `true` when the user was previously authenticated but the current session is gone.
  ::
  ::field{name="client" type="Better Auth client | null"}
  The raw Better Auth client. Use this for direct `signIn.*`, `signUp.*`, `signOut()`, password reset, magic link, and OAuth provider calls.
  ::
  ::field{name="refreshAuth" type="() => Promise<void>"}
  Manually refresh Convex auth state after a direct Better Auth client action.
  ::
  ::field{name="authError" type="Readonly<Ref<Error | null>>"}
  The last auth refresh error, normalized as an `Error`.
  ::
  ::field{name="signOut" type="() => Promise<...>"}
  Signs out the user. Clears local state immediately and calls Better Auth's `signOut()`.
  ::
::

### The `ConvexUser` type

The `user` ref contains a `ConvexUser` object with the following fields:

::field-group
  ::field{name="id" type="string"}
  Unique user identifier.
  ::
  ::field{name="name" type="string"}
  User's display name.
  ::
  ::field{name="email" type="string"}
  User's email address.
  ::
  ::field{name="emailVerified" type="boolean | undefined"}
  Whether the email has been verified.
  ::
  ::field{name="image" type="string | undefined"}
  URL to the user's profile image.
  ::
  ::field{name="createdAt" type="string | undefined"}
  When the user account was created.
  ::
  ::field{name="updatedAt" type="string | undefined"}
  When the user account was last updated.
  ::
::

## Signing out

Call `signOut()` from `useConvexAuth()`. It clears local state immediately for a responsive UI, then calls Better Auth's server-side sign-out:

```vue [app/components/UserMenu.vue]
<script setup lang="ts">
const { signOut } = useConvexAuth()

async function handleSignOut() {
  await signOut()
  navigateTo('/auth/signin')
}
</script>

<template>
  <button @click="handleSignOut">Sign out</button>
</template>
```

## Signing in

There are two separate layers involved in authentication, each with a distinct responsibility:

```
Better Auth client          better-convex-nuxt module
─────────────────           ─────────────────────────
client.signIn.email()  ──►  sets session cookie
client.signIn.social() ──►  redirects to provider
client.signUp.email()  ──►  creates session cookie

useConvexAuth()        ◄──  reads auth state, exposes client, signOut(), refreshAuth()
useConvexAuthActions() ◄──  wraps auth actions with refresh, error handling, redirect
```

The module does not wrap Better Auth's method surface. Instead, `useConvexAuth()` exposes the raw typed client and `useConvexAuthActions()` adds the Convex-specific parts: refresh, normalized errors, and redirect handling.

### Email and password sign-in

```vue [app/pages/auth/signin.vue]
<script setup lang="ts">
const { client } = useConvexAuth()
const { execute, pending, error } = useConvexAuthActions()

const email = ref('')
const password = ref('')

async function handleSignIn() {
  if (!client)
    return

  await execute(
    () => client.signIn.email({
      email: email.value,
      password: password.value,
    }),
    { redirectTo: '/' },
  )
}
</script>

<template>
  <form @submit.prevent="handleSignIn">
    <div>
      <label for="email">Email</label>
      <input id="email" v-model="email" type="email" required />
    </div>
    <div>
      <label for="password">Password</label>
      <input id="password" v-model="password" type="password" required />
    </div>
    <p v-if="error" style="color: red">{{ error.message }}</p>
    <button type="submit" :disabled="pending">Sign in</button>
  </form>
</template>
```

### OAuth sign-in

For OAuth providers (GitHub, Google, etc.), call the Better Auth client directly. These flows redirect the browser away from your app, so they are not a good fit for `useConvexAuthActions()`:

```vue [app/components/OAuthButtons.vue]
<script setup lang="ts">
const { client } = useConvexAuth()

async function signInWithGitHub() {
  if (!client)
    return

  await client.signIn.social({
    provider: 'github',
  })
}

async function signInWithGoogle() {
  if (!client)
    return

  await client.signIn.social({
    provider: 'google',
  })
}
</script>

<template>
  <div>
    <button @click="signInWithGitHub">Sign in with GitHub</button>
    <button @click="signInWithGoogle">Sign in with Google</button>
  </div>
</template>
```

::tip
OAuth flows redirect the user away from your app and back after authorization. The auth state is automatically picked up on the return navigation -- you do not need to call `refreshAuth()` for OAuth sign-ins since the page reloads.
::

### Sign up with email and password

```vue [app/pages/auth/signup.vue]
<script setup lang="ts">
const { client } = useConvexAuth()
const { execute, pending, error } = useConvexAuthActions()

const name = ref('')
const email = ref('')
const password = ref('')

async function handleSignUp() {
  if (!client)
    return

  await execute(
    () => client.signUp.email({
      name: name.value,
      email: email.value,
      password: password.value,
    }),
    { redirectTo: '/' },
  )
}
</script>

<template>
  <form @submit.prevent="handleSignUp">
    <div>
      <label for="name">Name</label>
      <input id="name" v-model="name" type="text" required />
    </div>
    <div>
      <label for="email">Email</label>
      <input id="email" v-model="email" type="email" required />
    </div>
    <div>
      <label for="password">Password</label>
      <input id="password" v-model="password" type="password" required />
    </div>
    <p v-if="error" style="color: red">{{ error.message }}</p>
    <button type="submit" :disabled="pending">Sign up</button>
  </form>
</template>
```

## Refreshing auth state

For direct Better Auth client calls that do not go through `useConvexAuthActions()`, call `refreshAuth()` from `useConvexAuth()` to sync the Convex auth state. This triggers a fresh token fetch and updates the reactive auth state (`user`, `isAuthenticated`, `authError`):

```ts
const { client, refreshAuth } = useConvexAuth()

// After sign-in succeeds:
await client!.signIn.email({ email, password })
await refreshAuth()
```

In the common case, prefer `useConvexAuthActions()` instead. It runs the auth action, normalizes Better Auth errors, refreshes Convex auth state, and applies redirect handling in one place.

`refreshAuth()` deduplicates concurrent calls. If multiple parts of your app call it simultaneously, only one token fetch runs. It throws if the refresh fails or times out (default 5 seconds).

## Complete sign-in page example

Here is a complete sign-in page with email/password and OAuth:

```vue [app/pages/auth/signin.vue]
<script setup lang="ts">
definePageMeta({ skipConvexAuth: true })

const { client, isAuthenticated } = useConvexAuth()
const { execute, pending, error } = useConvexAuthActions()

const email = ref('')
const password = ref('')

// Redirect if already authenticated
watch(isAuthenticated, (authed) => {
  if (authed) navigateTo('/')
}, { immediate: true })

async function handleEmailSignIn() {
  if (!client)
    return

  await execute(
    () => client.signIn.email({
      email: email.value,
      password: password.value,
    }),
    { redirectTo: '/' },
  )
}

async function handleOAuthSignIn(provider: string) {
  if (!client)
    return

  await client.signIn.social({ provider })
}
</script>

<template>
  <div>
    <h1>Sign In</h1>

    <form @submit.prevent="handleEmailSignIn">
      <div>
        <label for="email">Email</label>
        <input id="email" v-model="email" type="email" required />
      </div>
      <div>
        <label for="password">Password</label>
        <input id="password" v-model="password" type="password" required />
      </div>
      <p v-if="error" style="color: red">{{ error.message }}</p>
      <button type="submit" :disabled="pending">
        {{ pending ? 'Signing in...' : 'Sign in' }}
      </button>
    </form>

    <hr />

    <button @click="handleOAuthSignIn('github')">
      Sign in with GitHub
    </button>
    <button @click="handleOAuthSignIn('google')">
      Sign in with Google
    </button>

    <p>
      Don't have an account?
      <NuxtLink to="/auth/signup">Sign up</NuxtLink>
    </p>
  </div>
</template>
```

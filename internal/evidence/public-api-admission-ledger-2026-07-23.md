# Public API admission ledger — 2026-07-23

## Outcome

Every package entry, Nuxt auto-import, global component, and command in the vNext candidate was checked
against the RFC public API admission test. The canonical machine-readable symbol inventory remains
`scripts/package-entry-manifest.mjs` plus `src/module-api-surface.ts`; this document records the
admission reasoning and evidence rather than creating another export allowlist.

The audit found accidental surface in four places:

- the Nuxt root and auth-client declarations exposed internal normalized/configuration aliases;
- the server entry exposed the caller's classified-fetch and boundary-normalizer implementation;
- Vue exported names for inferred result/plugin shapes that no consumer implements;
- MCP and the experimental App entry exported internal handler/diagnostic/result shapes that callers
  already receive through inference.

Those names were removed. Nuxt's root, auth-client, server, and user-sync entries now use the same exact
declaration allowlist enforcement already used by Vue, MCP, and Convex Auth.

## Removed names

The following names failed admission because they exposed implementation structure, duplicated an
inferable type, or had no independent consumer contract:

- Nuxt root: `LogLevel`, `ConvexAuthPageMeta`, `AuthProxyDefaults`, `ConvexDebugOptions`,
  `ConvexRouteProtectionConfig`, `NormalizedConvexAuthConfig`, `ConvexIdentityKey`, `QueryDefaults`,
  and `UploadDefaults`.
- auth-client: `AuthClientPlugins` and `ConvexAuthClientDefinitionOptions`.
- server: `createClassifiedConvexFetch`, `normalizeServerConvexBoundaryError`, and
  `ConvexCredential`.
- user sync: `UserSyncRebuildResult`.
- Vue root: `BetterConvexPlugin`, `ConvexQuerySkip`, and `UseConvexQueryResult`.
- Vue MCP App: `McpAppPhase`, `UseMcpAppOptions`, and `UseMcpAppReturn`.
- MCP: `ConvexMcpHandler`, `ConvexMcpHandlerOptions`, `ConvexMcpRequestContext`,
  `McpToolDiagnostic`, and `McpToolDiagnosticOptions`.

No runtime behavior or compatibility shim replaced these names. Public functions retain precise
inferred inputs and results. Internal shared types moved behind the entry only where runtime validation
and the public function genuinely share one definition.

## Enforcement

- Every package entry now sets `exactDeclaredExports: true`.
- The generated API reference reads the package-entry manifest and Nuxt auto-import registry.
- Package checks reject missing, extra, duplicate, value/type-space-mismatched, or transitive public
  declarations.
- Vue MCP Apps remains an isolated experimental subpath; it is not exported from the Vue root.
- Final MCP wire compliance remains separately blocked on the published 2026-07-28 specification and is
  not implied by this admission audit.

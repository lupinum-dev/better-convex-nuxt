## Status vocabulary

- `pending`: dependencies are not yet complete or it is not one of the next executable tasks.
- `in_progress`: the single task currently being changed or evidenced.
- `blocked`: no safe completion is possible without the named external state or human authority.
- `done`: the acceptance proof passed and the completion commit/evidence is recorded.
- `rejected`: evidence showed the task's proposed path was wrong; the rejected path and deletion are recorded.

## Completion review questions

Before marking any task or phase done, answer:

1. Did this create a second source of truth?
2. Could an existing path have been deleted or simplified instead?
3. Did replaced code leave in the same commit or an explicitly dependent deletion task?
4. Did a frontend/transport layer acquire backend authority?
5. Did any token, provider reference, argument, locator, or raw cause enter a public or diagnostic surface?
6. Is the named proof behavioral and exact-artifact aware where publication behavior is claimed?
7. Does the implementation still honor the one-topology, one-client-engine, and application-owned-policy decisions?
8. Is a human/external action being reported honestly rather than simulated by repository tests?

# Better Convex Nuxt product context

## Product

Better Convex Nuxt is a Nuxt 4 integration for Convex with SSR-to-realtime queries, Better Auth, request-scoped server calls, optimistic updates, uploads, and a shared structured error model.

## Users and purpose

It serves Nuxt application teams building realtime products that want the framework integration and lifecycle boundaries handled coherently. The documentation should make identity, SSR, hydration, subscriptions, server ownership, and authorization boundaries understandable before showing feature recipes.

## Personality

Direct, operational, confident, and technically disciplined. Convex red is a focused signal color against a neutral shadcn-style interface; the design should feel fast and realtime without becoming noisy or futuristic.

## Accessibility

Target WCAG 2.2 AA. All navigation and interactive examples must be keyboard usable, expose clear status and error states, respect reduced motion, and maintain readable contrast in light and dark modes.

## Design principles

- Explain the lifecycle and ownership model before shortcuts.
- Treat security boundaries as product behavior, not footnotes.
- Prefer runnable, typed examples with explicit states.
- Keep the path from evaluation to production operations visible.

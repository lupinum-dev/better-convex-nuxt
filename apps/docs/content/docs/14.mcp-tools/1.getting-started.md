Expose one MCP tool end to end. This page walks the narrowest safe path: an endpoint, middleware that authenticates bearer tokens, a runtime, and a read-only tool. Destructive tools come later — they have their own requirements.

::steps{level="2"}

### Define the MCP endpoint

```ts [server/api/mcp/[...].ts]
export default defineMcpHandler({
  name: 'mcp-reference',
  browserRedirect: '/',
})
```

Small handler. The runtime does the work.

### Authenticate requests in middleware

```ts [server/middleware/mcp-auth.ts]
export default defineEventHandler(async (event) => {
  if (!event.path?.startsWith('/api/mcp')) return

  const header = getHeader(event, 'authorization')
  if (!header?.startsWith('Bearer ')) return

  const token = header.slice('Bearer '.length)
  const hash = await sha256(token)

  const mcpAuth = await validateMcpToken(event, hash)
  if (mcpAuth) event.context.mcpAuth = mcpAuth
})
```

The pattern: read `Authorization: Bearer`, hash, validate the hash in Convex, put the result on `event.context.mcpAuth`. Tokens stay hashed at rest; the runtime gets a clean transport identity.

### Define the MCP runtime

```ts [server/mcp/runtime.ts]
export const mcpRuntime = defineMcpApp({
  callConvex: async (event, principal) =>
    createServerConvexCaller(
      event,
      principal.kind === 'agent'
        ? {
            auth: 'trusted',
            actor: { userId: principal.userId },
            principal,
          }
        : { auth: 'none' },
    ),
  resolvePrincipal: async event => getMcpPrincipal(event),
  resolveCapabilities: async ({ principal, convex }) => {
    // derive tool-level capabilities
  },
})
```

| Field | Does |
| --- | --- |
| `callConvex` | Per-request Convex caller bound to the MCP principal |
| `resolvePrincipal` | Turns middleware context into a principal |
| `resolveCapabilities` | Coarse capability snapshot for tool gating |

### Export the tool factory

```ts [server/mcp/runtime.ts]
export const tool = mcpRuntime.tool
```

Individual tools import this.

### Start with a read tool

```ts [server/mcp/tools/list-workspaces.ts]
import { tool } from '~/server/mcp/runtime'

export default tool({
  name: 'list-workspaces',
  description: 'List workspaces the caller has access to',
  ref: api.workspaces.list,
  kind: 'query',
})
```

One read-only tool. The real business rules stay in the protected handler.

::

## Why read-only first?

::caution
Destructive tools have extra requirements — they must be backed by an [operation](/apps/docs/permissions/operations) with preview, and `destructiveSafety` must be configured. Getting the transport and auth path working with a read tool first keeps those concerns separate.
::

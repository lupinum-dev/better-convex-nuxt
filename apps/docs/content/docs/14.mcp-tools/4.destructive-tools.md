Destructive MCP tools use operation-backed tooling, not raw one-shot mutations. That way preview, confirmation, and execute all tie to the *same* protected business action — the agent can't accidentally run a destructive mutation without first seeing what it's about to do.

```ts [server/mcp/tools/delete-runbook.ts]
import { tool } from '~/server/mcp/runtime'
import { removeRunbookOp, remove, previewRemove } from '~/convex/runbooks'

export default tool.fromOperation(removeRunbookOp, {
  execute: remove,
  preview: previewRemove,
  permission: runbookDelete,
  meta: {
    name: 'delete-runbook',
  },
})
```

One tool, one underlying operation, three refs wired through Trellis.

## Why this is required

Trellis binds destructive tools to three things:

| Requirement | Enforced because |
| --- | --- |
| An operation with a stable `id` | Confirmation payload references the operation |
| An `execute` ref projected from that operation | Guarantees execute runs the same handler preview described |
| A `preview` ref projected from that same operation | Prevents a different preview from misleading the agent |

::warning
If the execute and preview refs don't match the operation's projections, Trellis rejects the binding. That prevents a destructive tool from drifting away from the operation it claims to represent — the kind of bug that would otherwise surface as "the agent confirmed X and the system did Y."
::

## The confirmation flow

::steps{level="3"}

### Preview

Agent calls the preview. Trellis runs the operation's preview phase against current state.

### Present summary and confirmation state

Preview returns `display` (human-readable) and `confirm` (machine-checked). The agent shows the summary and waits for user approval.

### Confirm

Agent sends the confirmation payload back to Trellis.

### Execute

Trellis validates the confirmation matches the previewed state, then runs the execute path.

::

If the confirmation drifts (state changed between preview and confirm), Trellis denies the execute. The agent re-previews instead of overriding.

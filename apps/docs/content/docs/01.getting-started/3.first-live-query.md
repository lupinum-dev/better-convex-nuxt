Build the smallest Trellis app that feels real: render a list on the page, add a record, and watch the list update with no refetch code anywhere.

By the end of this page, you'll have one shared args definition, one public query, one public mutation, one Nuxt page rendering the list, and one live update you can see with your own eyes. It follows the same shape as [`examples/01-public-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/01-public-todo).

::note
You need a working installation first. If you haven't done that yet, go to [Installation](/docs/getting-started/installation). For this smallest path, the only required env value is `CONVEX_URL`.
::

::tip
If you want the paved generated shape instead of the hand-built walkthrough, start with `npx trellis init my-app --template personal` and then compare this page against the generated `convex/domain/` and `shared/schemas/` files. This guide keeps the flat example shape because it mirrors `01-public-todo`.
::

## Build it

::steps{level="3"}

### Define one shared args schema

Put the input shape in `shared/` — not inside a Convex handler — so both the backend and the Nuxt side can reuse it without going out of sync.

```ts [shared/schemas/todo.ts]
import { defineArgs } from '@lupinum/trellis/args'
import { v } from 'convex/values'

export const createTodo = defineArgs({
  description: 'Create a public todo item',
  args: {
    title: v.string(),
  },
})
```

::tip
Defining args with `defineArgs` once means the backend and the Nuxt side validate against the same contract. You do not end up with one input shape in Convex and another in the page.
::

### Add the smallest Trellis runtime and handlers

Wire `defineTrellis` so you get `mutation` and `query` builders, then define a list, a create, and a toggle. Every handler uses `open`, which is the explicit public baseline.

```ts [convex/todos.ts]
import { open, requireRecord } from '@lupinum/trellis/auth'
import { defineTrellis } from '@lupinum/trellis/functions'
import { v } from 'convex/values'

import { createTodo } from '../shared/schemas/todo'
import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'

const { mutation, query } = defineTrellis({
  query: generatedQuery,
  mutation: generatedMutation,
})

export const list = query({
  args: {},
  guard: open,
  handler: async (ctx) => {
    return await ctx.db.query('todos').order('desc').collect()
  },
})

export const create = mutation({
  args: createTodo.args,
  guard: open,
  handler: async (ctx, args) => {
    return await ctx.db.insert('todos', {
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})

export const toggle = mutation({
  args: { id: v.id('todos') },
  guard: open,
  load: async (ctx, args) => {
    const todo = await ctx.db.get(args.id)
    requireRecord(todo, 'Todo')
    return { todo }
  },
  handler: async (ctx, args, { todo }) => {
    await ctx.db.patch(args.id, {
      completed: !todo.completed,
    })
  },
})
```

This is the whole point of the first live query path: Trellis stays small when the app is small.

### Render the query on a page

Use `useConvexQuery` for the list and `useConvexMutation` for creation.

```vue [pages/index.vue]
<script setup lang="ts">
import { computed, ref } from 'vue'

import { api } from '#trellis/api'

const { data: todos, pending, error } = await useConvexQuery(api.todos.list, {})
const createTodo = useConvexMutation(api.todos.create)

const title = ref('')
const todoItems = computed(() => todos.value ?? [])

async function handleCreate() {
  await createTodo({ title: title.value })
  title.value = ''
}
</script>

<template>
  <form @submit.prevent="handleCreate">
    <input v-model="title" placeholder="Write something small and concrete" required />
    <button type="submit">Add</button>
  </form>

  <p v-if="error">{{ error.message }}</p>
  <p v-else-if="pending">Loading...</p>
  <ul v-else>
    <li v-for="todo in todoItems" :key="todo._id">
      {{ todo.title }}
    </li>
  </ul>
</template>
```

The important thing is what you _don't_ have to write — no fetch endpoint, no manual refetch after create, no transport glue between Nuxt and Convex. `useConvexQuery` returns a live subscription, so when the mutation commits, the list updates on its own.

### Verify the live update

Run the app and add a todo. You should see:

- the page render the initial list on load (from SSR);
- the form create a todo when submitted;
- the list update without a manual refresh.

That's the first real Trellis win.

::

## Common pitfalls

- Adding sign-in or protected data rules before the public baseline is working.
- Treating `useConvexQuery` like a one-shot fetch and manually refetching after writes — it's already live.
- Hiding the shared args shape inside one handler file instead of keeping it in `shared/`.

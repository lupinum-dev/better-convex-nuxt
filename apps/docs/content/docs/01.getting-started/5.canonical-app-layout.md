This page defines the current canonical Trellis app layout.

It is based on what `trellis init <name> --template public|personal|workspace|cms [--mcp]` generates today.

## Rule

If you start with the CLI, keep the generated shape.

Trellis is meant to make apps boring to navigate. The point is not to preserve every starter file forever. The point is to keep the shell stable and put business code where it changes together.

## Canonical tree

```text
nuxt.config.ts
convex/
  auth.ts
  auth.config.ts
  convex.config.ts
  functions.ts
  http.ts
  schema.ts
  test.setup.ts
  auth/
  permissions/
  features/
shared/
  features/
app/
  features/
pages/
server/
  api/
  mcp/
```

`public`, `personal`, `workspace`, and `cms` all converge directly on that feature-first tree.

This is the load-bearing contract:

- `nuxt.config.ts` owns module wiring.
- `convex/auth.ts`, `convex/auth.config.ts`, `convex/http.ts`, and `convex/convex.config.ts` own auth and Convex host wiring.
- `convex/functions.ts` owns the app’s backend entry builders.
- `convex/schema.ts` owns tables and indexes.
- `convex/auth/` owns app identity decisions.
- `convex/permissions/` owns frontend permission projection.
- `convex/features/` owns backend feature modules.
- `shared/features/` owns runtime-neutral contract artifacts.
- `app/features/` owns feature-scoped UI code and local mini-Trellis boundaries.
- `server/api/` owns app server routes.
- `server/mcp/` owns MCP projection.

Each top-level feature component should behave like a mini-Trellis boundary: route shell outside, one feature seam inside.

## Starter baselines

All official starters share the same tree. They differ in what they populate first.

### `personal`

Use this when the app has signed-in users but no tenant model yet.

Generated first:

- `shared/features/todos/contract.ts`
- `convex/features/todos/{schema,domain,index}.ts`
- `convex/features/users/{schema,index}.ts`
- `app/features/personal/components/PersonalStarterPage.vue`

### `workspace`

Use this when the app needs roles, tenant isolation, or backend-owned permission projection.

Generated first:

- `convex/auth/principal.ts`
- `convex/features/index.ts`
- `shared/features/todos/contract.ts`
- `convex/features/todos/{schema,permissions,domain,feature,index}.ts`
- `convex/features/workspaces/{schema,domain,feature,index}.ts`
- `convex/features/users/{schema,feature,index}.ts`
- `convex/permissions/context.ts`
- `app/features/workspace/components/WorkspaceStarterPage.vue`

### `workspace --mcp`

MCP is a capability added to `workspace`, not a separate starter.

Additional generated files:

- `convex/features/mcpKeys/domain.ts`
- `server/middleware/mcp-auth.ts`
- `server/mcp/index.ts`
- `server/mcp/runtime.ts`
- `server/mcp/tools/*`

### `cms`

Use this when the app is fundamentally a public content surface plus a signed-in editorial studio.

Generated first:

- `shared/features/pages/contract.ts`
- `convex/features/pages/domain.ts`
- `convex/permissions/context.ts`
- `pages/[slug].vue`
- `pages/studio.vue`

The CMS starter follows the same root-shell plus `features/*` split as the other starters.

## Where new code goes

Keep the structure boring:

- New backend modules go under `convex/features/<name>/`
- New runtime-neutral contracts go under `shared/features/<name>/`
- New feature UI goes under `app/features/<name>/`
- New guard and actor logic goes under `convex/auth/`
- New permission projection goes under `convex/permissions/`
- New MCP tool files go under `server/mcp/tools/`

Do not scatter the same concern across ad-hoc `lib/`, `utils/`, and parallel backend folders.

## Stable now

The official Trellis starter surface today is:

- `trellis init <name> --template public|personal|workspace|cms`
- `trellis add mcp|uploads|operation|entity`
- canonical root shell plus `convex/features|shared/features|app/features`
- optional MCP surface under `server/mcp/`

Future app families may graduate later, but they are not part of the current product contract until they ship.

## Practical test

If two Trellis apps take more than a minute to orient, the platform shape is still too loose.

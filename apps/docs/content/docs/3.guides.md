Use guides when you already know the job you are trying to finish.

Not every guide is equally early-path:

- `Data fetching`, `Mutations`, and `Authentication` are the common first layers.
- `Permissions`, `Server-side`, and `MCP tools` are deeper lanes once the app crosses those boundaries.

::card-group

::card{title="Data fetching" icon="i-lucide-database" to="/apps/docs/data-fetching/queries"}
Queries, pagination, cached detail views, and connection-aware UI behavior.
::

::card{title="Mutations" icon="i-lucide-edit" to="/apps/docs/mutations/mutations"}
State changes, optimistic updates, actions, and destructive UI flows.
::

::card{title="Authentication" icon="i-lucide-lock" to="/apps/docs/auth-security/authentication"}
Better Auth wiring, route protection, auth flows, and troubleshooting.
::

::card{title="Permissions" icon="i-lucide-shield" to="/apps/docs/permissions/setup"}
Workspace-scoped permissions, tenant isolation, and reusable backend-owned policy once the app needs them.
::

::card{title="Server-side" icon="i-lucide-server" to="/apps/docs/server-side/ssr-overview"}
SSR, hydration, server routes, and trusted callers on the same protected model.
::

::card{title="File uploads" icon="i-lucide-upload" to="/apps/docs/file-uploads/single-file-upload"}
Single uploads, queue-based uploads, and storage URL usage.
::

::card{title="MCP tools" icon="i-lucide-bot" to="/apps/docs/mcp-tools/getting-started"}
Expose safe tools without forking the backend permission model.
::

::

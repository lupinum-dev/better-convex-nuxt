Reference for `trellis.mcp`. These are module-level flags only. Tool definitions, runtime auth, delegation, and rate-limit stores live in the MCP runtime files.

```ts [nuxt.config.ts]
trellis: {
  mcp: {
    name: 'My App',
    sessions: false,
  },
}
```

## Option reference

| Option     | Default | Purpose                   |
| ---------- | ------- | ------------------------- |
| `name`     | `''`    | Name shown to MCP clients |
| `sessions` | `false` | Enable MCP session state  |

## What is not configured here

MCP rate limiting is no longer configured through `trellis.mcp`. If a tool uses `rateLimit`, the runtime must provide an explicit external store when you need distributed enforcement.

Use the MCP runtime for that:

```ts [server/mcp/runtime.ts]
import Redis from 'ioredis'
import { createRedisMcpRateLimitStore, defineMcpApp } from '@lupinum/trellis/mcp'

const redis = new Redis(process.env.REDIS_URL!)

export const mcpRuntime = defineMcpApp({
  rateLimitStore: createRedisMcpRateLimitStore({
    client: redis,
  }),
  // ...
})
```

Without `rateLimitStore`, Trellis falls back to process-local memory. That is fine for local work, tests, and single-process throwaway environments. It is not production-grade distributed enforcement.

## Scope

This page is about module config, not tools. If you're defining tools, resources, or prompts, start in the [MCP tools section](/docs/mcp-tools/getting-started).

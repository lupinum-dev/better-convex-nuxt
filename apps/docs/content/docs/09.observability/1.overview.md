Trellis observability isn't "more logs." It's a vocabulary of decisions — every denial, every trust-boundary crossing, every destructive preview/confirm cycle emits a structured event that explains *why*, not just *what*.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  trellis: {
    observability: { enabled: true },
  },
})
```

One flag turns on the full event stream. Delivery runs through `evlog`.

## What Trellis actually traces

| Family | Examples | Why it matters |
| --- | --- | --- |
| Identity | `principal.resolved`, `actor.resolved`, `actor.missing` | Who the runtime believes called |
| Authorization | `guard.allowed`, `guard.denied`, `authorize.denied`, `rls.denied` | Which layer allowed or denied |
| Trust boundary | `db.cross_tenant.used`, `db.raw.used`, `service.access.*` | Every intentional escape hatch |
| Operations | `operation.preview.*`, `operation.confirm.*`, `operation.execute.*` | Destructive flow integrity |
| MCP | `tool.called`, `tool.denied`, `tool.executed`, `tool.failed` | Agent-facing traffic |
| Browser/runtime | `query.*`, `mutation.*`, `upload.*`, `connection.*` | Client-side state transitions |

The shape lets you ask decision-level questions — *who was denied, by which guard, on which resource* — instead of stitching log lines by timestamp.

## Correlation is built in

Events share correlation context across boundaries:

- Browser → Nuxt server → Convex → MCP → services, webhooks.

A denial in one layer stays connected to the request that started it. You don't have to correlate log fragments by timestamp and hope.

::note
The correlation header is configurable. Default is `x-trellis-correlation-id`; set your own if your infra already uses something like `traceparent`.
::

## Configure it at the module level

```ts [nuxt.config.ts]
trellis: {
  observability: {
    enabled: true,
    // capture: ...
    // level: ...
    // sampling: ...
    // correlationHeader: 'traceparent',
    // serviceName: 'my-app',
    // explain: ...
  },
}
```

| Option | Controls |
| --- | --- |
| `capture` | Which surfaces emit (browser, server, Convex, MCP) |
| `level` | Verbosity floor |
| `sampling` | Per-event-family sample rates |
| `correlationHeader` | Header name for request correlation |
| `serviceName` | Service label on emitted events |
| `explain` | Explainability (reason codes, suggested actions) |

See [Configuration: module options](/apps/docs/configuration/module-options) for the full shape.

## Not a generic logging pitch

Trellis doesn't try to replace your app's logging. The value is the event *vocabulary* — a denial is `guard.denied` with a reason code, not a free-form string your log aggregator has to regex for.

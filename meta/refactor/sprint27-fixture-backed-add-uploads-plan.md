# Sprint 27: Fixture-Backed `trellis add uploads`

## Goal

Delete the final `.tpl` template source path.

By the end of this sprint, `trellis add uploads` should render its authored
files from a small fixture-backed add slice, and `src/cli/templates/init` should
be empty or removed.

Retained add behavior:

- add `shared/features/files/contract.ts`;
- add `convex/features/files/domain.ts`;
- add `app/features/uploads/components/UploadsStarterPage.vue`;
- add `app/pages/uploads.vue`.

## Why This Sprint Comes Next

Sprint 26 moved `trellis add mcp` onto the existing `workspace-mcp` fixture and
deleted the MCP `.tpl` files. The only remaining files under
`src/cli/templates/init` are uploads templates:

```text
uploadsContractTemplate.tpl
uploadsDomainTemplate.tpl
uploadsPageTemplate.tpl
```

This is now the last old template-backed CLI source. Deleting it completes the
starter/template source cleanup started in Slice 7.

## Non-Goals

- Do not redesign the upload API.
- Do not replace `mutation.unsafe({ bypass: ... })` with typed permits in this
  sprint. That belongs to the unsafe-permit cleanup slice.
- Do not add schema patching for uploads unless tests show it is already part of
  the supported behavior. This sprint preserves current output only.
- Do not create a generic add-plugin system.
- Do not add app-starter uploads by default.

## Design Target

### Small Add-Slice Fixture

Unlike MCP, uploads is not already part of a retained app starter fixture. The
simplest clean source of truth is a small add-slice fixture:

```text
src/cli/add-fixtures/uploads/
  add.manifest.json
  shared/features/files/contract.ts
  convex/features/files/domain.ts
  app/features/uploads/components/UploadsStarterPage.vue
  app/pages/uploads.vue
```

The route file should be explicit fixture content, not generated by
`routeShellTemplate(...)`. That lets us delete `src/cli/lib/init-templates.ts`
entirely instead of keeping one helper around.

### Reuse Existing Fixture Rendering

The add fixture should reuse the existing manifest renderer where possible:

- include/exclude patterns;
- deterministic file ordering;
- exact file content copied from fixture source;
- loud failure if the fixture or manifest is missing.

If a tiny wrapper helper is needed, keep it boring and local to CLI fixture
rendering. Do not introduce a broader plugin/adaptor abstraction.

### Delete Old Template Directory

Delete:

```text
src/cli/lib/init-templates.ts
src/cli/templates/init/uploadsContractTemplate.tpl
src/cli/templates/init/uploadsDomainTemplate.tpl
src/cli/templates/init/uploadsPageTemplate.tpl
src/cli/templates/init/
```

`src/cli/templates/init` should not remain as an empty compatibility directory.
